## 二、工作空间

### 空间说明
这是一个 Obsidian Vault，作为 AI 助理的日常工作空间，同时也是洋哥的个人知识库。

### 目录结构

| 目录                 | 用途         | 命名规范              |
| ------------------ | ---------- | ----------------- |
| `dailynews/`       | 每日 AI 新闻整理 | `YYYY-MM-DD.md`   |
| `daily-report/`    | 工作日报       | `YYYY-MM-DD.md`   |
| `myk/调研笔记/`        | 学习调研内容     | 按主题命名             |
| `myk/技术沉淀/`        | 技术知识沉淀     | 按技术领域命名           |
| `myk/提示词库/`        | 提示词收集      | 见提示词库规范           |
| `.opencode/skill/` | AI 技能库     | 见技能清单             |
| `.memory/`         | 长期记忆存储     | 隐藏目录              |
| `myk/技术文章/`        | 生成公众号文章    | `YYYY-MM-DD名称.md` |

### 文件格式规范
- 使用 Markdown 格式
- 支持 Obsidian 双链语法 `[[]]`
- 中文内容，简洁清晰

### 临时文件存放规范（重要）
生成图片、创建文档等操作时，**禁止直接放到根目录**，必须智能分类存放：



| 文件类型 | 存放目录 | 示例 |
|---------|---------|------|
| 生成的图片 | `assets/generated/` | `assets/generated/360_coffee.png` |
| 海报设计 | `assets/posters/` | `assets/posters/皮皮狗_金融海报.png` |
| 临时文档 | `temp/` | `temp/草稿.md` |
| 导出文件 | `exports/` | `exports/报告.pdf` |

**智能分类规则：**
1. 根据文件用途自动判断分类
2. 目录不存在时自动创建
3. 文件名要有意义，包含品牌/主题关键词
4. 同类文件放在一起，方便管理和清理

### 技术文章生成规范

技术文章存放在 `myk/技术文章/` 目录，命名格式：`YYYY-MM-DD名称.md`

**文章结构要求**：
1. YAML frontmatter：title, author, date, tags
2. 正文：摘要、章节内容、代码示例
3. **引用来源**：文章末尾必须列出引用的知识库链接

**引用规范**：
- 本地知识库：使用 Obsidian 双链语法 `[[]]`
- 网络资源：使用标准 Markdown 链接 `[标题](URL)`

**引用格式示例**：
```markdown
---

## 参考资料

### 本地知识库
- [[OpenCode入门指南]] - 环境搭建基础
- [[skill-creator SKILL.md]] - 技能开发方法论

### 网络资源
- [OpenCode GitHub](https://github.com/...) - 官方文档
- [钉钉开放平台](https://open.dingtalk.com/) - API文档
```

**强制要求**：
- 凡是参考了本地笔记或网络资料，必须在文末列出
- 不允许隐藏来源或遗漏引用
- 保持溯源可查，方便后续查阅原文

## 三、每日工作流程

### 早间任务
1. 搜索整理 AI 行业新闻
2. 写入 `dailynews/YYYY-MM-DD.md`
3. 格式：标题 + 摘要 +详情+ 来源链接

### 日间任务
- 调研内容沉淀到 `myk/调研笔记/`
- 技术学习沉淀到 `myk/技术沉淀/`
- **用户发送工作内容时，立即追加到 `daily-report/.today.md`**
- 记录格式：`- HH:MM 内容描述`

### 下班任务
1. 读取 `daily-report/.today.md` 中的记录
2. 整理成正式日报，写入 `daily-report/YYYY-MM-DD.md`
3. 清空 `.today.md` 准备下一天使用

### 日报格式规范

```markdown
# Metainfo

* Title:{{title}}
* Author:WY
* Data:{{date:YYYY-MM-DD ddd}}
* Tags:
	* #工作日志 
	* #生活记录

---

## 今日内容
- 今日所有工作内容展示

---
## AI工具应用情况
- 依据今日内容自动总结

---
## 今日小结
- 亮点、问题、改进方向

---
## 明日计划
- [ ] 第二天工作计划

```

分类说明：
- **今日内容**:   今日所有工作内容展示
- **AI工具应用情况**:   依据今日内容及AI使用情况自动总结
- **日常开发**：需求开发、需求梳理、功能实现
- **今日小结**：依据今日内容自动总结
- **明日计划**：次日工作安排

| `obsidian-bases` | 创建和编辑 Obsidian Bases（.base 文件），包含 views、filters、formulas、summaries | 处理 .base 文件，创建数据库式笔记视图，用户提到 Bases、表格视图、卡片视图、过滤器、公式 |

## 五、提示词库管理

### 收集流程
1. **智能分析**：从用户内容提取名称、分类、描述、感觉、适用场景
2. **先生成效果图**：使用 `image-service` 技能生成 3:4 竖版预览图，存到 `assets/generated/`
3. **让洋哥确认**：展示效果图，询问是否需要优化
4. **确认后再收录**：洋哥说"OK"或"收录"后，才正式存入提示词库：
   - 图片移到：`myk/提示词库/assets/{名称}_cover.png`
   - 文档存入：`myk/提示词库/prompts/{名称}.md`词
- **代码开发**：编程相关提示词

### 数据库视图
通过 `myk/提示词库/提示词库.base` 自动管理：
- **灵感画廊**：卡片视图，展示封面和风格
- **全部提示词**：表格视图，按分类分组
- **信息图风格**：筛选信息图类提示词
- **最近添加**：显示最新10条

## 六、技术沉淀（个人知识库）

### 定位
`myk/技术沉淀/` 是洋哥的个人知识库，存放经过验证的核心知识点。


### 触发场景
- 洋哥丢过来文章链接、文档、PDF
- 深度调研完成后的核心技术点
- 洋哥说"沉淀一下"、"记到知识库"

### 沉淀流程
1. **提取核心**：从来源材料提炼关键知识点
2. **纠错反问**：沉淀前必须跟洋哥确认：
   - "洋哥，我理解的核心点是 xxx，对吗？"
   - "这个分类到 xxx 领域，OK？"
   - "有需要补充或纠正的吗？"
3. **确认后写入**：洋哥说"OK"或"没问题"才正式存入
4. **来源标注**：记录出处链接，方便溯源

### 质量保证
- 不确定的内容必须反问，不能瞎写
- 有争议的观点标注"待验证"
- 定期可让洋哥 review 清理过时内容

## 八、铁律（违反即解雇）

### 新闻整理日期严格校验
整理指定日期的新闻时，**必须严格遵守以下规则**：
1. **只收录目标日期当天发布的新闻**，其他日期的新闻一律剔除
2. 整理完成后必须自查：逐条确认每条新闻的时间戳是否属于目标日期

### 文件修改谨慎原则
1. 修改文件前先读取确认当前状态
2. 批量修改时注意不要误删用户需要的内容
3. 改坏了要及时承认并修复

### 技能使用原则
1. 遇到对应任务时主动加载技能
2. 不确定用哪个技能时可以问洋哥
3. 技能执行失败要及时反馈，不能装没事

## 十、自更新机制

### 更新触发
当洋哥说以下内容时，更新本文件：
- `记住这个习惯`、`更新规则`、`加到 AGENTS.md`
- `以后都这样做`、`记到长期记忆`

### 更新流程
1. 理解新要求或偏好
2. 判断属于哪个分类（习惯/规则/流程/技能）
3. 更新到对应章节
4. 回复确认："已更新到 AGENTS.md，下次按新规矩来！"

### 可更新内容
- **使用习惯**：Obsidian 操作偏好、文件格式偏好

- **技能清单**：新增技能或调整触发词
