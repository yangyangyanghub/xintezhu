# 城市体检与城市更新一体化平台实施计划

> 依据规格：`docs/superpowers/specs/2026-06-03-城市体检更新平台-design.md`
> 目标：一期 MVP，可演示、可测试、可验收。
> 边界：不做完整 CIM、全量实时物联接入、财政拨付、复杂 AI 自动决策和规划编制软件替代。

## 1. 一期范围

一期只做支撑年度城市体检和初步城市更新闭环的 MVP：

1. 组织、用户、角色、权限。
2. 住房、小区社区、街区、城区城市四级体检对象库。
3. 指标库、指标版本、指标填报、简单公式计算、异常检查。
4. PC 填报与移动采集 MVP。
5. 数据导入、字段映射、基础质检。
6. 问题清单、整治建议、整改流转。
7. GIS 一张图基础图层与专题展示。
8. 城市更新资源库、项目库、项目优先级排序。
9. 年度计划与资金申报支撑信息。
10. Excel / Word / PDF 成果导出与报送包。

## 2. 推荐文件结构

如果新建平台，建议结构如下；如果接入现有系统，按现有目录落位，但保持职责边界一致。

```text
city-health-renewal-platform/
  apps/web/                  # 前端应用
  apps/api/                  # 后端 API
  packages/domain/           # 领域模型、枚举、接口定义
  packages/validation/       # 表单、导入、指标校验规则
  packages/report-templates/ # 报告与清单模板
  data/dictionaries/         # 指标、问题、资源、项目字典
  data/seeds/                # 示例数据
  docs/api/                  # API 文档
  docs/data-standards/       # 数据标准
  docs/user-manual/          # 用户手册
  tests/e2e/                 # 端到端测试
  tests/fixtures/            # 测试数据
```

## 3. 核心领域模型

- `Organization`：市、县区、街道、社区、行业部门。
- `User` / `Role` / `Permission`：用户、角色、组织权限、空间权限、流程权限、数据权限、配置权限。
- `HealthObject`：统一体检对象，派生住房、小区社区、街区、城区城市对象。
- `SpatialBoundary`：点、线、面空间对象。
- `Indicator` / `IndicatorVersion` / `IndicatorFormula` / `IndicatorValue`：指标定义、版本、公式和结果。
- `Issue` / `IssueCategory` / `IssueSeverity` / `IssueAssignment`：问题、分类、等级、派发与整改。
- `RectificationSuggestion`：整治建议模板和实例。
- `RenewalResource` / `RenewalProject` / `ProjectPriorityScore`：更新资源、项目和优先级。
- `AnnualPlan` / `FundingApplication`：年度计划和资金申报支撑。
- `ReportTemplate` / `ExportJob` / `SubmissionPackage`：模板、导出任务、报送包。

## 4. 实施阶段

### 阶段 0：项目基线

**任务 0.1 确认技术栈**

- 确认前端、后端、数据库、GIS、报告导出、部署方式。
- 默认建议：Vue3 + OpenLayers + FastAPI 或 Spring Boot + PostgreSQL/PostGIS。
- 写入技术说明。

验证：技术说明包含选型、理由、风险和替代方案。

**任务 0.2 创建项目骨架**

- 创建前端、后端、领域模型、校验、模板、测试数据目录。
- 配置 lint、format、test 命令。
- 创建健康检查接口和首页。

验证：前端、后端可启动；测试命令可运行。

### 阶段 1：组织、用户与权限

**任务 1.1 写测试**

- 测试组织层级、组织编码唯一性、组织范围查询。
- 测试不同角色访问不同组织范围。
- 测试管理员才能修改指标、字典和模板。

验证：测试先失败。

**任务 1.2 实现功能**

- 实现组织、用户、角色、权限实体。
- 实现组织层级查询。
- 在 API 层接入权限校验。

验证：组织与权限测试通过，未授权访问返回明确错误。

### 阶段 2：四级体检对象库

**任务 2.1 写测试**

- 测试住房、小区社区、街区、城区城市对象。
- 测试对象与组织、空间对象关联。
- 测试对象编码唯一性。

验证：测试先失败。

**任务 2.2 实现对象模型和接口**

- 创建体检对象实体和空间字段或空间引用。
- 实现对象增删改查。
- 支持按组织、对象类型、空间范围查询。

验证：对象测试通过。

**任务 2.3 实现对象导入与地图展示**

- 支持 Excel 导入对象清单。
- 支持 SHP / GeoJSON 导入空间边界。
- 前端地图展示四级对象图层，支持筛选。

验证：示例对象能导入、查询、正确上图。

### 阶段 3：指标体系与计算

**任务 3.1 写测试**

- 测试指标名称、编码、维度、类别、阈值、数据来源、版本。
- 测试指标与体检对象关联。

验证：测试先失败。

**任务 3.2 实现指标库**

- 实现指标定义、版本、数据来源实体。
- 实现指标增删改查与维度、类别筛选。
- 导入国家基础、省级特色、市级特色、专项指标示例。

验证：指标可按维度和类别检索。

**任务 3.3 实现计算与异常检查**

- 支持人工填报指标。
- 支持简单公式指标。
- 预留空间统计指标接口。
- 支持缺失值、阈值和异常状态检查。

验证：给定示例输入，输出确定结果；异常指标可筛选。

### 阶段 4：数据采集与质检

**任务 4.1 表单 schema**

- 定义文本、数字、枚举、日期、坐标、照片、附件字段。
- 定义必填、范围、枚举规则。

验证：schema 能被校验器加载。

**任务 4.2 PC 填报与移动采集**

- 前端根据 schema 渲染填报表单。
- 后端保存草稿和提交记录。
- H5 或小程序支持定位、拍照、表单提交、任务列表。

验证：PC 和移动端提交数据均进入后台。

**任务 4.3 数据质检**

- 实现必填、类型、空间坐标缺失、附件完整性校验。

验证：错误数据被拦截并提示原因。

### 阶段 5：问题清单与整治建议

**任务 5.1 写测试**

- 测试问题类型、等级、来源、位置、责任单位、状态。
- 测试问题与对象、指标结果关联。

验证：测试先失败。

**任务 5.2 实现问题清单**

- 实现问题实体和接口。
- 支持人工录入、指标异常转问题、地图定位。

验证：问题能创建、查询、筛选、上图。

**任务 5.3 实现分级和建议模板**

- 配置安全隐患、民生急难愁盼、设施短板、空间低效、历史文化、管网生命线分类。
- 按问题类型生成默认整治建议。
- 支持人工调整建议。

验证：不同类型问题能自动带出处理策略和建议。

**任务 5.4 实现派发与销号**

- 支持责任单位认领、整改反馈、审核销号。

验证：问题状态能从新建流转到已销号。

### 阶段 6：GIS 一张图

**任务 6.1 图层管理**

- 支持体检对象、问题、更新资源、项目图层。
- 支持图层开关和按类型筛选。

验证：图层可开关、筛选、定位。

**任务 6.2 专题图与基础分析**

- 实现问题地图、意愿地图、更新资源地图、项目分布图。
- 实现设施服务半径分析。
- 15 分钟生活圈先做接口和示例结果。
- 管网隐患、危旧房分布先按图层统计实现。

验证：专题图和统计面板能加载示例数据。

### 阶段 7：更新资源与项目库

**任务 7.1 写测试**

- 测试资源类型、位置、权属、现状、需求、附件。
- 测试资源与体检对象、问题关联。

验证：测试先失败。

**任务 7.2 实现资源库和项目库**

- 实现更新资源录入、导入、查询、地图展示。
- 实现项目实体。
- 支持问题转项目、资源转项目、片区转项目。
- 支持项目类型、投资估算、实施主体、状态管理。

验证：示例资源能入库上图；问题和资源能生成项目。

**任务 7.3 实现项目优先级评分**

- 定义安全风险、群众诉求、影响人口、政策匹配度、资金可获得性、实施难度、投资规模、重点片区评分字段。
- 实现可配置权重。

验证：相同输入产生稳定、可解释的排序。

### 阶段 8：年度计划与资金申报

**任务 8.1 年度计划**

- 支持项目加入年度计划。
- 支持年度、批次、责任单位、计划状态。

验证：项目能按年度筛选。

**任务 8.2 资金申报信息**

- 支持专项债、中央预算内投资、超长期特别国债、城市更新基金等资金类型。
- 管理申报材料、投资估算、资金来源、绩效目标、资金到位和拨付进度信息。
- 不执行实际拨付。

验证：项目能关联资金申报信息，资金进度仅作为管理信息。

### 阶段 9：成果导出与报送

**任务 9.1 模板字段设计**

- 定义年度体检报告、问题清单、整治建议清单、项目库、成效评估报告模板字段。

验证：模板字段与系统字段可映射。

**任务 9.2 Excel 导出**

- 导出问题清单、整治建议清单、项目库。

验证：导出文件字段完整、行数正确。

**任务 9.3 Word / PDF 报告导出**

- 使用模板生成年度体检报告。
- 支持按年度、行政区、专项模块生成报告。

验证：报告可打开，章节和关键数据正确。

**任务 9.4 报送包**

- 按年度生成报送包。
- 包含报告、清单、指标表和附件目录。

验证：报送包内容完整，文件命名符合规则。

### 阶段 10：专项体检模块

**任务 10.1 通用专项框架**

- 每个专项共享指标、对象、采集、问题、报告、转项目能力。
- 不为每个专项复制一套流程。

验证：新增专项类型不需要修改核心流程。

**任务 10.2 一期优先专项**

- 优先实现地下管网、排水防涝、完整社区。
- 历史文化、燃气安全、重点更新片区、城市更新资源盘点、燕赵宜居县城作为配置预留。

验证：优先专项能配置指标、采集数据、生成问题和报告。

## 5. 测试策略

1. 领域模型测试：组织、对象、指标、问题、资源、项目。
2. 权限测试：不同角色的数据可见性和操作权限。
3. 导入测试：Excel、GeoJSON、SHP 示例数据导入。
4. 指标测试：人工填报、简单公式、异常检查。
5. 流程测试：采集 → 指标 → 问题 → 建议 → 项目 → 年度计划。
6. GIS 测试：图层加载、筛选、定位、专题图展示。
7. 导出测试：Excel、Word、PDF、报送包。
8. e2e 测试：市级管理员、县区用户、街道社区用户、行业部门用户四条主流程。

## 6. 完成标准

一期完成必须满足：

1. 四级体检对象可建库、查询、导入、落图。
2. 指标可配置、填报、计算、异常检查、导出。
3. PC 和移动采集可完成基础任务。
4. 问题可分类分级、派发、整改、销号。
5. GIS 一张图可展示对象、问题、资源、项目。
6. 更新资源可入库，问题和资源可生成项目。
7. 项目可进入年度计划并关联资金申报信息。
8. 体检报告、问题清单、整治建议清单、项目库可导出。
9. 权限控制覆盖市、县区、街社、行业部门、第三方、管理员。
10. 核心自动化测试通过。

## 7. 执行注意事项

- 每个阶段先写失败测试，再实现最小功能。
- 每个任务完成后运行相关测试。
- 不要把专项体检写成重复模块，应抽象为共享专项框架。
- 不要在一期承诺实时物联、完整 CIM、复杂 AI 和财政拨付。
- GIS 能力先满足业务闭环，不追求炫技展示。
- 报告导出字段要优先对齐验收材料，而不是追求排版复杂度。
- 每个阶段完成后形成一次可演示增量。
