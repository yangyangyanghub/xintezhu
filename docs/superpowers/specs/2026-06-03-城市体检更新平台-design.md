# 城市体检与城市更新一体化平台设计规格

## 1. 平台定位

城市体检与城市更新平台不应只是填报系统，而应是城市更新全过程数字化工作平台。

核心闭环：

```text
体检发现问题 → 更新解决问题 → 项目资金衔接 → 实施成效评估 → 年度滚动复检
```

平台以城市体检为入口，以问题清单为抓手，以更新项目库为核心，以资金申报和实施监管为落点，支撑“先体检后更新、无体检不更新”的工作机制。

建设目标：

1. 支撑年度城市体检组织、采集、计算、分析、报告和报送。
2. 建立人口、建筑、基础设施、公共服务、城市更新资源等底图底数。
3. 支撑住房、小区社区、街区、城区城市四级体检。
4. 支撑地下管网、排水防涝、完整社区、历史文化、燃气安全、重点更新片区等专项体检。
5. 将体检问题转化为整治建议、城市更新项目和年度更新计划。
6. 支撑项目申报、资金谋划、实施监管和成效评估。
7. 形成可年度滚动更新的数据资产和工作机制。

## 2. 设计原则

1. **先体检后更新**：所有更新项目应能追溯到体检问题、群众诉求、资源潜力或专项体检结论。
2. **问题导向与目标导向结合**：既识别群众急难愁盼问题，也评估城市韧性、安全、服务、生态、文化和发展质量。
3. **一张图统筹**：体检对象、问题点位、更新资源、项目范围和实施进度应尽量空间化表达。
4. **数据标准先行**：同步建设四级体检对象编码、指标字典、采集表单、数据格式、质检规则和成果模板。
5. **分期建设**：一期支撑体检交付，二期打通更新闭环，三期扩展 CIM、数字孪生和智能分析。

## 3. 用户角色与权限

| 角色 | 核心职责 | 主要功能 |
|---|---|---|
| 市级住建主管部门 | 统筹年度体检和更新工作 | 工作调度、成果审核、全市态势、项目库统筹 |
| 县区住建部门 | 组织本辖区体检和整改 | 任务分派、数据审核、问题确认、项目入库 |
| 街道 / 社区 | 配合现场调查和基层治理 | 现场核查、居民诉求、整改反馈、满意度回访 |
| 行业主管部门 | 提供专题数据并认领整改责任 | 数据填报、问题认领、整改计划、专项体检 |
| 第三方技术团队 | 提供技术支撑和成果编制 | 数据采集、指标计算、空间分析、报告编制 |
| 专家评审 | 把关成果质量 | 指标审查、问题诊断审查、报告评审 |
| 公众 / 居民 | 参与体检和反馈成效 | 问卷、诉求上报、改造意愿、满意度评价 |
| 平台管理员 | 维护平台基础配置 | 用户、权限、字典、流程、模板、日志 |

权限体系至少包括组织权限、空间权限、流程权限、数据权限和配置权限。

## 4. 业务闭环

### 4.1 主流程

```text
工作组织
  ↓
体检对象建库
  ↓
数据采集与部门汇交
  ↓
数据质检与标准化
  ↓
指标计算与空间分析
  ↓
问题诊断
  ↓
问题清单与整治建议
  ↓
更新资源识别
  ↓
更新项目生成
  ↓
年度计划与资金申报
  ↓
项目实施监管
  ↓
整改销号与成效评估
  ↓
年度复检与滚动更新
```

### 4.2 三类闭环

| 闭环 | 流程 | 成果 |
|---|---|---|
| 体检闭环 | 数据采集 → 指标计算 → 问题清单 → 体检报告 | 体检数据库、问题清单、体检报告 |
| 更新闭环 | 问题清单 → 整治建议 → 项目库 → 年度计划 | 更新资源库、项目库、年度计划 |
| 治理闭环 | 项目实施 → 整改销号 → 成效评估 → 年度复检 | 整改台账、成效评估、复检结论 |

## 5. 功能模块

### 5.1 工作组织与协同调度

年度工作方案管理、市县区街社多级任务分派、部门责任清单、进度看板、催办预警、数据报送审核、专家评审、成果报送归档、工作日志和操作留痕。

### 5.2 用户角色与权限管理

用户管理、组织机构管理、角色管理、数据范围授权、操作权限控制、流程节点授权、登录与操作日志。

### 5.3 四级体检对象底图

建立住房、小区 / 社区、街区、城区 / 城市四级对象库，支持行政区、街道、社区、小区、楼栋对象管理，建成区边界管理，街区单元划定，重点更新片区边界管理，对象编码、空间落图和历史体检记录。

### 5.4 数据采集与移动调查

数据采集模块应同时支持“自由空间采集”和“矢量任务下发调查”两种方式，不应只做单一填报表单。

**方式一：自由空间采集。** 适用于外业现场发现的新问题、新资源、新设施和临时补采对象。移动端或 PC 端地图应支持点选、画线、画面、编辑几何、删除几何、自动计算长度面积、自动叠加行政区 / 社区 / 片区归属，并联动动态表单填写属性、照片和附件。

**方式二：矢量任务下发调查。** 适用于内业已整理好的待核查对象，如老旧小区边界、疑似危旧房点位、排水管线、燃气管线、历史建筑点位、闲置地块、重点更新片区等。PC 端先导入 SHP / GeoJSON / GDB / CAD 转换成果，完成字段映射、任务图层生成和人员分配；移动端领取任务后按图斑、点位或线段逐个核查，回填调查结果、修正几何、上传照片并提交质检。

采集模块能力包括 PC 端部门填报、Excel / 矢量导入、字段映射、任务图层管理、任务分派、移动端现场核查、地图点选、画线、画面、定位拍照、附件上传、离线采集与联网同步、问卷调查、居民改造意愿采集、数据复核和质检退回。

一期建议优先采用 H5 或小程序形态，避免一开始建设重型 App；但地图组件必须预留离线任务包、矢量缓存和几何修正能力。

### 5.5 数据汇聚、标准化与质检

支持 Excel、SHP、GeoJSON、CAD、PDF、图片等数据导入，预留 API 接入。提供字段映射、数据清洗、必填校验、异常值校验、空间拓扑检查、重复对象检查、附件完整性检查、数据版本管理和来源追溯。

### 5.6 指标体系与自动计算

指标体系采用：

```text
国家基础指标 + 省级特色指标 + 市级特色指标 + 县区自定义指标 + 专项体检指标
```

邯郸本地可采用：

```text
55 个基础指标 + 6 个河北省特色指标 + 8 个邯郸市特色指标 + 专项指标
```

功能包括指标库管理、指标版本管理、公式配置、阈值配置、数据来源配置、自动计算、人工修正、趋势分析、空间分布分析和异常预警。

### 5.7 GIS 一张图与空间分析

城市体检一张图、问题地图、意愿地图、更新资源地图、更新潜力地图、项目分布图、成效评估图、15 分钟生活圈分析、公共服务设施覆盖分析、管网隐患分布分析、危旧房分布分析和重点片区画像。

### 5.8 综合体检与专项体检管理

专项模块包括地下管网、排水防涝、完整社区、历史文化、燃气安全、重点更新片区、城市更新资源盘点、燕赵宜居县城专项体检。

每个专项模块应支持专项指标配置、专项对象建库、专项数据采集、专项问题识别、专项报告生成和专项问题转项目。

### 5.9 问题清单与整治建议

问题自动识别、人工录入、分类分级、安全隐患标记、群众急难愁盼问题标记、责任单位匹配、整治建议模板、整改时限配置、整改跟踪和问题销号。

问题分级建议：

| 类型 | 处理策略 |
|---|---|
| 安全隐患类 | 立即整改，限期销号 |
| 民生急难愁盼类 | 限期整改，纳入督办 |
| 设施短板类 | 纳入年度更新计划 |
| 空间低效类 | 纳入片区策划 |
| 历史文化类 | 纳入保护更新专项 |
| 管网生命线类 | 纳入专项工程项目 |

### 5.10 城市更新资源与项目库

资源类型包括危旧房、非成套住房、老旧小区、城中村、老旧街区、老旧厂区、闲置低效空间、桥下空间、边角地、插花地、夹心地、历史建筑、历史街区、工业遗产、公共空间、绿地、滨水空间、慢行系统。

功能包括更新资源建库、空间落图、潜力评价、问题转项目、资源转项目、片区转项目、项目优先级排序、投资估算、项目储备和项目状态管理。

项目优先级综合安全风险、群众诉求强度、影响人口、政策匹配度、资金可获得性、实施难度、投资规模、是否位于重点更新片区等因素。

### 5.11 资金申报与年度计划

年度城市更新计划管理、专项债项目储备、中央预算内投资项目储备、超长期特别国债项目储备、城市更新基金项目储备、项目申报材料库、投资估算、资金来源管理、资金到位情况、资金拨付进度和绩效目标管理。

平台不直接替代财政资金拨付系统，只管理资金计划、申报支撑和拨付进度信息。

### 5.12 实施监管、成效评估与成果报送

项目进度监管、项目节点预警、现场照片反馈、设计 / 施工 / 监理资料归档、整改销号、指标改善评估、群众满意度回访、年度复检、成效评估报告、省市县成果报送包生成。

## 6. 数据体系

| 数据库 | 主要内容 |
|---|---|
| 体检对象库 | 住房、小区、社区、街区、城区 |
| 人口数据库 | 常住人口、老年人口、儿童人口、人口密度 |
| 建筑数据库 | 楼栋、建成年代、结构、层数、面积、安全状态 |
| 公共服务设施库 | 教育、医疗、养老、托育、文化、体育、商业 |
| 市政基础设施库 | 道路、排水、供水、燃气、供热、照明、环卫 |
| 地下管网库 | 管线类型、材质、管径、年代、隐患 |
| 历史文化资源库 | 历史建筑、街区、文物、工业遗产、古树名木 |
| 城市更新资源库 | 危旧房、老旧小区、老旧厂区、低效空间 |
| 问题台账库 | 问题类型、位置、等级、责任单位、整改状态 |
| 采集任务库 | 任务图层、任务对象、下发范围、调查人员、核查状态、几何修正记录 |
| 更新项目库 | 项目名称、类型、投资、资金、进度、成效 |

## 7. 指标体系

| 维度 | 指标方向 |
|---|---|
| 住房 | 安全耐久、功能完备、绿色智能 |
| 小区社区 | 设施完善、环境宜居、管理健全 |
| 街区 | 功能完善、整洁有序、特色活力 |
| 城区城市 | 生态宜居、历史文化保护利用、产城融合、职住平衡、安全韧性、智慧高效 |

指标配置字段至少包括指标名称、指标编码、所属维度、指标类别、计算公式、目标阈值、数据来源、体检范围、责任部门、更新时间、结果解释、对应问题类型、对应整治建议模板。

## 8. 成果交付物

交付物包括平台软件系统、城市体检数据库、四级体检对象库、指标体系配置表、数据标准与字段字典、移动采集表单、问题清单、整治建议清单、更新资源库、更新项目库、专项体检成果、年度城市体检报告、年度城市更新计划、成效评估报告、操作手册、培训材料和运维方案。

## 10. 分期建设

### 10.1 一期：体检采集与评估 MVP

目标是支撑年度体检交付。范围包括对象建库、数据采集、数据质检、指标计算、问题清单、GIS 一张图、报告导出和成果报送。

### 10.2 二期：城市更新闭环平台

目标是打通问题到项目、资金、整改和成效评估。范围包括整治建议库、更新资源库、更新项目库、年度计划、资金申报支撑、项目监管和成效评估。

### 10.3 三期：CIM / 数字孪生与智能决策

目标是提升展示、推演和智能分析能力。范围包括 CIM 三维底座、城市生命线监测接入、AI 报告生成、智能项目策划和多年趋势预测。

## 11. 一期明确不做

一期不做全量实时物联感知接入，不做完整 CIM 精模城市，不承诺所有部门系统实时互联，不做复杂 AI 自动决策，不替代各部门法定审批系统，不直接处理资金拨付，不代替规划编制软件。

## 12. 验收标准

一期验收建议关注：

1. 四级体检对象能够建库、查询、落图。
2. 指标体系能够配置、填报、计算和导出。
3. 移动采集能够完成定位、拍照、表单填写和数据同步，并支持地图点选、画线、画面及矢量任务下发调查。
4. 问题清单能够分类分级、定位上图、分派责任和跟踪状态。
5. GIS 一张图能够展示体检对象、问题、资源和项目。
6. 年度体检报告、问题清单、整治建议清单、项目库等成果能够模板化导出。
7. 平台能够支持市、县区、街道社区和部门多角色协同。

## 13. 后续待确认事项

正式进入实现计划前，需要进一步确认：

1. 首期服务对象是市级平台、县区平台，还是市县一体化平台。
2. 一期是否必须包含移动端，移动端采用 H5、小程序还是原生 App。
3. 现有政务系统是否已有统一身份认证、GIS 底图、CIM 平台或数据共享接口。
4. 一期优先覆盖哪些专项体检模块。
5. 报告模板、报送格式和验收材料是否已有固定要求。
