# DESIGN.md - 登录页面设计规范

> 文档版本：v1.0  
> 创建日期：2026-04-11  
> 页面类型：登录页  
> 适用范围：用户身份验证入口

---

## 页面目标

- **核心目标**：让用户快速、清晰地完成身份验证，进入应用主流程
- **业务目标**：降低登录门槛，提升转化率，减少因登录流程复杂导致的用户流失
- **体验目标**：
  - 表单填写直观流畅，无需用户反复尝试或猜测
  - 错误提示即时清晰，帮助用户快速修正
  - 安全可信，传递专业可靠的品牌印象
- **设计原则**：
  - 信息层级清晰：用户一眼识别核心操作区域
  - 视觉简洁克制：避免装饰干扰注意力
  - 状态反馈明确：每个交互都有可预期的视觉回应

---

## 1. Visual Theme

- **Design Intent**：现代极简风格，突出安全可靠的品牌气质。通过克制的设计语言降低认知负担，让用户快速聚焦登录任务本身。
- **Style Keywords**：极简主义、信任感、专业、清晰、对称、克制
- **Reference Notes**：
  - 参考风格：Notion 登录页（极简居中）、Stripe 登录页（信任感强）
  - 避免风格：花哨动效、复杂背景、过度装饰的复古风格

---

## 2. Color Palette

- **Primary Colors**：
  - **品牌主色**：用于登录按钮、关键链接，传递品牌识别
  - **背景色**：浅色（如白色或极浅灰）或深色主题下的深灰，保持高对比度
- **Secondary Colors**：
  - **辅助色**：用于次要文本、分割线、边框
  - **状态色**：
    - 成功：绿色系（用于验证成功提示）
    - 错误：红色系（用于错误提示、验证失败）
    - 禁用：灰度降低（用于不可用状态的按钮）
- **Usage Rules**：
  - 背景保持低干扰，确保表单区域高对比度
  - 文本层级通过灰度区分，而非颜色堆砌
  - 状态色仅用于反馈，避免滥用影响整体色调

---

## 3. Typography

- **Type System**：
  - **标题级**：页面标题"登录"或品牌名，字号偏大（24-32px），粗体或中等粗度
  - **正文级**：表单标签、输入提示，16px 左右，中等粗度
  - **辅助级**：帮助文本、错误提示、隐私声明，14px，轻粗度
- **Font Guidance**：
  - 推荐：系统字体栈（-apple-system, BlinkMacSystemFont, 'Segoe UI'）或现代无衬线字体（Inter, Roboto）
  - 避免：衬线字体、手写字体、装饰性字体
- **Text Rules**：
  - 行高：正文 1.5-1.6，标题 1.2-1.3
  - 字距：保持默认或略微增加，确保可读性
  - 最大行长：表单标签单行，避免换行影响对齐

## 4. Components

### Core Components

| 组件 | 用途 | 关键属性 |
|------|------|---------|
| 邮箱输入框 | 用户邮箱地址 | 必填，email 类型，格式验证 |
| 密码输入框 | 用户密码 | 必填，password 类型，可选显示/隐藏切换 |
| "记住我"复选框 | 保持登录状态 | 可选，默认不勾选 |
| 登录按钮 | 提交表单 | 主按钮样式，加载状态，禁用状态 |
| "忘记密码？"链接 | 密码找回入口 | 文本链接，次要视觉层级 |
| 第三方登录按钮 | 社交账号登录 | 占位状态，图标 + 文案 |

### States

**表单状态**：
- **默认**：输入框空白，按钮可点击，无错误提示
- **填写中**：用户正在输入，可选实时验证反馈
- **验证失败**：输入格式错误（如邮箱格式不对），显示错误提示
- **提交中**：按钮显示加载动画，禁用点击
- **成功**：表单提交成功，跳转或显示成功反馈
- **错误（服务端）**：登录失败（如密码错误），显示错误提示

**按钮状态**：
- **默认**：主色背景，白色文本，可点击
- **悬停**：背景色加深 10-15%，微光标提示
- **禁用**：灰度降低，不可点击（如表单未填写完整）
- **加载中**：显示旋转图标或进度条，文本改为"登录中..."

### Behavior Notes

- **表单验证**：
  - 邮箱：实时或提交时验证格式
  - 密码：提交时验证（如长度、复杂度）
  - 错误提示：显示在输入框下方或右侧，红色文本
- **交互反馈**：
  - 输入框聚焦：边框高亮，可选微弱阴影
  - 按钮点击：视觉反馈（悬停 → 点击 → 加载 → 成功）
  - "记住我"：勾选/取消勾选的视觉切换
- **一致性规则**：
  - 所有输入框统一样式（边框、高度、内边距）
  - 错误提示统一位置和格式
  - 链接统一样式（颜色、下划线规则）

## 5. Layout

### Information Hierarchy

**优先级顺序**（从高到低）：
1. **登录表单**（邮箱、密码、登录按钮）—— 核心操作区
2. **"记住我" + "忘记密码？"** —— 辅助操作区
3. **第三方登录占位** —— 次要入口区
4. **品牌/标题** —— 视觉锚点（可选）

### Page Structure

**布局方案**：居中对称布局
- **背景**：大面积留白或浅色背景，传递简洁感
- **核心区域**：登录表单居中，宽度固定（如 360-400px）
- **辅助元素**：
  - "记住我"和"忘记密码？"横排在登录按钮下方或输入框区域底部
  - 第三方登录占位在表单下方或底部边缘
- **可选元素**：
  - 页面顶部：品牌 logo 或标题"登录"
  - 页面底部：隐私声明链接（如"使用条款"、"隐私政策")

### Spacing System

- **表单内间距**：
  - 输入框之间：16-24px
  - 输入框到"记住我"：12-16px
  - "记住我"到登录按钮：16-20px
  - 登录按钮到第三方登录：24-32px
- **表单与边距**：
  - 表单容器内边距：24-32px
  - 表单与页面边缘：根据居中位置，两侧等距留白

### Content Flow

- **阅读顺序**：从上到下，表单 → 辅助操作 → 第三方登录 → 底部声明
- **视觉焦点**：登录按钮通过颜色、大小或位置成为最大焦点
- **关键路径**：用户视线从输入框 → 登录按钮，形成自然的操作流

## 6. Depth

- **Elevation Strategy**：
  - **表单容器**：可选微弱阴影或边框，浮于背景之上，传递"卡片"感
  - **背景**：纯色或极浅渐变，保持平面
  - **按钮**：主按钮可有微弱阴影，强化可点击感
- **Borders and Surfaces**：
  - 输入框：细边框（1-2px），聚焦时加粗或变色
  - 表单容器：可选细边框或无边框（极简风格）
- **Motion Cues**：
  - 表单提交：按钮加载动画（旋转图标或进度条）
  - 错误提示：可选微弱震动或闪烁提示
  - 输入聚焦：边框颜色渐变过渡

---

## 7. Icon System

- **Icon Source**：
  - 推荐：Lucide Icons 或 Heroicons（开源、现代、线条清晰）
  - 备选：Phosphor Icons（风格细腻）
- **Usage Guidelines**：
  - **邮箱图标**：可选，显示在输入框左侧或标签旁
  - **密码图标**：可选，显示"显示/隐藏"切换图标
  - **第三方登录**：社交平台图标（Google, GitHub, WeChat 等）
  - **尺寸**：16-20px（小图标），24px（主要图标）
  - **线条粗细**：1.5-2px，保持视觉一致
- **Conflict Handling**：
  - 仅选用一个图标库，避免混用风格冲突
  - 如需多图标库，通过命名空间区分（如 `lucide:mail`, `hero:lock`）

---

## 8. Responsive

- **Breakpoints**：
  - **桌面端**：>= 768px，表单居中，宽度固定（360-400px）
  - **移动端**：< 768px，表单宽度自适应（90-100%宽度），减少侧边距
- **Adaptation Rules**：
  - **布局**：桌面居中对称，移动端表单占满宽度，居中变单列
  - **间距**：移动端减少表单内间距（如 12-16px）
  - **输入框**：移动端高度略微增加（48-56px），方便触摸操作
  - **按钮**：移动端按钮高度增加（48-56px），宽度占满表单宽度
- **Priority Changes**：
  - **桌面端**：所有元素等权重展示
  - **移动端**：
    - 登录表单为核心，第三方登录可折叠或简化展示
    - 底部隐私声明可省略或简化文本

## 交互规则

### 表单验证规则

- **邮箱验证**：
  - 格式验证：符合 email 格式（如 user@example.com）
  - 验证时机：提交时验证，或实时验证（用户输入后延迟 300-500ms）
  - 错误提示："请输入有效的邮箱地址"
- **密码验证**：
  - 格式验证：最小长度（如 6 字符），可选复杂度验证
  - 验证时机：提交时验证
  - 错误提示："密码至少需要 6 个字符"或"密码格式不正确"
- **实时反馈**：
  - 输入框聚焦：边框高亮，可选绿色提示（格式正确时）
  - 输入框失焦：如格式错误，显示错误提示

### 按钮交互

- **点击触发**：
  - 表单完整填写后，按钮可点击
  - 点击后：显示加载状态，禁用重复点击
  - 成功后：跳转至主页面或显示成功反馈
  - 失败后：恢复按钮可点击，显示错误提示
- **禁用状态**：
  - 表单未填写完整（邮箱或密码为空）时，按钮禁用
  - 禁用视觉：灰度降低，鼠标悬停无效果

### "记住我"交互

- **默认状态**：不勾选
- **勾选效果**：
  - 视觉：复选框填充或图标切换
  - 功能：延长登录状态有效期（如 30 天）
- **取消勾选**：登录状态仅保持当前会话

### "忘记密码？"交互

- **点击触发**：跳转至密码找回页面或打开密码找回模态框
- **视觉层级**：次要文本链接，颜色低于主按钮

### 第三方登录交互

- **占位状态**：
  - 显示"使用第三方账号登录"文案
  - 显示社交平台图标（如 Google, GitHub）
  - 按钮样式：次要按钮，灰度或品牌色图标
- **点击触发**：跳转至第三方授权页面或触发 OAuth 流程

## 状态设计

### 状态流程图

```
[默认状态]
  ↓
[用户开始输入] → [填写中状态]
  ↓
[输入完成] → [表单验证]
  ├→ [验证失败] → [错误状态] → [用户修正] → [填写中状态]
  └→ [验证成功] → [提交中状态] → [按钮加载动画]
  ↓
[服务端响应]
  ├→ [登录成功] → [成功状态] → [跳转或成功反馈]
  └→ [登录失败] → [错误状态] → [显示错误提示] → [恢复可点击]
```

### 状态视觉表现

| 状态 | 输入框 | 按钮 | 提示文本 |
|------|-------|------|---------|
| 默认 | 空白，边框灰色 | 可点击，主色 | 无提示 |
| 填写中 | 聚焦，边框高亮 | 可点击或禁用（未填写完整） | 无提示或可选实时验证提示 |
| 验证失败 | 边框红色，错误图标 | 禁用或可点击（允许重试） | 红色错误提示文本 |
| 提交中 | 边框灰色，可选禁用输入 | 加载动画，禁用点击 | 无提示或"登录中..." |
| 成功 | 边框绿色，成功图标（可选） | 成功反馈（可选） | 绿色成功提示或跳转 |
| 错误（服务端） | 边框灰色 | 可点击 | 红色错误提示（如"密码错误") |

## 视觉方向

### 设计方向建议

**推荐方案：极简居中卡片式**
- **布局**：登录表单居中，固定宽度（360-400px），白色卡片浮于浅色背景
- **视觉**：
  - 背景：浅灰或极浅渐变，传递简洁感
  - 卡片：白色背景，可选微弱阴影（box-shadow: 0 2px 8px rgba(0,0,0,0.05))
  - 边框：输入框细边框（1px #ddd），聚焦时变为主色边框
- **品牌感**：通过 logo 或页面标题传递品牌识别，避免过度装饰

**备选方案：左右分栏式**
- **布局**：左侧品牌展示区（如品牌图或插图），右侧登录表单
- **适用场景**：需要强化品牌视觉、传递产品氛围的场景
- **注意**：移动端需要折叠左侧区域，保持表单为核心

### 视觉细节建议

- **品牌识别**：页面顶部显示品牌 logo 或标题"登录"
- **表单卡片**：
  - 内边距：24-32px
  - 边框：可选无边框（极简）或细边框
  - 阴影：可选微弱阴影，传递"浮层"感
- **输入框**：
  - 高度：40-48px
  - 内边距：12-16px
  - 边框：1px，颜色灰色到主色渐变
- **按钮**：
  - 高度：44-48px
  - 圆角：4-6px（现代感）或无圆角（极简）
  - 文本：粗体，字号 16px

## 待确认问题

以下问题需要在设计评审阶段确认，确保设计方向与业务需求一致：

### 高优先级问题

1. **品牌视觉要求**：
   - 是否有品牌 logo、品牌色、品牌字体要求？
   - 是否需要强化品牌视觉展示（如左侧插图区域）？

2. **目标用户画像**：
   - 登录页面的主要用户群体是谁？（如 B2B 用户、C2C 用户、内部员工）
   - 用户群体是否影响视觉风格选择？（如 B2B 偏专业克制，C2C 偏友好活泼）

3. **第三方登录集成**：
   - 需要集成哪些第三方登录？（如 Google, GitHub, WeChat, 企业微信）
   - 第三方登录是否为占位状态，还是需要真实集成？

4. **安全策略要求**：
   - 是否需要显示密码强度提示？
   - 是否需要显示安全声明（如"您的数据安全加密"）？
   - 是否需要防暴力破解机制（如验证码、登录频率限制）？

### 中优先级问题

5. **"忘记密码"流程**：
   - 点击后跳转新页面还是打开模态框？
   - 是否需要在登录页直接提供密码找回表单？

6. **多语言支持**：
   - 是否需要多语言切换？（如中英文切换）
   - 语言切换位置：页面顶部或底部？

7. **无障碍访问要求**：
   - 是否需要满足 WCAG 2.0 或更高标准？
   - 是否需要键盘导航、屏幕阅读器支持？

8. **隐私声明展示**：
   - 是否需要显示"使用条款"、"隐私政策"链接？
   - 位置：页面底部还是表单底部？

### 低优先级问题

9. **动效需求**：
   - 是否需要表单提交动效（如旋转图标、进度条）？
   - 是否需要页面入场动效（如表单渐入）？

10. **错误反馈方式**：
    - 错误提示位置：输入框下方还是页面顶部统一提示？
    - 错误提示是否需要震动或闪烁效果？

11. **记住我功能细节**：
    - 记住我功能的保持时长？（如 7 天、30 天）
    - 是否需要用户手动设置保持时长？
