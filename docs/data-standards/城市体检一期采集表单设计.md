# 城市体检一期采集表单设计

## 1. 设计结论

一期采集表单不设计成一张超级大表，而设计成“通用底座字段 + 业务模板 + 空间组件 + 附件组件 + 质检组件”的组合式表单体系。

空间采集必须同时支持两种方式：

1. **自由空间采集**：现场直接在地图上点选、画线、画面，填写属性并上传照片。
2. **矢量任务下发调查**：内业整理待调查矢量，形成任务图层，下发给外业人员逐个核查、修正和回填。

这两种方式共同服务于综合体检、专项体检、重点更新片区体检和城市更新资源盘点。

## 2. 表单入口

```text
城市体检一期数据采集
├─ 自由采集
│  ├─ 新增点对象
│  ├─ 新增线对象
│  └─ 新增面对象
├─ 任务采集
│  ├─ 我的调查任务
│  ├─ 地图任务图层
│  ├─ 待核查对象列表
│  └─ 草稿与退回任务
├─ 部门填报
│  ├─ 指标填报
│  ├─ 台账导入
│  └─ 数据源登记
└─ 审核质检
   ├─ 待质检数据
   ├─ 退回修改
   └─ 入库确认
```

## 3. 通用底座字段

所有采集表单都应包含以下字段组。

### 3.1 任务信息

| 字段 | 控件 | 必填 | 说明 |
|---|---|---|---|
| 采集分类 | 下拉 | 是 | 综合体检、专项体检、城市更新资源、重点更新片区 |
| 采集任务 | 下拉 | 是 | 排水防涝、完整社区、历史文化、燃气安全等 |
| 体检年度 | 年份选择 | 是 | 默认当前年度 |
| 数据分类 | 下拉 | 是 | 调查类、统计类、测绘类、互联网数据 |
| 采集方式 | 下拉 | 是 | 自由采集、任务采集、部门填报、批量导入、接口接入 |
| 采集状态 | 系统字段 | 是 | 草稿、待质检、退回、通过、已入库 |

### 3.2 对象信息

| 字段 | 控件 | 必填 | 说明 |
|---|---|---|---|
| 对象类型 | 下拉 | 是 | 住房、小区社区、街区、城区城市、设施、管线、更新片区、更新资源 |
| 对象名称 | 输入 / 选择 | 是 | 可选择已有对象，也可新增临时对象 |
| 对象编码 | 自动生成 / 选择 | 是 | 对接 `health.health_object` |
| 所属行政区 | 空间叠加 / 下拉 | 是 | 区县、街道、社区 |
| 详细地址 | 输入 | 条件必填 | 建筑、设施、资源类建议必填 |
| 关联对象 | 选择器 | 否 | 关联小区、社区、街区、片区、项目或问题 |

### 3.3 来源与责任信息

| 字段 | 控件 | 必填 | 说明 |
|---|---|---|---|
| 数据来源单位 | 组织选择 | 是 | 对接 `base.data_source` |
| 负责单位 | 组织选择 | 是 | 可与来源单位不同 |
| 采集人 | 系统字段 | 是 | 当前登录用户或外业人员 |
| 采集时间 | 系统字段 | 是 | 保存或提交时间 |
| 数据截止日期 | 日期 | 统计类必填 | 统计口径截止日期 |
| 质量等级 | 下拉 | 否 | 高、中、低、未知 |
| 来源说明 | 多行文本 | 否 | 说明数据来源、口径、转换过程 |

## 4. 自由空间采集表单

自由空间采集适用于现场新发现对象或问题。

### 4.1 流程

```text
选择采集任务
  ↓
选择几何类型：点 / 线 / 面
  ↓
地图点选 / 画线 / 画面
  ↓
系统自动计算空间属性
  ↓
填写业务属性
  ↓
上传照片和附件
  ↓
提交质检
```

### 4.2 空间字段

| 字段 | 控件 | 生成方式 | 说明 |
|---|---|---|---|
| 几何类型 | 单选 | 人工选择 | Point、LineString、Polygon |
| geom | 地图绘制 | 地图点选、画线、画面 | 对接 `geo.spatial_boundary.geom` |
| centroid | 自动计算 | 系统生成 | 点对象取自身，线面对象取中心点 |
| length | 自动计算 | 线对象生成 | 单位米，可转公里 |
| area | 自动计算 | 面对象生成 | 单位平方米，可转公顷 |
| spatialReference | 自动 / 选择 | 系统默认 | 一期统一记录 CGCS2000，移动端原始坐标需记录来源坐标系 |
| spatialAccuracy | 下拉 | 人工选择 | 坐标点、米级、社区级、街道级等 |
| parentBoundary | 空间叠加 | 系统生成 | 所属区县、街道、社区、片区 |

### 4.3 业务字段

| 字段 | 控件 | 必填 | 说明 |
|---|---|---|---|
| 问题类型 | 多选 | 条件必填 | 安全隐患、设施短板、环境问题、空间低效、历史文化风险等 |
| 现状描述 | 多行文本 | 是 | 现场描述 |
| 风险等级 | 单选 | 条件必填 | 高、中、低、未知 |
| 是否需整改 | 单选 | 是 | 是、否、待确认 |
| 整改建议 | 多行文本 | 否 | 现场初判建议 |
| 是否建议转资源 | 单选 | 否 | 是、否 |
| 是否建议转项目 | 单选 | 否 | 是、否 |
| 备注 | 多行文本 | 否 | 补充说明 |

## 5. 矢量任务下发调查表单

矢量任务下发调查适用于内业已整理好的待核查对象。下发矢量是“待核查线索”，不是最终正式成果。

### 5.1 内业下发流程

```text
准备调查对象
  ↓
导入 SHP / GeoJSON / GDB / CAD 转换成果
  ↓
字段映射
  ↓
生成任务图层
  ↓
设置调查表单模板
  ↓
按区域 / 人员 / 类型分配任务
  ↓
下发到移动端
```

### 5.2 外业调查流程

```text
领取任务
  ↓
地图查看待调查对象
  ↓
导航或到点核查
  ↓
确认对象是否存在
  ↓
确认或修正空间几何
  ↓
填写核查表单
  ↓
上传照片
  ↓
保存草稿或提交质检
```

### 5.3 任务图层字段

| 字段 | 控件 | 必填 | 说明 |
|---|---|---|---|
| taskLayerId | 系统字段 | 是 | 任务图层唯一 ID |
| taskName | 输入 | 是 | 如老旧小区核查、疑似危旧房核查 |
| taskType | 下拉 | 是 | 老旧小区、易涝点、历史建筑、更新资源、燃气管线等 |
| sourceFile | 附件 | 是 | 原始矢量文件 |
| sourceFormat | 下拉 | 是 | SHP、GeoJSON、GDB、CAD 转换成果 |
| geometryType | 自动 | 是 | 点、线、面 |
| spatialReference | 自动 / 选择 | 是 | 坐标系 |
| assignedOrg | 组织选择 | 是 | 下发单位或调查责任单位 |
| status | 系统字段 | 是 | 草稿、已下发、调查中、已完成、已归档 |

### 5.4 任务对象字段

| 字段 | 控件 | 必填 | 说明 |
|---|---|---|---|
| taskFeatureId | 系统字段 | 是 | 任务对象 ID |
| sourceFeatureId | 系统字段 | 否 | 原始矢量对象 ID |
| objectName | 输入 / 原始属性 | 是 | 可来自矢量属性，也可外业修正 |
| objectType | 下拉 / 模板带出 | 是 | 与任务类型匹配 |
| originalGeom | 地图展示 | 是 | 下发几何，只读保留 |
| surveyedGeom | 地图编辑 | 条件必填 | 外业确认或修正后的几何 |
| originalProperties | 属性表 | 否 | 原始矢量属性，只读展示 |
| assignedTo | 用户选择 | 是 | 调查人员 |
| surveyStatus | 单选 | 是 | 未调查、已到达、已核查、无法核查、已退回 |
| surveyResult | 单选 | 是 | 确认存在、不存在、已拆除、位置需修正、属性需修正、新增问题 |
| finalObjectId | 系统字段 | 否 | 审核入库后的正式对象 ID |

### 5.5 外业核查字段

| 字段 | 控件 | 必填 | 说明 |
|---|---|---|---|
| 是否找到对象 | 单选 | 是 | 是、否 |
| 对象是否存在 | 单选 | 是 | 存在、不存在、已拆除、无法进入、无法确认 |
| 位置是否准确 | 单选 | 是 | 准确、需修正、完全错误 |
| 属性是否准确 | 单选 | 是 | 准确、需修正 |
| 现状描述 | 多行文本 | 是 | 现场核查描述 |
| 问题类型 | 多选 | 条件必填 | 有问题时必填 |
| 风险等级 | 单选 | 条件必填 | 高、中、低、未知 |
| 更新潜力 | 单选 | 资源类必填 | 高、中、低、未知 |
| 是否建议入库 | 单选 | 是 | 是、否、待复核 |
| 是否建议转项目 | 单选 | 否 | 是、否、待策划 |
| 无法核查原因 | 多选 | 条件必填 | 封闭、危险、无权限、资料不足、位置错误 |
| 调查备注 | 多行文本 | 否 | 补充说明 |

## 6. 空间修正机制

| 操作 | 适用场景 | 处理方式 |
|---|---|---|
| 确认原几何 | 下发几何准确 | `surveyedGeom` 复制 `originalGeom`，记录确认人和时间 |
| 修改点位 | 点位偏移 | 保存原点和新点，进入空间质检 |
| 调整线 | 管线、道路、街巷走向有偏差 | 保存原线和修正线，记录长度差异 |
| 调整面 | 小区、地块、片区边界不准 | 保存原面和修正面，记录面积差异 |
| 新增关联对象 | 现场发现新增资源或问题 | 从任务表单进入自由采集子表单 |
| 标记无法核查 | 无法进入、危险、资料不足 | 必须填写原因并上传照片或说明 |

## 7. 业务表单模板

| 模板 | 字段组 | 适用对象 |
|---|---|---|
| 综合体检模板 | 对象信息、指标填报、问题识别、整治建议、照片附件、质检结果 | 住房、小区社区、街区、城区城市 |
| 排水防涝模板 | 设施基础信息、空间信息、排水能力、风险问题、应急能力、附件材料 | 易涝点、雨水管渠、排涝泵站、排涝通道、地下空间 |
| 完整社区模板 | 社区基础信息、设施配置、服务能力、缺口判断、闲置资源、群众反馈 | 社区、小区、公共服务设施、闲置地块、低效构筑物 |
| 历史文化模板 | 资源身份、空间信息、价值信息、保护状态、使用状态、风险问题、附件材料 | 历史建筑、潜在历史建筑、历史文化街区、不可移动文物、古树名木、工业遗产 |
| 燃气安全模板 | 对象信息、管网信息、安全状态、管理信息、设备信息、风险等级、检测材料 | 燃气管线、燃气用户、餐饮商户、第三方施工点、特许经营区域 |
| 重点更新片区模板 | 片区基础信息、现状情况、主要问题、更新资源、群众意愿、更新方向、项目建议 | 老旧小区、城中村、老旧厂区、商业街区、公共空间、历史街区 |
| 城市更新资源模板 | 资源身份、权属信息、现状用途、规划约束、空间信息、问题风险、更新潜力、项目转化、附件材料 | 居住类、产业类、公共设施类、公共空间类、历史风貌类、其他类资源 |

## 8. PC 端与移动端分工

| 端 | 主要职责 | 页面 |
|---|---|---|
| PC 端 | 内业任务组织、部门填报、矢量导入、字段映射、任务分配、质检审核、成果入库 | 任务图层管理、矢量导入、字段映射、调查进度、质检问题、入库审核 |
| 移动端 | 外业调查执行、地图核查、拍照上传、几何修正、草稿同步 | 我的任务、地图任务、对象核查表单、自由采集、草稿箱、同步中心 |

移动端字段应少而准，复杂统计口径、批量导入和最终入库审核留给 PC 端处理。

## 9. 表单质检规则

| 场景 | 必检规则 | 不通过处理 |
|---|---|---|
| 自由采集提交 | 对象类型、几何、采集人、采集时间、照片、问题描述 | 阻止提交或保存草稿 |
| 任务采集提交 | taskFeatureId、核查结果、位置是否准确、对象是否存在、照片 | 阻止提交或退回补充 |
| 空间修正提交 | 几何合法、坐标系统一、不得越界、面积长度差异可解释 | 进入人工复核 |
| 部门填报提交 | 指标编码、指标值、单位、来源单位、数据截止日期 | 阻止入库 |
| 批量导入 | 字段映射、枚举合法、必填完整、空间拓扑、附件可打开 | 生成错误清单 |
| 入库确认 | 质检通过、审核通过、来源可追溯、附件完整、安全分级完成 | 未通过不得入正式库 |

## 10. 入库流转

```text
自由采集 / 任务采集 / 部门填报 / 批量导入
  ↓
采集记录 survey_record
  ↓
质检记录 qc_record
  ↓
人工审核 review_record
  ↓
正式入库
  ├─ health.health_object
  ├─ geo.spatial_boundary
  ├─ health.indicator_value
  ├─ renewal.renewal_resource
  ├─ problem_issue
  └─ base.attachment
```

任务下发数据流转：

```text
task_layer
  ↓
task_feature
  ↓
survey_record / survey_geometry_edit / survey_attachment
  ↓
审核通过后生成或更新正式对象、问题、资源、指标值
```

## 11. 一期最小可用表单清单

一期至少应实现以下表单：

1. 自由点采集表单
2. 自由线采集表单
3. 自由面采集表单
4. 矢量任务图层导入表单
5. 任务对象核查表单
6. 综合体检指标填报表单
7. 专项体检调查表单
8. 城市更新资源采集表单
9. 数据源登记表单
10. 质检审核表单
11. 采集表单字段矩阵配置表

> 说明：以上 1-10 是产品表单入口，字段级配置详见 [[城市体检一期采集表单字段矩阵]]。
