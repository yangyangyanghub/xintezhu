# 城市体检一期指标采集标准

## 1. 编制目的

本标准用于一期城市体检与城市更新平台的数据采集工作，面向现场采集人员、街道社区填报人员、行业部门数据提供人员和第三方技术团队，明确一期"采什么、按什么对象采、谁提供、怎么留痕、如何入库"。

一期采集以住房、小区（社区）、街区、城区（城市）四级对象为主线，专项体检和城市更新资源盘点共用同一对象底座、空间边界、指标库、问题清单和更新资源库。采集成果应能支撑"体检问题—整治建议—更新资源—更新项目—年度计划"的闭环。

## 3. 总体采集规则

1. 数据采集范围必须与指标要求范围一致，严格区分深度体检范围、建成区、市辖区、全市范围、县城建成区等边界。
2. 统计类数据宜采集上一年度 12 月 31 日截止数据；调查类、测绘类、互联网类实时更新数据应采集最新数据；需要趋势或累计计算的指标按指标定义采集多个时间点。
3. 数据采集内容应包括数据来源、负责单位、时间精度、空间精度、数据格式、采集方法和说明材料。
4. 测绘类数据采用标准地理空间格式，坐标基准按 CGCS2000 和 1985 国家高程执行；空间可视化成果不得混用坐标系。
5. 每条采集记录应保留台账、图纸、规划、照片、检测鉴定报告、部门统计表或访谈问卷等可追溯材料。
6. 调查类数据采用逻辑校验、异常值检测、抽样复检；统计类数据采用交叉验证、趋势分析、格式审查；测绘类数据采用拓扑检查、坐标统一和空间校正；互联网数据采用时间戳验证和逻辑关联校验。
7. 能落图的对象必须保留空间位置或边界，避免采集成果无法上图分析。
8. 采集成果应能支撑"对象—指标—问题—建议—资源—项目—计划"的全链路闭环。

## 5. 统一对象与编码

一期所有采集记录应关联统一体检对象。对象编码建议采用"城市代码-对象类型-年份-序号"，例如 130400-BLOCK-2026-0001。

| 对象层级 | 对象类型 | 采集边界 | 适用任务 | 关键说明 |
|---|---|---|---|---|
| 住房 | housing | 单栋住宅、公共建筑、历史建筑等建筑实体 | 综合体检、重点更新片区、更新资源、历史文化 | 重点记录建筑位置、建成年份、结构类型、楼层、面积、安全隐患和改造需求 |
| 小区（社区） | community | 小区、社区、完整社区基本单元 | 综合体检、完整社区、重点更新片区、更新资源 | 以社区管理范围和居住小区边界为基础，记录设施、环境、治理和居民意愿 |
| 街区 | block | 街道或十五分钟生活圈衔接形成的街区单元 | 综合体检、重点更新片区、老旧生活街区、老旧商业街区 | 重点记录公共服务、交通、活力、老旧片区和更新潜力 |
| 城区（城市） | city | 建成区、市辖区或指标指定范围 | 综合体检、排水防涝、燃气安全、宜居县城、历史文化 | 严格按指标要求区分建成区、市辖区、全市范围 |
| 更新片区 | renewalArea | 重点更新片区边界，可跨社区和街道 | 重点更新片区、更新资源、项目生成 | 独立建模，关联主导问题、资源潜力和项目建议 |

## 6. 统一指标字段

所有指标采集表统一使用以下字段。专项可增加附加字段，但不得减少必填字段。

| 字段 | 填写要求 | 示例 | 说明 |
|---|---|---|---|
| 采集分类 | 必填，八类之一 | 综合体检 | 用于区分指标来源和业务流程 |
| 指标来源 | 必填 | 附件 2 指标 1 | 附件 2 至附件 8 或附件 9 |
| 基本单元 | 必填 | 住房 / 小区（社区） / 街区 / 城区（城市） / 专项对象 | 与技术标准要求一致 |
| 对象编码 | 必填 | 130400-COMMUNITY-2026-0001 | 关联统一体检对象 |
| 对象名称 | 必填 | 春晖社区 | 使用现行管理或台账名称 |
| 所属行政区 | 必填 | 丛台区 | 必要时精确到街道、社区、小区 |
| 空间边界或位置 | 条件必填 | SHP 边界 / 点位 / 起止点 | 能落图对象必须填写 |
| 指标序号 | 必填 | 1 / T1 / H1 | 保留附件原序号 |
| 指标名称 | 必填 | 存在结构安全隐患的住宅数量（栋） | 使用来源文件原名称 |
| 指标维度 | 必填 | 安全耐久 | 来源文件有维度时照录 |
| 体检内容 | 必填 | 查找住宅结构安全问题 | 可简化表达，但不得改变含义 |
| 数据值 | 必填 | 5 | 与计量单位一致 |
| 计量单位 | 必填 | 栋 / 个 / % / 千米 | 使用来源文件单位 |
| 数据分类 | 必填 | 调查类 / 统计类 / 测绘类 / 互联网数据 | 按技术标准分类 |
| 数据获取方式 | 必填 | 现场调研填报 / 行业部门统计 / 仪器检测 | 与指标要求匹配 |
| 数据提供单位 | 必填 | 社区、物业、燃气公司、行业部门 | 明确责任来源 |
| 负责单位 | 必填 | 街道、社区、住建、城管等 | 明确数据质量责任 |
| 时间精度 | 必填 | 年 / 月 / 日 / 实时 | 统计类一般到年，现场类到日 |
| 空间精度 | 条件必填 | 小区边界 / 建筑点位 / 管线线要素 | 用于落图和空间分析 |
| 数据格式 | 必填 | 表格、SHP、PDF、JPG | 可多选，按实际附件填写 |
| 说明材料 | 必填 | 台账、图纸、照片、鉴定报告 | 支撑可追溯 |
| 校验方式 | 必填 | 逻辑校验、交叉验证、抽样复检、拓扑检查 | 按数据类型选择 |
| 问题关联 | 选填 | 问题编号 | 指标异常时关联问题 |
| 更新资源关联 | 选填 | 资源编号 | 涉及资源盘点或项目转化时填写 |

## 7. 数据采集方法与留痕要求

| 数据分类 | 适用内容 | 采集方法 | 留痕材料 | 校验要求 |

| 调查类数据 | 楼道安全、围护安全、小区设施、居民意愿、现场问题 | 现场实地踏勘、问卷调查、座谈访谈、移动端采集 | 调研记录表、定位照片、访谈纪要、问卷结果 | 逻辑校验、异常值检测、抽样复检 |

## 8. 综合体检采集标准

综合体检按“住房—小区（社区）—街区—城区（城市）”四级组织。邯郸市 2026 年指标体系为“55 个基础指标 + 6 个河北省特色指标 + 8 个邯郸市特色指标”，一期平台按指标版本管理。

## 12. 入库与交付要求

1. 体检对象入 `health.health_object`，并关联组织、空间边界、年度和来源。
2. 指标定义、指标版本、指标值入指标库，指标名称、序号、维度和单位保持来源文件原口径。
3. 建筑、小区、街区、城区、设施、管网、历史文化资源等入专题对象库。
4. 指标异常、现场问题和群众诉求统一生成问题清单，并记录来源类型、关联对象、空间位置、严重程度、治理难度和状态。
5. 城市更新资源独立入库，不作为项目备注；资源应记录类型、权属、现状用途、规划用途、面积、安全风险、潜力等级、复用价值。
6. 重点更新片区独立入库，不只挂在项目表；片区应记录边界、主导问题、资源潜力、群众意愿、规划引导和状态。
7. 能转化为项目的资源和问题应建立关联关系，支撑问题转项目、资源转项目、片区转项目。
8. 最终交付至少包括：指标采集表、对象清单、问题清单、资源清单、空间数据、说明材料目录、照片和附件目录。

## 13. 现场采集人员工作提示

1. 先确认对象层级，再填写指标。不要把小区问题填到街区指标，也不要把市辖区统计值填到建成区指标。
2. 先拍全景，再拍问题细节；照片应能看出对象、位置和问题类型。
3. 能画边界的资源必须画边界；线性设施记录起止点；点状设施记录位置。
4. 居民意愿、产权权属、改造计划、鉴定结论等不能凭现场判断代填，必须有材料支撑。
5. 对附件 2 至附件 8 的指标，使用原指标名称、序号和单位；对附件 9 的资源，使用原大类、中类、小类和统计单位。
6. 遇到同一对象同时属于综合体检、专项体检和更新资源时，只建一个对象，多挂指标、问题和资源关系。
7. 数据不确定时在备注中说明口径和缺口，不得编造数值。
