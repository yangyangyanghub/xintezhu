# 城市体检一期数据源登记模板

## 1. 使用说明

本模板用于城市体检一期数据标准包的数据源登记、部门移交、入库前审查和后续治理。登记对象包括规划、基础测绘、地下管线、房产与不动产、遥感影像、历史建筑、项目档案、统计报送、现场调查、互联网/API 和其他来源数据。

填报原则：

1. **一源一登记**：每个独立数据来源登记一行；可按专题、年度、批次拆分的，应拆分登记。
2. **可追溯**：记录来源单位、责任人、采集时间、数据版本、存储位置和说明材料。
3. **可落库**：登记字段应能映射至 `base.data_source`，支撑对象库、指标库、问题库引用。
4. **可校验**：同步登记质量等级、可用性、坐标系统、安全分级和待处理问题。
5. **不改原件**：本模板只登记数据源，不修改原始文件、图纸、接口、数据库或历史模板。

## 5. 数据质量分级

| 等级 | 判定标准 | 处理建议 |
|---|---|---|
| A | 完整、准确、现势性强，空间和属性字段齐全，可直接追溯来源 | 可直接入库或简单格式转换后入库 |
| B | 基本完整、较准确、现势性一般，存在少量字段缺失或格式不统一 | 简单清洗、字段映射、格式转换后入库 |
| C | 有明显缺失、部分需更新，坐标或口径需复核 | 需较大工作量治理，完成复核后再入库 |
| D | 严重缺失、来源不清、坐标不明、无法复核或已明显过期 | 暂不入库，重新采集或补充证明材料 |

质量检查应覆盖准确性、完整性、一致性和时效性。测绘类数据重点检查拓扑、坐标系统、属性字段；统计类数据重点检查口径、年度和交叉验证；调查类数据重点检查逻辑、异常值、抽样复核；互联网数据重点检查 URL、时间戳、原始内容和哈希留存。

## 6. 可用性评估

| 可用性 | 判定条件 | 入库动作 |
|---|---|---|
| 立即可用 | 字段、格式、坐标、安全分级均满足一期入库要求 | 直接入库 |
| 需简单处理 | 仅需格式转换、字段重命名、轻量清洗或附件补齐 | 处理后入库 |
| 需较大工作量 | 坐标转换、空间校正、口径统一、拓扑修复或多源校验工作量较大 | 建立治理任务后入库 |
| 不可用 | 来源不清、质量 D 级、涉密不可共享、关键字段缺失且无法补齐 | 暂不入库 |

## 7. 坐标系统要求

1. 测绘类、规划类、管线类、遥感解译类和可落图项目档案原则上统一到 CGCS2000。
2. 高程数据原则上使用 1985 国家高程基准；如为地方高程，必须登记转换参数或转换责任部门。
3. 北京54、西安80、地方坐标系、WGS84、GCJ02 等数据应登记原坐标系、转换方法、控制点来源和转换后精度。
4. 无空间坐标的数据应说明空间关联方式，如行政区、街道、社区、小区、建筑编码、项目编号或地址匹配。
5. 空间可视化成果应标注比例尺、图例、分类字段、单位和坐标系统；矢量数据应附带属性表。

## 8. 安全分级与共享控制

| 安全分级 | 典型数据 | 管控要求 |
|---|---|---|
| 核心数据 | 市域空间矢量 GIS 地图、实景三维、高精度全域地下管线、涉及城市安全运行的高精度数据 | 独立安全域或受控环境存储，严格审批，禁止明文传输，使用后按要求销毁副本 |
| 重要数据 | 居民信息、产权明细、社区公共基础设施、部门统计明细、调查原始记录 | 加密存储、脱敏共享、记录流向和使用日志，签署数据安全承诺 |
| 一般数据 | 公开统计年鉴、公开政策资料、脱敏满意度统计、公开互联网信息 | 可按授权范围共享，保留访问日志和来源记录 |

定级原则：范围越大、精度越高、涉及个人权益或城市安全越强，安全等级越高；存在多重属性时就高不就低。
