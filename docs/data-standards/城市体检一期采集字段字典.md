# 城市体检一期采集字段字典

## 1. 适用范围

本文档定义城市体检一期 MVP 的采集字段字典，用于统一 PC 填报、移动采集、导入映射和数据质量检查。字段设计依据实施计划阶段 4「数据采集与质检」、河北省城市体检数据采集表附录 A，以及对象库数据库设计中的 health.health_object、health.indicator_value、base.data_source、geo.spatial_boundary。

> 说明：数据库设计中未展开附件表、采集任务表和审核记录表。本文涉及照片、附件、审核流转的字段均标记为「平台扩展字段」，用于指导一期表单 schema 实现，不表示已完成物理表定义。

## 2. 字段分层

| 字段层级 | 业务含义 | 主要存储目标 | 典型用途 |
|---|---|---|---|
| 对象字段 | 描述住房、小区社区、街区、城区城市等体检对象 | health.health_object | 对象建库、对象选择、导入匹配 |
| 指标采集字段 | 记录指标对应的数据名称、数据值、单位和口径 | health.indicator_value | 指标填报、计算、异常检查 |
| 空间字段 | 记录点位、中心点、边界、面积和坐标系统 | geo.spatial_boundary | 地图落点、边界导入、空间质检 |
| 来源元数据字段 | 记录数据从哪里来、由谁提供、格式和质量如何 | base.data_source | 溯源、共享、导入映射 |
| 证明材料字段 | 保存照片、台账、矢量数据、图纸、规划和政策材料 | 平台扩展字段：base.attachment | 现场核验、附件归档 |
| 审核质检字段 | 记录草稿、提交、质检、审核、驳回和归档状态 | 平台扩展字段：采集任务/审核记录 | 流程闭环、质量控制 |

## 3. 通用字段类型

### 3.1 文本字段（text）

文本字段用于名称、编码、地址、说明、备注、审核意见等内容。

| 定义项 | 规则 |
|---|---|
| 控件 | 单行输入框、多行文本框 |
| 基础校验 | 去除首尾空格；必填字段不得为空；编码类字段不得包含不可见字符 |
| 长度建议 | 编码 10~64 字符；名称 2~100 字符；说明类文本 0~2000 字符 |
| 导入规则 | 空字符串按缺失值处理；全角空格应在导入预处理时清洗 |
| 质检规则 | 必填性、编码唯一性、名称重复、说明材料缺失 |

| 字段名 | 中文名称 | 必填 | 存储目标 | 说明 |
|---|---|---|---|---|
| objectCode | 统一对象编码 | 是 | health.health_object.objectCode | 建议按「城市代码-对象类型-年份-序号」生成 |
| objectName | 对象名称 | 是 | health.health_object.objectName | 住房、小区社区、街区、城区城市名称 |
| indicatorName | 指标名称 | 是 | health.indicator.indicatorName / 导入映射 | 应与指标库一致 |
| dataName | 数据名称 | 是 | health.indicator_value 平台扩展属性 | 对应附录 A「数据名称」 |
| dataDescription | 数据内容说明 | 否 | health.indicator_value 平台扩展属性 | 说明原始数据含义、口径或处理方式 |
| remarks | 备注 | 否 | 平台扩展字段 | 仅作补充说明，不替代来源材料 |

### 3.2 数字字段（number）

数字字段用于指标值、面积、数量、比例、服务能力、缺口值、版本号等内容。

| 定义项 | 规则 |
|---|---|
| 控件 | 数字输入框、百分比输入框、带单位输入框 |
| 基础校验 | 类型、必填、最小值、最大值、小数位、单位一致性 |
| 空值规则 | 未采集应留空，不得用 0 代替缺失；0 只表示真实数值为零 |
| 范围规则 | 按指标库阈值或采集表配置执行；比例类通常限定为 0~100 |
| 质检规则 | 负值异常、单位错填、超阈值、与历史值偏离过大 |

| 字段名 | 中文名称 | 必填 | 存储目标 | 说明 |
|---|---|---|---|---|
| indicatorValue | 数据值/指标值 | 是 | health.indicator_value.value 或同类结果字段 | 对应附录 A「数据值」 |
| numericUnit | 数值单位 | 视指标而定 | health.indicator_value 平台扩展属性 | 如栋、套、平方米、公里、百分比、个/万人 |
| area | 面积 | 面对象必填 | geo.spatial_boundary.area | 可由空间边界计算生成 |
| version | 版本 | 是 | health.health_object.version / geo.spatial_boundary.version | 支持对象和边界年度版本化 |
| inspectionYear | 体检年份 | 是 | health.health_object.inspectionYear / health.indicator_value.inspectionYear | 年度体检和滚动复检依据 |

### 3.3 枚举字段（enum）

枚举字段用于对象类型、状态、数据分类、采集方法、数据格式、质量等级等固定选项。枚举应进入统一字典管理，导入时支持中文名称到系统值的映射。

| 定义项 | 规则 |
|---|---|
| 控件 | 下拉框、单选框、标签选择器 |
| 基础校验 | 必填、枚举合法性、废弃值拦截 |
| 导入规则 | 支持中文别名映射；无法识别的值进入导入错误清单 |
| 质检规则 | 枚举超范围、同义词未归一、对象类型与基本单元不一致 |

| 字段名 | 中文名称 | 推荐枚举 | 必填 | 存储目标 |
|---|---|---|---|---|
| objectType | 对象类型 | 住房、小区社区、街区、城区城市、更新片区 | 是 | health.health_object.objectType |
| status | 对象状态 | 有效、合并、拆分、移除 | 是 | health.health_object.status |
| dataCategory | 数据分类 | 调查类、统计类、测绘类、互联网数据 | 是 | base.data_source 平台扩展属性 |
| collectionMethod | 采集方法 | 现场踏勘、问卷调查、座谈访谈、部门统计报送、政务接口、遥感识别、无人机识别、物联网采集、仪器检测、API 调用、手机信令 | 是 | base.data_source 平台扩展属性 |
| sourceCategory | 数据源类别 | 规划、测绘、管线、房产测绘、遥感、项目、档案、人工采集 | 是 | base.data_source.sourceCategory |
| dataFormat | 数据格式 | xlsx、csv、geojson、shp、pdf、image、api、text、database | 是 | base.data_source.dataFormat |
| qualityLevel | 质量等级 | 高、中、低、未知 | 否 | base.data_source.qualityLevel |
| sourceType | 空间来源类型 | shp、geojson、manual、remoteSensing、planning | 空间数据必填 | geo.spatial_boundary.sourceType |

### 3.4 日期字段（date）

日期字段用于采集时间、数据截止时间、生效日期、失效日期、提交时间和审核时间。统计类数据宜以年度统计为基础，采集截止时间通常为上一年度 12 月 31 日；调查类、测绘类、互联网数据等动态数据应采集最新数据。

| 定义项 | 规则 |
|---|---|
| 控件 | 年份选择器、日期选择器、日期时间选择器 |
| 基础校验 | 日期格式合法；不得晚于提交时间；生效日期不得晚于失效日期 |
| 时间精度 | 年、月、日、时分秒，应与采集要求一致 |
| 质检规则 | 数据年份与体检年份不匹配、动态数据缺少时间戳、历史数据时间范围不足 |

| 字段名 | 中文名称 | 必填 | 存储目标 | 说明 |
|---|---|---|---|---|
| collectionDate | 采集日期 | 是 | health.indicator_value 平台扩展属性 | 现场、问卷、测绘、接口采集时间 |
| dataAsOfDate | 数据截止日期 | 统计类必填 | health.indicator_value 平台扩展属性 | 统计类数据的截止时点 |
| validFrom | 生效日期 | 是 | health.health_object.validFrom / geo.spatial_boundary.validFrom | 对象或空间边界开始有效日期 |
| validTo | 失效日期 | 否 | health.health_object.validTo / geo.spatial_boundary.validTo | 对象或空间边界失效日期 |
| submittedAt | 提交时间 | 提交后必填 | 平台扩展字段：采集任务 | 用于流程追踪和时效性检查 |
| reviewedAt | 审核时间 | 审核后必填 | 平台扩展字段：审核记录 | 用于审核留痕 |

### 3.5 坐标字段（coordinate）

坐标字段用于移动端定位、点位采集、中心点记录和空间核验。测绘类空间数据应以 CGCS2000 和 1985 国家高程为基准；用于空间分析前，应统一坐标系统并说明投影。

| 定义项 | 规则 |
|---|---|
| 控件 | 地图选点、定位按钮、经纬度输入框 |
| 基础校验 | 坐标非空、数值范围合法、坐标系明确、空间精度明确 |
| 坐标系 | 默认记录 CGCS2000；互联网底图坐标需转换或标记来源坐标系 |
| 质检规则 | 坐标缺失、坐标漂移、坐标系混用、点位超出所属对象边界 |

| 字段名 | 中文名称 | 必填 | 存储目标 | 说明 |
|---|---|---|---|---|
| longitude | 经度 | 点位采集必填 | geo.spatial_boundary.centroid 或平台扩展字段 | 与纬度成对出现 |
| latitude | 纬度 | 点位采集必填 | geo.spatial_boundary.centroid 或平台扩展字段 | 与经度成对出现 |
| altitude | 高程 | 否 | 平台扩展字段 | 涉及地下管网、桥梁、坡度或高程时使用 |
| spatialReference | 坐标系 | 空间数据必填 | base.data_source.spatialReference | 如 CGCS2000、高斯-克吕格投影说明 |
| spatialAccuracy | 空间精度 | 是 | base.data_source 平台扩展属性 | 对应附录 A「空间精度」 |

### 3.6 照片字段（photo）

照片字段用于现场踏勘、设施破损、隐患点和说明材料影像化留痕。照片属于平台扩展字段，应与对象、指标值、问题或空间点位建立关联。

| 定义项 | 规则 |
|---|---|
| 控件 | 拍照、相册上传、照片预览 |
| 基础校验 | 文件类型、文件大小、最少张数、拍摄时间、定位信息 |
| 推荐要求 | 移动端照片宜带时间、地点、采集人水印 |
| 质检规则 | 必要照片缺失、重复照片、照片与坐标不一致、照片时间晚于提交时间 |

| 字段名 | 中文名称 | 必填 | 存储目标 | 说明 |
|---|---|---|---|---|
| photoFiles | 现场照片 | 条件必填 | 平台扩展字段：base.attachment | 现场踏勘、隐患核查、设施短板采集时必填 |
| photoTakenAt | 拍摄时间 | 照片存在时必填 | 平台扩展字段：base.attachment | 用于时间戳校验 |
| photoLocation | 拍摄位置 | 移动端照片建议必填 | 平台扩展字段：base.attachment | 可复用坐标字段 |
| photoDescription | 照片说明 | 否 | 平台扩展字段：base.attachment | 描述照片对应问题或对象部位 |

### 3.7 附件字段（attachment）

附件字段用于近 5 年历史数据、统计台账、矢量数据、图纸、相关规划、工作总结、政策材料等说明材料。附件属于平台扩展字段。

| 定义项 | 规则 |
|---|---|
| 控件 | 文件上传、附件清单、材料分类选择 |
| 基础校验 | 文件类型、文件大小、数量、材料分类、是否可打开 |
| 材料类型 | 历史数据、统计台账、矢量数据、图纸规划、工作总结、政策材料、其他说明材料 |
| 质检规则 | 说明材料缺失、附件格式错误、附件与数据源不匹配、矢量数据缺少属性表 |

| 字段名 | 中文名称 | 必填 | 存储目标 | 说明 |
|---|---|---|---|---|
| attachments | 说明材料 | 条件必填 | 平台扩展字段：base.attachment | 对应附录 A「说明材料」 |
| attachmentType | 材料类型 | 附件存在时必填 | 平台扩展字段：base.attachment | 区分台账、图纸、规划、政策等 |
| attachmentSourceId | 所属数据源 | 附件存在时必填 | base.data_source.id 关联 | 保证附件可追溯 |
| fileHash | 文件校验值 | 否 | 平台扩展字段：base.attachment | 互联网数据或重要材料可用于完整性校验 |

### 3.8 几何字段（geometry）

几何字段用于对象边界、街区范围、建成区范围、问题区域、资源范围等空间数据。几何字段不直接在普通表单中手工填写，应通过地图绘制、SHP/GeoJSON 导入或移动端轨迹采集生成。

| 定义项 | 规则 |
|---|---|
| 控件 | 地图点选、画线、画面、边界上传、矢量任务对象选择器、空间对象选择器 |
| 基础校验 | 几何类型合法、拓扑正确、坐标系统一、包含属性表 |
| 坐标系 | CGCS2000，高斯-克吕格投影；导入时自动检测坐标系并转换 |
| 质检规则 | 几何缺失、拓扑错误、坐标系混用、矢量不含属性表、面积/范围异常 |

| 字段名 | 中文名称 | 必填 | 存储目标 | 说明 |
|---|---|---|---|---|
| geom | 几何（边界） | 空间对象必填 | geo.spatial_boundary.geom | PostGIS geometry，支持 Point/LineString/Polygon |
| centroid | 中心点 | 否 | geo.spatial_boundary.centroid | 由几何计算生成或手动指定 |
| boundaryCode | 边界编码 | 是 | geo.spatial_boundary.boundaryCode | 与对象编码可以不同，独立维护 |
| boundaryName | 边界名称 | 是 | geo.spatial_boundary.boundaryName | 如某某小区边界、某某街区范围 |
| boundaryType | 边界类型 | 是 | geo.spatial_boundary.boundaryType | city/district/street/community/block/building/parcel/renewalArea |

## 4. 来源元数据字段

来源元数据对应附录 A 中的提供单位、负责单位、数据分类、时间精度、空间精度、数据格式、采集方法等内容。元数据主要用于数据溯源、共享管理和导入映射。

| 字段名 | 中文名称 | 必填 | 存储目标 | 说明 |
|---|---|---|---|---|
| sourceName | 数据源名称 | 是 | base.data_source.sourceName | 如国土空间规划数据、基础测绘数据、地下管线数据 |
| providerOrgName | 提供单位/负责单位 | 是 | base.data_source 平台扩展属性 | 对应附录 A「提供单位」和「负责单位」 |
| providerOrgId | 提供单位 ID | 否 | base.data_source.providerOrgId | 引用 base.organization |
| updateCycle | 更新周期 | 否 | base.data_source.updateCycle | 如每年、每月、实时、一次性 |
| collectionMethod | 采集方法 | 是 | base.data_source 平台扩展属性 | 枚举见 3.3 节 |
| dataCategory | 数据分类 | 是 | base.data_source 平台扩展属性 | 调查类、统计类、测绘类、互联网数据 |
| dataFormat | 数据格式 | 是 | base.data_source.dataFormat | 枚举见 3.3 节 |
| timeAccuracy | 时间精度 | 是 | base.data_source 平台扩展属性 | 对应附录 A「时间精度」，如年、月、日 |
| spatialAccuracy | 空间精度 | 是 | base.data_source 平台扩展属性 | 对应附录 A「空间精度」，如街道、社区、经纬度、米级精度 |
| spatialReference | 坐标系 | 空间数据必填 | base.data_source.spatialReference | 如 CGCS2000、高斯-克吕格投影说明 |
| qualityLevel | 质量等级 | 否 | base.data_source.qualityLevel | 高、中、低、未知 |

## 5. 审核质检字段

审核质检字段不在采集表中直接暴露给采集人员，但属于数据流转和质检的必控维度。一期实现应在采集任务记录和审核记录中实现以下字段。

| 字段名 | 中文名称 | 必填 | 存储目标 | 说明 |
|---|---|---|---|---|
| collectionStatus | 采集状态 | 是 | 平台扩展字段：采集任务 | draft/submitted/qualityPassed/qualityRejected |
| qualityCheckStatus | 质检状态 | 是 | 平台扩展字段：采集任务 | pending/passed/failed |
| qualityCheckResult | 质检结果 | 否 | 平台扩展字段：采集任务 | 记录每项校验的通过/不通过详情 |
| reviewedBy | 审核人 | 审核后必填 | 平台扩展字段：审核记录 | 参考 base.user |
| reviewedAt | 审核时间 | 审核后必填 | 平台扩展字段：审核记录 | 用于审核留痕 |
| reviewComment | 审核意见 | 否 | 平台扩展字段：审核记录 | 驳回时说明原因 |
| reviewStatus | 审核状态 | 是 | 平台扩展字段：审核记录 | pending/approved/rejected/recheckRequired |

## 6. 附录 A 字段映射对照

河北省城市体检数据采集表附录 A 的列与本文字段类型的对应关系：

| 附录 A 列名 | 对应字段类型 | 对应标准字段 | 备注 |
|---|---|---|---|
| 基本单元 | 枚举 | objectType | 住房、小区社区、街区、城区城市 |
| 序号 | 平台业务字段 | rowNumber | 平台扩展字段，仅用于表内排序和导入定位 |
| 指标名称 | 文本 | indicatorName | 应与指标库匹配 |
| 数据名称 | 文本 | dataName | 支撑指标的原始数据项名称 |
| 数据值 | 数字/文本/枚举 | indicatorValue | 由指标定义决定具体类型 |
| 提供单位 | 文本/组织引用 | providerOrgName / providerOrgId | 映射到数据源元数据 |
| 数据分类 | 枚举 | dataCategory | 调查类、统计类、测绘类、互联网数据 |
| 时间精度 | 枚举/文本 | timeAccuracy | 年、月、日、时分秒等 |
| 空间精度 | 枚举/文本 | spatialAccuracy | 行政区、街道、社区、经纬度、米级精度等 |
| 数据格式 | 枚举 | dataFormat | xlsx、csv、geojson、shp、pdf、api 等 |
| 采集方法 | 枚举 | collectionMethod | 现场踏勘、部门报送、遥感识别、API 调用等 |
| 负责单位 | 文本/组织引用 | providerOrgName / providerOrgId | 与提供单位可相同，也可区分管理责任单位 |
| 说明材料 | 附件 | attachments | 平台扩展字段，保存台账、图纸、规划、政策材料等 |

## 7. 质量检查规则

| 检查类别 | 适用字段 | 检查规则 | 不通过处理 |
|---|---|---|---|
| 必填检查 | 所有必填字段 | 字段不得为空 | 阻止提交或导入 |
| 类型检查 | text/number/date/enum/coordinate/geometry | 值必须符合字段类型 | 标记字段错误 |
| 范围检查 | number/date/coordinate | 数值范围、日期范围、坐标范围合法 | 标记异常并要求确认或修改 |
| 枚举检查 | enum | 值必须在字典范围内 | 进入导入错误清单 |
| 空间检查 | coordinate/geometry | 坐标系明确，点位不应超出所属对象边界，几何拓扑合法 | 阻止入库或进入人工复核 |
| 附件完整性 | photo/attachment | 必要照片、台账、图纸、政策材料齐全且可打开 | 阻止提交或提示补充 |
| 溯源检查 | source metadata | 数据源、提供单位、时间精度、空间精度、采集方法完整 | 标记为溯源不完整 |
| 时效性检查 | date/source metadata | 统计类数据截止日期与体检年份匹配，动态数据使用最新时间戳 | 标记为时效性风险 |

## 8. 表单 schema 实现约束

1. 每个采集字段至少应包含字段名、中文名称、字段类型、是否必填、存储目标、校验规则和说明。
2. 指标类字段应优先落到 health.indicator_value；对象身份和对象层级字段应优先落到 health.health_object。
3. 空间边界、中心点、面积和空间来源应统一关联 geo.spatial_boundary，不应在多个业务表中重复保存边界几何。
4. 数据来源、提供单位、数据格式、坐标系、更新周期和质量等级应统一沉淀到 base.data_source。
5. 照片、附件、审核、采集任务等未在对象库数据库设计中展开的内容，必须标注为平台扩展字段。
6. 导入映射必须保留原始列名、目标字段、转换规则和错误信息，保证问题可追溯。
7. 移动端采集必须对坐标缺失、照片缺失、附件缺失和提交时间进行质检。
8. 空间采集应同时支持自由采集和任务采集：自由采集支持地图点选、画线、画面；任务采集支持内业整理矢量下发调查、外业确认或修正几何后回填属性。
9. 不同维度的专项字段可以在表单中扩展，但必须声明所属对象类型、指标编码和存储目标。
