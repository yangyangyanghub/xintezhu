# 城市更新资源采集标准

## 1. 编制依据与范围

本标准用于城市体检更新平台一期城市更新资源采集、空间落图、资源入库和重点更新片区策划，主要依据 [[邯郸市2026年城市体检和县城体检工作方案]] 附件 9"邯郸市城市更新资源类型划分一览表"，并对齐 `renewal.renewal_resource` 与 `renewal.renewal_area` 数据设计。

资源分类包括：居住类、产业类、公共设施类、公共空间类、历史风貌类、其他类。采集成果必须支撑"体检问题—更新资源—更新片区—更新项目"闭环。

## 2. 入库字段对齐

### 2.1 资源对象：`renewal.renewal_resource`

| 字段 | 采集要求 |
|---|---|
| `resourceCode` | 必填。统一资源编码，建议 `城市代码-资源类型-年份-序号`。 |
| `resourceName` | 必填。资源现状名称、登记名称或通用名称。 |
| `resourceType` | 必填。对应本标准资源类型编码。 |
| `relatedObjectId` | 条件必填。关联住房、小区、街区、城区等体检对象。 |
| `relatedIssueId` | 条件必填。由体检问题识别形成资源时填写。 |
| `renewalAreaId` | 条件必填。纳入重点更新片区时填写。 |
| `spatialBoundaryId` | 必填。关联空间点、线、面边界。 |
| `currentUse` | 必填。现状用途、现状功能或使用状态。 |
| `plannedUse` | 条件必填。规划用地性质、拟更新功能或更新方向。 |
| `ownershipType` | 必填。国有、集体、混合、争议、不明。 |
| `landArea` | 条件必填。用地面积，平方米或公顷。 |
| `buildingArea` | 条件必填。建筑面积，平方米。 |
| `safetyRiskLevel` | 必填。无、低、中、高、重大。 |
| `potentialLevel` | 必填。低、中、高。依据区位、闲置低效程度、意愿、规划适配度判定。 |
| `reuseValue` | 必填。居住改善、产业转型、公共服务补短板、公共空间提升、历史文化保护、基础设施更新等。 |

### 2.2 片区对象：`renewal.renewal_area`

| 字段 | 采集要求 |
|---|---|
| `areaCode` | 必填。重点更新片区编码，建议 `城市代码-AREA-年份-序号`。 |
| `areaName` | 必填。片区名称。 |
| `areaType` | 必填。老旧生活街区、老旧厂区、商业街区、城中村、历史文化街区、公共空间提升片区、综合更新片区等。 |
| `spatialBoundaryId` | 必填。片区边界面要素。 |
| `dominantProblem` | 必填。房屋安全、设施短板、空间低效、产业衰退、风貌失序、管网老化等。 |
| `resourcePotentialLevel` | 必填。低、中、高。 |
| `stakeholderWillingnessLevel` | 必填。居民、产权单位、运营单位、属地政府等意愿等级。 |
| `planningGuidance` | 必填。规划用地性质、控制要求、保护要求、更新方向。 |
| `status` | 必填。draft、confirmed、planned、implemented、archived。 |

## 3. 数据成果要求

| 数据类型 | 格式 | 存储目标 | 采集要求 |
|---|---|---|---|
| 空间数据 | SHP / GeoJSON | geo.spatial_boundary、资源空间附件 | 建筑和设施采点或面；道路、管线、慢行系统采线；街区、小区、村庄、厂区、绿地、滨水空间、历史街区采面。属性包含资源编码、名称、类型、现状情况、主要问题、权属、规划用地性质。 |
| 档案数据 | PDF | base.data_source、附件表、资源档案 | 包含基本情况、权属材料、鉴定报告、普查登记表、保护规划、改造计划、统计表格、用地规划图等。 |
| 影像数据 | JPG | 附件表、影像库 | 至少包含现状全景和问题细节；建筑类含正面、侧面、内部或楼道；道路管网类含典型病害；历史风貌类含全景、细节、历史照片。 |
| 视频数据 | MP4 | 附件表、影像库 | 城市记忆场所可采集现状视频、访谈视频、民俗活动视频或工艺流程视频。 |

## 5. 编码规则与质量控制

### 5.1 编码规则

资源编码采用：`城市代码-资源类型代码-年份-序号`。

- 城市代码：采用 6 位行政区划代码，如 `130400`。
- 资源类型代码：采用本标准第 4 章编码。
- 年份：采集或体检年度，如 `2026`。
- 序号：4 位流水号，如 `0001`。

示例：`130400-RES-OLD-QUARTER-2026-0001`。

### 5.2 必填校核

1. 每条资源必须有 `resourceCode`、`resourceName`、`resourceType`、`spatialBoundaryId`、`currentUse`、`ownershipType`、`safetyRiskLevel`、`potentialLevel`、`reuseValue`。
2. 面状资源必须填写 `landArea`；建筑类资源必须填写 `buildingArea`；道路、管线、慢行系统必须填写长度。
3. 危险住房、疑似危险住房、危旧桥梁、老旧地下管网必须填写安全问题、鉴定或检测依据。
4. 历史建筑、潜在历史建筑、历史文化街区、城市记忆场所必须填写价值属性和保护/活化问题。
5. 纳入重点更新片区的资源必须填写 `renewalAreaId`，并保证片区边界覆盖或相交资源边界。

### 5.3 存储目标

| 数据成果 | 目标表或库 | 说明 |
|---|---|---|
| 资源主记录 | `renewal.renewal_resource` | 存储资源基本属性、风险、潜力和利用价值。 |
| 片区主记录 | `renewal.renewal_area` | 存储重点更新片区边界、主导问题、潜力等级和规划指引。 |
| 空间边界 | `geo.spatial_boundary` | 存储点、线、面几何与版本信息。 |
| 数据来源 | `base.data_source` | 登记规划、测绘、管线、房产、档案、手工采集等来源。 |
| 问题关联 | `issue.issue` | 支撑问题转资源、资源转项目。 |
| 附件资料 | 附件表 / 文件库 | 存储 PDF、JPG、MP4 等采集材料。 |

## 6. 与更新片区划定的关系

符合以下任一情况时，应从资源对象进一步抽象形成 `renewal.renewal_area`：

1. 同一街区或相邻街区内集聚多个老旧小区、城中村、老旧生活街区资源；
2. 老旧厂区、老旧厂房、低效商务楼宇、商业街区具备连片更新和功能重构条件；
3. 道路、地下管网、公共服务设施、公共空间问题叠加，需统筹改造；
4. 历史建筑、历史文化街区、城市记忆场所与周边公共空间、商业街区存在整体保护利用需求；
5. 滨水空间、绿色空间、慢行系统、桥下空间可形成连续公共空间提升带。

片区划定后，应同步填写 `dominantProblem`、`resourcePotentialLevel`、`stakeholderWillingnessLevel` 和 `planningGuidance`，并将片区内资源通过 `renewalAreaId` 挂接。
