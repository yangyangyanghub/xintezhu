# 今日内容
- [ ] #开发每日例会 本次会议详细介绍了“氚云”系统的现有流程，包括项目信息录入、合同管理以及任务分配等模块的具体功能和逻辑。
	小结1
	​​1. 系统现有流程与模块介绍​​
	系统包含多个表单模块，主要分为项目生产管理（表单1-5）、日常证件管理（表单6）、以及项目概要等。
	表单1-5为核心业务流程，表单6为静态信息表，表单7为特定项目申请。
	系统通过API实现对甲方信息的模糊和精确查询，以保证数据准确性。
	
	​​2. 项目信息录入流程​​
	​​表单一​​： 委托单位信息登记，此表为静态表，仅供后续流程调用，非必要不显示给用户。
	​​表单二​​： 项目信息登记，作为核心汇总表单，会自动关联并填充来自表单一的信息，以减少重复录入。
	​​表单三​​： 项目信息变更，用于在项目生命周期内修改部分信息，如项目名称、状态等。
	​​表单四​​： 小项目金额变更，用于未签合同的小项目，手动录入金额以确保项目信息的完整性。
	​​3. 合同管理流程​​
	​​合同会审​​： 流程复杂，分为不同金额和类型的合同（如是否为补录项），需经过多级审批。
	​​合同签订​​： 区分“甲方先盖章”和“双方共同盖章”两种情况，通过二杠二和二杠三表单完成。
	​​合同类型​​： 收入合同与合作合同流程相似，但在填写合作合同时，需注意将“被委托方”信息正确填写。
	​​4. 任务分配与实施​​
	项目登记完成后即可开始任务分配，与合同签订流程并行，不受限制。
	任务分配由项目负责人或所队长发起，系统会将任务派发给相关人员。
	小结2
	​​1. 项目管理核心流程​​
	​​三杠三质量会审​​: 此为部门内部的审核流程，项目负责人提交后，由核对人进行审核，审核完成后流程结束。
	​​三杠四重点项目会审​​: 此为全院层面的审核流程，仅针对特定条件的“重点项目”（如项目子类为规划竣工、不动产等，或金额大于十万）。流程为：发起人提交 -> 部门质检确认 -> 部门审核 -> 汇交会审意见 -> 由院总工/专业总工最终审批。
	​​三杠六重点项目调度会​​: 此流程与“重点项目会审”作用相似，但形式更灵活，主要用于记录项目在不同阶段的会议内容，可多次更新。
	​​2. 项目规模填报​​
	此表单主要用于市政和建筑专业，旨在提前登记项目规模（大型、中型、小型），以便后续统一管理和资质申报。
	项目规模的大小将根据预设的指标范围进行自动判断和生成。
	​​3. 成果出具与归档​​
	​​成果出具登记​​: 在此环节，用户需关联项目并填写成果信息（如报告、图纸），系统将自动填充相关信息，简化操作。
	​​无合同项目处理​​: 对于有项目但未签合同的情况，系统仍需关联项目，但会通过“无合同项目金额变更”等特殊表单进行处理。	
	小结3
	​​1. 开票与收款流程​​
	​​开票流程​​: 新增“五杠零”表单用于线上开票申请，关联具体项目并经过生产处、财务等多级审批。系统将自动生成“五杠一”表单，用于记录开票信息。
	​​收款流程​​: “五杠二”表单用于线上收款登记，财务人员上传银行回单后，需由生产处负责人确认收款归属项目，完成后系统将自动同步更新“五杠一”和“一杠三”中的收款状态。
	​​2. 大额资金支付与项目分账​​
	​​大额支付​​: 超过2万元的支付需进行线上“大额支付申请”，并经过严格的线上审批流程。系统内置了防止超额支付和支付金额大于剩余合同金额的校验规则。
	​​项目分账​​: 在项目收款完成后，可通过“项目收款分配”功能进行内部分账。此操作由生产处填写并提交，完成后相关信息会同步至“一杠三”表单，并在数据驾驶舱中实时展示。
	​​3. 基本收入及目标收入指标​​
	此为年度KPI指标表，由院里每年初下达，用于考核各部门的业绩。该表与项目数据通过“高级数据源”关联，在数据驾驶舱中以图表形式直观展示各部门的基本收入和目标收入完成情况。
	​​4. 系统权限管理​​
	系统的权限配置与钉钉组织架构同步。管理员可在后台为不同角色（如项目负责人、部门副职、总工等）分配权限，确保用户只能访问其职责范围内的数据。
	小结4
	​​1. 系统整体架构与模块划分​​
	系统将划分为五个核心模块：工作台、合同管理、项目管理、财务管理、人员与资质管理。
	工作台作为统一门户，包含待办事项、数据概览等核心功能，其数据概览部分将设计为数据大屏，供不同权限的用户查看。
	​​2. 合同与项目管理​​
	合同管理模块将整合收入合同、合作合同及合同变更等功能，其中合同金额需精确到小数点后六位。
	项目管理将采用标签页形式，方便用户在项目基础信息、合同、进展、收款等不同模块间快速切换。
	项目详情将根据用户权限进行展示，如项目负责人仅能看到自己创建的项目。
	​​3. 财务管理​​
	财务管理模块将整合收款登记、付款申请、开票申请等所有涉及资金的操作。
	明确了项目支出、已收款、未收款等财务数据将在此模块进行管理。
	​​4. 人员与资质管理​​
	讨论了将人员管理、职称、注册证等信息整合到统一的“人员管理”模块下的可行性。
	单位资质管理将包含组织架构、单位资质、人员资质等信息。
	​​5. 权限与流程设计​​
	权限体系将根据用户角色（如项目负责人、部门领导、副院长等）进行精细化配置，确保不同用户看到的信息和操作权限一致。
	讨论了采用流程图动态显示项目各环节进度（如合同签署、填报等）的直观性，替代静态的文字描述。
