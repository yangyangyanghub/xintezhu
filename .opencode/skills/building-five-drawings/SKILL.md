# Skill: 建筑五图生成器（主控）

建筑设计五张图纸一键生成，串联五个独立技能形成完整工作流。

## 子技能清单

| 序号 | 技能名称 | 功能 | 文件位置 |
|------|---------|------|---------|
| 1 | 设计概念图生成器 | 概念图、意向图 | `building-concept/SKILL.md` |
| 2 | 总平面图生成器 | 总平面图、场地规划 | `building-masterplan/SKILL.md` |
| 3 | 分析图生成器 | 功能分析、流线分析 | `building-analysis/SKILL.md` |
| 4 | 效果图生成器 | 透视图、鸟瞰图 | `building-rendering/SKILL.md` |
| 5 | 施工图生成器 | 节点大样、材料做法 | `building-construction/SKILL.md` |

## 触发场景

当用户提到以下关键词时使用此技能：
- 建筑五图、建筑设计全套图纸
- 建筑方案、设计汇报
- 建筑出图、建筑设计
- 概念图+总平面+分析图+效果图+施工图

## 核心指令

### 流程总览

```
用户输入项目需求
    ↓
[1. 设计概念图] → 设计理念、风格方向
    ↓
[2. 总平面图] → 建筑布局、环境关系
    ↓
[3. 分析图] → 功能分区、流线组织
    ↓
[4. 效果图] → 建筑外观、材质表现
    ↓
[5. 施工图] → 节点大样、材料做法
    ↓
输出整理 → 五张图纸打包
```

### 步骤1：需求解析

输入：项目描述（文字/图片/草图）

解析维度：
- 建筑类型（住宅/商业/办公/文化）
- 建筑规模（层数、占地面积）
- 设计风格（现代/古典/极简/地域）
- 场地条件（朝向、周边环境）
- 特殊要求（功能需求、材质偏好）

输出：结构化项目信息

### 步骤2：调用子技能

按顺序调用五个子技能：

**2.1 设计概念图**
```
调用：building-concept
输入：项目描述 + 设计理念
输出：概念图2-3张 + 概念词总结
```

**2.2 总平面图**
```
调用：building-masterplan
输入：项目描述 + 场地条件
输出：总平面图DWG + 彩色总平面PNG
```

**2.3 分析图**
```
调用：building-analysis
输入：项目描述 + 分析需求
输出：功能分区图 + 流线分析图 + 日照分析图
```

**2.4 效果图**
```
调用：building-rendering
输入：项目描述 + 概念图参考
输出：透视图 + 鸟瞰图 + 夜景图
```

**2.5 施工图**
```
调用：building-construction
输入：项目描述 + 材料做法
输出：墙身大样 + 门窗节点 + 屋面做法
```

**FreeCAD脚本模板**：
```python
import FreeCAD
import Part
import Draft

def create_wall_section():
    # 墙身剖面
    # 结构层
    structure = Part.makeBox(200, 240, 100)
    # 保温层
    insulation = Part.makeBox(200, 50, 100)
    # 饰面层
    finish = Part.makeBox(200, 20, 100)
    return [structure, insulation, finish]

def add_dimensions():
    # 添加尺寸标注
    Draft.makeDimension(start, end, offset)

def generate_construction_drawing():
    parts = create_wall_section()
    add_dimensions()
    Part.export(parts, "wall_section.dwg")
```

- 格式：DWG（CAD）+ PNG（预览）

## 约束条件

### 必须遵守
- 符合建筑设计规范（消防、日照、无障碍）
- 图纸内容准确、标注清晰
- 风格统一、逻辑自洽
- 输出文件可直接用CAD/3D软件打开

### 禁止行为
- 过度艺术化导致结构失真
- 关键信息遗漏（尺寸、材料、做法）
- 图纸之间矛盾（平面与立面不符）
- 使用不规范的表达方式
