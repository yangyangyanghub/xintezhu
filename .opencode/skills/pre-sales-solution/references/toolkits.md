## 工具A：AI辅助预判三步法（见客户前10分钟必备）

### 三步核心动作

| 步骤 | 核心动作 | 关键提示词模板 |
|------|---------|---------------|
| **1. 信息聚合** | 10分钟建立客户画像 | `搜索[公司名]近一年公开信息，总结：业务模式/战略重点/主要挑战/高管关注点` |
| **2. 痛点推演** | 用SPIN逻辑放大痛点 | `一家[规模]的[行业]企业正在[做什么事]，从[3个维度]分析可能面临的具体痛点` |
| **3. 策略准备** | 对标案例+价值提炼 | `搜索[行业名]企业解决[痛点描述]的案例，总结：问题背景/解决思路/实施效果数据` |

### 预判验证法
见面时用引导性问题验证你的痛点预判，不要直接推销：
> 对应痛点"排程效率低"的验证问题："目前排程主要靠什么方式？换产频率大概多少？"

## 工具B：四大经典销售方法论

### B1：SPIN提问法——深挖客户需求

```
S 背景 → P 难点 → I 暗示 → N 需求-效益
```

| 阶段 | 目的 | 示例问题 |
|------|------|---------|
| S 背景 | 建立基础认知 | "您目前在用什么系统？团队规模多大？" |
| P 难点 | 挖掘具体困难 | "现有系统在高峰期会不会卡慢？" |
| I 暗示 | 放大业务影响 | "这个问题会不会影响项目交付周期？会不会增加额外成本？" |
| N 需求-效益 | 引导客户说出价值 | "如果能解决这个问题，对你们业务有什么帮助？" |

### B2：FABE法则——方案呈现结构化

```
F 特征 → A 优势 → B 利益 → E 证据
```

| 阶段 | 含义 | ❌ 错误写法 | ✅ 正确写法 |
|------|------|-----------|-----------|
| F 特征 | 产品是什么 | "分布式架构" | "采用分布式云原生架构" |
| A 优势 | 比别人好在哪 | "性能更强" | "比传统架构处理速度提升5倍" |
| B 利益 | 给客户带来什么 | "提高效率" | "高峰期系统不卡顿，不影响业务开展" |
| E 证据 | 怎么证明 | "很多客户在用" | "某省厅上线后，高峰期响应从10秒降到2秒" |

### B3：波特五力——行业分析框架

用于方案第一章"行业趋势分析"：

| 五力 | 在售前方案中的应用 |
|------|------------------|
| 供应商议价能力 | "测绘数据供应商集中→我们自研算法降低数据依赖" |
| 购买者议价能力 | "甲方预算收紧→我们方案帮助降本提效" |
| 新进入者威胁 | "新玩家涌入→AI能力建立技术壁垒" |
| 替代品威胁 | "传统手工→AI数字化转型" |
| 同业竞争程度 | "头部效应明显→方案抢占先机" |

### B4：九大方法论全景

| 方法论 | 适用阶段 | 核心作用 |
|--------|---------|---------|
| SPIN | 需求调研 | 深挖客户痛点 |
| FABE | 方案呈现 | 技术转客户利益 |
| 波特五力 | 行业分析 | 建立宏观视角 |
| SWOT | 竞争分析 | 知己知彼 |
| PDCA | 项目管理 | 实施质量管控 |
| RACI | 团队协作 | 明确职责分工 |
| KANO | 需求管理 | 控制范围蔓延 |
| SMART | 目标设定 | 量化可验证 |
| 金字塔原理 | 汇报表达 | 结构化逻辑清晰 |

## 工具C：沟通与汇报实战技巧

### C1：技术负责人沟通5问顺序（绝对不能乱）

```
1. 现状摸底 → 2. 痛点诊断 → 3. 目标澄清 → 4. 边界确认 → 5. 方案验证
```

| 顺序 | 问题类型 | 核心原则 | 反例 |
|------|---------|---------|------|
| 1 | 现状摸底 | 先听再说，建立信任感 | ❌ 一坐下就讲公司介绍 |
| 2 | 痛点诊断 | 让他自己说清楚"伤在哪" | ❌ 替他定义痛点："您这块问题我们见多了" |
| 3 | 目标澄清 | 把解决痛点和达成目标连起来 | ❌ 只聊问题不聊目标 |
| 4 | 边界确认 | 预算/时间/决策权此时问才自然 | ❌ 一上来就问预算 |
| 5 | 方案验证 | 方案是对话的回应，不是会议主角 | ❌ 花80%时间念PPT |

### C2：分层介绍术（领导/中层/技术混坐一桌）

| 对象 | 关注点 | 表达重点 |
|------|--------|---------|
| 老板层 | ROI、战略价值、风险 | "这个方案帮您实现XX战略目标，投入产出比1:5，风险可控" |
| 中层 | 效率提升，管理方便 | "上线后，工作流全流程线上化，管理效率提升50%" |
| 技术层 | 架构、兼容、可维护性 | "微服务架构，兼容现有系统，有API方便二次开发" |

### C3：无稿开口三层逻辑树

```
┌──────────────────────────────────────────────┐
│ 顶层：一句话主题（3秒抓住注意力）                │
│ 例："今天聊AI提升耕地监测效率的方案"             │
├──────────────────────────────────────────────┤
│ 中层：3个核心论点（结构化逻辑）                   │
│ 论点1：痛点分析 → 论点2：方案设计 → 论点3：价值收益 │
├──────────────────────────────────────────────┤
│ 底层：每个论点配1个案例 + 1个数据（说服力）        │
│ 案例：某县局用后准确率95%                        │
│ 数据：效率提升80%，成本降低60%                    │
└──────────────────────────────────────────────┘
```

### C4：领导必问6类问题预判

提前准备好三层答案：

| 问题类型 | 示例问题 | 三层答案法 |
|---------|---------|-----------|
| 数据来源 | "这个数据哪里来的？" | 1.直接答数据来源 → 2.解释数据可靠性 → 3.对决策有什么帮助 |
| 竞品对比 | "跟超图/Esri比优势在哪？" | 1.直接答差异化 → 2.解释技术优势 → 3.对客户意味着什么 |
| 价格成本 | "预算大概多少？ROI怎么算？" | 1.直接给价格区间 → 2.分解成本结构 → 3.量化投资回报 |
| 风险点 | "有没有什么坑？万一失败怎么办？" | 1.承认风险存在 → 2.给出应对预案 → 3.降低客户顾虑 |
| 实施周期 | "多久能上线？人天多少？" | 1.给具体时间 → 2.分阶段里程碑 → 3.客户收益时间点 |
| 成功案例 | "有没有同行业客户在用？" | 1.举1-2个案例 → 2.效果数据 → 3.客户证言 |

### C5：格局拔高法——汇报拉开差距

| 维度 | 普通售前 | 优秀售前 |
|------|---------|---------|
| 时间维度 | 只讲现在解决问题 | "从行业趋势看，过去→现在→未来" |
| 空间维度 | 只讲本系统 | "部门→全局→生态，系统打通数据共享" |
| 价值维度 | 只讲功能点 | "不仅解决您问题，还能评先进，当标杆" |

## 工具D：售前认知进阶

### D1：甲方思维

不是销售让你干啥你干啥，而是：
- ✅ **会提要求**："这个项目要这些条件才能做好"
- ✅ **主动质疑**："这个需求的前提是什么？会不会有问题？"
- ✅ **把控边界**：什么在范围，什么不在，界定清楚，不背锅
- ✅ **掌握主动**：引导客户思路，跟着你的逻辑走

### D2：痛点深挖公式

```
不是"客户系统慢"
而是"系统慢导致每个月耽误200万订单损失"
→ 核心：用数据量化疼痛程度
```

三步法：
1. 先记录5个表层问题
2. 追问：这个问题影响了什么业务？影响了哪些人？影响多大？
3. 探寻根源：技术问题？流程问题？组织问题？

### D3：气场建立两招

1. **不完美才真实**：不要什么都懂，适度暴露小破绽，反而可信
2. **少术语多案例**：不说"行业领先"，多说"我在某局做过一个项目..."

## 踩坑避坑速查

| 坑点 | 正确做法 |
|------|---------|
| 一坐下就讲公司介绍 | 先用SPIN问现状，建立信任再推方案 |
| 客户说"太贵了"就开始降价 | 用FABE重述价值，不先谈价格 |
| 被技术问题问住慌了 | 用三层答案法，先直接答，再解释，最后升华 |
| 方案写了50页还没重点 | 砍到只聚焦1-2个核心场景，每个场景讲透 |
| 汇报照PPT念 | 每页只记3个关键词，围绕关键词展开，不背稿 |
