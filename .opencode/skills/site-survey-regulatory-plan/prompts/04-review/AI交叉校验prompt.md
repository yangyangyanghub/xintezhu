# AI交叉校验 Prompt

## 任务
对已生成的调研报告和各类表格做AI自动交叉校验，发现数据不一致、逻辑矛盾、规范不符合等问题。

## 输入
- 调研报告全文（Word/PDF/Markdown）
- 所有统计表格（Excel）
- CAD现状图（可选，如能解析）

## 输出
《AI交叉校验报告》，包含以下5个方面的检查结果：

---

### 检查一：数据一致性检查

| 序号 | 检查项 | 检查方法 | 问题描述 | 严重等级 |
|------|--------|---------|---------|---------|
| 1.1 | 同一数据在不同章节是否一致 | 搜索同一数据在不同位置的出现，对比数值 | | |
| 1.2 | 分项加总是否等于合计 | 检查所有表格的分项加总是否与合计行一致 | | |
| 1.3 | 面积/长度单位是否统一 | 检查是否混用亩/公顷/平方米，米/公里等 | | |
| 1.4 | 百分比计算是否正确 | 检查百分比=分项/总计×100是否正确 | | |
| 1.5 | 前后引用是否矛盾 | 如"第三章说有5栋建筑，第五章说有8栋"这类矛盾 | | |

**检查示例**：
- 第2.2节说耕地面积125亩，但第6.1节说耕地150亩
- 表格中建筑分项面积加总为4520㎡，但合计写5000㎡
- 同一条管线，在给水章节写DN300，在管线综合章节写DN400

---

### 检查二：逻辑合理性检查

| 序号 | 检查项 | 检查方法 | 问题描述 | 严重等级 |
|------|--------|---------|---------|---------|
| 2.1 | 分析结论是否与数据匹配 | 结论与支撑数据是否有因果关系 | | |
| 2.2 | 风险等级判定是否合理 | 问题描述与风险等级是否匹配 | | |
| 2.3 | 建议措施是否针对问题 | 给出的建议是否能解决对应的问题 | | |
| 2.4 | 是否有前后矛盾的表述 | 前文说"管线容量足够"，后文又说"需要扩容" | | |
| 2.5 | 数字大小逻辑是否合理 | 管径、距离、面积、标高等是否符合常识 | | |

**检查示例**：
- 问题描述只是"管线有点旧"，但判定为🔴一级风险 → 不合理
- 结论说"供水能力充足"，但数据显示现状已用95%容量 → 不匹配
- 建议"增加绿化"，但问题是"防洪标准不够" → 不对应

---

### 检查三：规范符合性检查

| 序号 | 检查项 | 检查依据 | 问题描述 | 严重等级 |
|------|--------|---------|---------|---------|
| 3.1 | 用地性质与控规是否一致 | 《城市用地分类与规划建设用地标准》GB50137 | | |
| 3.2 | 建筑退距是否符合规范 | 当地城市规划管理技术规定 | | |
| 3.3 | 管线安全距离是否足够 | 《城市工程管线综合规划规范》GB50289 | | |
| 3.4 | 防洪标准是否符合要求 | 《防洪标准》GB50201 | | |
| 3.5 | 消防间距是否符合规范 | 《建筑设计防火规范》GB50016 | | |

**检查示例**：
- 110kV高压线距建筑仅5米，规范要求15米以上 → 不符合
- 河道蓝线内画了建筑 → 不符合
- 加油站距住宅仅20米，规范要求50米 → 不符合

---

### 检查四：遗漏项检查

| 序号 | 检查项 | 检查内容 | 问题描述 | 严重等级 |
|------|--------|---------|---------|---------|
| 4.1 | 一票否决项是否全覆盖 | 机场净空、铁路/高速控制线、输油管道、军事设施、生态红线是否都核查了 | | |
| 4.2 | 各专业内容是否完整 | 给水/排水/电力/燃气/交通/生态每个专业是否都有现状、问题、建议三部分 | | |
| 4.3 | 所有测点是否都反映在报告中 | 现场测了100个点，报告中是否都用到了？有没有遗漏重要点？ | | |
| 4.4 | 所有照片是否都有对应说明 | 拍了200张照片，是否都有编号和文字说明？ | | |
| 4.5 | 关键参数是否都有数值 | 如"管径很大"但没写具体多大，"距离很远"但没写多少米 | | |

**检查示例**：
- 报告9个专业，唯独没提燃气 → 遗漏
- 现场发现了高压线，但报告中"特殊限制因素"章节没写 → 遗漏
- 大量模糊表述："水质不好"（具体指标？）、"交通有点堵"（具体流量？）

---

### 检查五：风险等级准确性复核

| 风险等级 | 判定标准（必须同时满足） | 检查情况 | 调整建议 |
|---------|------------------------|---------|---------|
| **🔴 一级风险** | 1. 可能导致项目选址变更<br>2. 或可能导致方案颠覆性调整<br>3. 或涉及重大安全隐患 | | |
| **🟠 二级风险** | 1. 可能导致设计方案重大调整<br>2. 或造价增加10%-30%<br>3. 或工期延误1个月以上 | | |
| **🟡 三级风险** | 1. 仅影响局部设计<br>2. 造价增加10%以内<br>3. 不影响整体方案可行性 | | |

**检查方法**：
1. 把所有🔴一级风险逐条拿出来，对照判定标准看是否真的够一级
2. 有没有应该是一级但被写成二级的？（漏网之鱼）
3. 有没有应该是三级但被夸大写成二级的？（过度恐慌）
4. 有没有同一问题在不同地方被写成不同等级？

## 使用提示

1. AI校验只是辅助工具，最终质量责任在项目负责人和审核人
2. AI不能替代专业工程师的判断，特别是涉及规范条文的解读
3. 凡是AI标注为"可能有问题"的，都需要人工二次确认
4. 建议AI校验后再安排不同专业的工程师交叉人工审核
5. 千万不要因为"AI校验通过了"就放松人工质量把关！
