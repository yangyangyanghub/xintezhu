# 日报智能分析器

从钉钉获取员工日报数据，进行六维度智能评分，生成Markdown和HTML可视化报告。

## 核心能力

| 能力 | 说明 |
|------|------|
| 数据获取 | 从钉钉OA审批获取日报数据，包含完整审批流程信息 |
| 六维度评分 | 完整度、内容质量、AI应用、及时性、工作量、计划性 |
| 流水账检测 | 自动识别"只写项目名不说具体工作"等敷衍内容 |
| 审批效率分析 | 统计审批耗时、首审延迟、部门完成率 |
| 可视化报告 | 生成HTML图表报告 + Markdown报告 + Excel清单 |

---

## 一、快速开始

### 1.1 配置钉钉凭证

创建配置文件 `~/.dingtalk/config.json`：

```json
{
  "app_key": "your_app_key",
  "app_secret": "your_app_secret"
}
```

### 1.2 申请权限

在钉钉开放平台申请：
- `ProcessInstance.Read` - 审批实例读权限
- `Workflow.Process.Read` - 审批流程读权限

### 1.3 一键执行

```bash
# 分析今日日报
python .opencode/skill/daily-report-analyzer/scripts/run_all.py --date=today

# 分析指定日期
python .opencode/skill/daily-report-analyzer/scripts/run_all.py --date=2026-03-12
```

### 1.4 查看报告

报告生成在 `daily-reports/YYYY-MM-DD/` 目录：
- `report.html` - HTML可视化报告（推荐）
- `daily-report.md` - Markdown分析报告
- `data-list.xlsx` - Excel数据清单

## 二、评分体系

### 2.1 六维度评分

日报评分基于六个维度综合计算，满分5分：

| 维度 | 权重 | 评估重点 |
|------|------|----------|
| 完整度 | 20% | 工作内容+复盘+计划是否完整 |
| 内容质量 | 25% | 是否详细描述，是否流水账 |
| AI工具应用 | 20% | 是否用AI解决具体问题 |
| 及时性 | 15% | 提交时间 |
| 工作量 | 10% | 内容丰富程度 |
| 计划性 | 10% | 明日计划和协调事项 |

**为什么这样设计**：内容质量和AI应用权重最高，因为这两个维度最能体现日报的实际价值。流水账式日报对团队协作没有帮助，AI工具的使用则能反映员工的工作效率意识。

### 2.2 AI工具应用评分

AI应用评分的关键是"是否解决具体问题"，而非仅仅提到工具名：

| 分值 | 标准 | 示例 |
|------|------|------|
| 5分 | 明确解决具体问题 | "使用AI协助接口开发" |
| 4分 | 有应用场景 | "查询相关规范标准号" |
| 3分 | 只提工具名 | "AI豆包应用" |
| 2分 | 诚实填写未使用 | "无" |
| 1分 | 未填写 | - |

### 2.3 流水账检测

系统自动识别以下问题并标记：
- 工作内容少于10字
- 只有项目名，没有具体工作描述（如"馆陶督导"、"项目对接"）
- 没有动作动词（完成、进行、开展等）

**详细评分规则**：见 `references/评分标准.md`

## 三、审批效率分析

当通过OA审批获取数据时，会自动分析审批效率：

### 3.1 分析指标

| 指标 | 说明 |
|------|------|
| 完成率 | 审批通过的日报占比 |
| 平均审批耗时 | 从提交到审批完成的平均时长 |
| 首审延迟 | 部门主管审批的平均等待时间 |
| 部门排名 | 按审批效率和完成率排名 |

### 3.2 价值

审批效率数据可以帮助管理者：
- 发现审批流程中的瓶颈
- 识别哪些部门审批较慢
- 优化审批流程配置

## 八、常见问题

**Q: 无法获取日报数据？**
- 检查权限是否已申请并生效
- 确认appKey/appSecret正确
- 检查API调用次数是否超限

**Q: 审批效率数据为空？**
- 确保使用 `fetch_oa_reports.py` 获取数据（而非日志API）
- 审批效率分析需要 `ProcessInstance.Read` 权限

**Q: 如何判断用OA审批还是日志？**
- OA审批：员工在「工作台」→「OA审批」提交
- 钉钉日志：员工在「工作台」→「日志」提交

## 十、注意事项

1. **API限制**：钉钉API标准版每月1万次调用
2. **数据安全**：日报数据包含敏感信息，注意存储安全
3. **权限原则**：仅申请必要权限，遵循最小权限原则
4. **数据处理**：保证数据真实性，必须使用API调用获取，不能虚构数据
