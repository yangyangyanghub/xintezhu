## 二、各维度评分细则

### 1. 完整度（权重20%）

**评分标准**：

| 分值 | 条件 |
|------|------|
| 5分 | 工作内容+复盘+明日计划都完整填写 |
| 4分 | 工作内容完整，且有复盘或明日计划 |
| 3分 | 仅工作内容完整（字数>50） |
| 2分 | 工作内容简单（字数10-50） |
| 1分 | 工作内容缺失或敷衍（字数<10） |

**完整度判断**：
- 工作内容完整：字数 > 50字
- 有复盘：填写了"今日工作复盘"且内容不为空
- 有计划：填写了"明日重点计划"且内容不为空

**示例**：

```
✅ 5分示例：
今日完成工作：完成用户模块接口开发，编写了5个API，覆盖增删改查功能...
今日工作复盘：今天主要解决了接口鉴权的问题，通过...
明日重点计划：继续完成订单模块开发

❌ 2分示例：
今日完成工作：项目对接
（只有项目名，没有具体工作）
```

---

### 2. 内容质量（权重25%）

**评分标准**：

| 分值 | 条件 |
|------|------|
| 5分 | 内容详实，有具体工作描述+量化数据，字数>60 |
| 4分 | 内容丰富，有具体工作描述，字数>40，有多项内容 |
| 3分 | 内容一般，有基本工作描述，字数>20 |
| 2分 | 内容简单，字数10-20，可能流水账 |
| 1分 | 流水账式记录，字数<10 |

**质量检测要素**：

| 要素 | 说明 | 示例 |
|------|------|------|
| 量化数据 | 包含具体数字 | 完成5个API、处理3个项目 |
| 动作描述 | 有具体工作动词 | 完成、进行、开展、处理、修改 |
| 内容细节 | 多项内容或详细说明 | 用逗号、分号分隔多个任务 |
| 内容长度 | 总字数 | 建议>50字 |

**流水账特征**（会被标记）：

- 工作内容少于10字
- 只有项目名没有具体工作描述（如"馆陶督导"、"项目对接"）
- 没有任何动作动词
- 多行内容都缺少具体描述

**示例**：

```
✅ 5分示例：
今日完成工作：
1. 完成用户模块接口开发，编写了5个API
2. 优化了查询性能，响应时间从500ms降至100ms
3. 修复了3个线上bug，涉及登录、注册功能

✅ 4分示例：
今日完成工作：
完成东区管廊内业报告整理，包括图纸审核、数据核对、报告撰写

❌ 1分示例（流水账）：
今日完成工作：馆陶督导
（只有2个字+项目名，没有任何具体工作描述）
```

### 3. AI工具应用（权重20%）

**评分标准**：

| 分值 | 条件 | 示例 |
|------|------|------|
| 5分 | 明确使用AI解决具体问题，有详细描述 | "使用AI协助生产管理系统curd常用接口开发工作" |
| 4分 | 使用AI，有应用场景但不够具体 | "查询相关规范标准号"、"灵感渲染处理效果图" |
| 3分 | 只提到AI工具名称，没有具体用途 | "AI豆包应用"、"建筑学长灵感" |
| 2分 | 诚实填写未使用 | "无"、"暂无"、"今日未使用" |
| 1分 | 未填写或填null | - |

**AI工具关键词**：

豆包、DeepSeek、Kimi、ChatGPT、文心、通义、讯飞、智谱、建筑学长、灵感渲染、AI工具、Copilot、Cursor

**高价值应用词**（体现AI解决了具体问题）：

辅助、协助、解决、生成、开发、分析、处理、完成、查找、搜索、计算、编写、设计、优化、梳理、学习、查阅、查询、探索、转换、渲染

**工作场景词**（体现AI用在什么工作上）：

接口、代码、规范、文档、数据、报告、方案、表格、图纸、效果图、程序、系统、标准、参数、问题、资料

**示例**：

✅ 5分示例：
- 使用AI协助生产管理系统curd常用接口开发工作

✅ 4分示例：

- 查询相关规范标准号

✅ 3分示例：
- AI豆包应用

## 四、优秀日报标准

一份优秀的日报应具备以下特征：

### 1. 内容充实
- 详细描述今日工作内容
- 不只用项目名，要有具体工作描述
- 建议字数 > 50字

### 2. 有量化数据
- 包含具体数字（完成数量、百分比、时间等）
- 示例：完成5个API、处理3个项目、优化50%性能

### 3. 有AI应用
- 使用AI工具解决具体问题
- 说明用什么工具、解决了什么问题
- 避免只写工具名不写用途

### 4. 有工作复盘
- 总结今日亮点
- 记录遇到的问题
- 思考改进方向

### 5. 有明日计划
- 明确下一步工作安排
- 列出需要协调的事项

## 五、流水账日报特征

以下情况会被判定为流水账：

| 特征 | 示例 | 问题 |
|------|------|------|
| 只有项目名 | "馆陶督导" | 没说具体做了什么 |
| 字数过少 | "项目对接" | 少于10字 |
| 没有动词 | "邯郸道施工图" | 没有动作描述 |
| 多行敷衍 | "项目A\n项目B\n项目C" | 每行都只有项目名 |
