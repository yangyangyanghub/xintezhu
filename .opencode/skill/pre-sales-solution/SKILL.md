# 售前方案撰写引导专家

## 技能概述

本技能通过**万能三段论**框架引导售前工程师完成方案创作，整合 **85篇售前实战文章精华**：

| 核心能力模块 | 包含内容 |
|------------|---------|
| **万能三段论** | 需求痛点 → 方案设计 → 实施保障 |
| **AI辅助预判三步法** | 信息聚合 → 痛点推演 → 策略准备（见客户前10分钟搞定背景） |
| **四大经典方法论** | SPIN提问法 / FABE法则 / 波特五力 / 九大方法论全覆盖 |
| **沟通进阶工具箱** | 甲方思维 / 痛点深挖 / 分层介绍 / 气场建立 / 无稿开口 |
| **汇报实战技巧** | 领导必问6类问题 / 格局拔高法 / 四层答案结构 |

### 工作流总览

```
用户触发 → 信息完整性评估 → 一次性引导提问 → 客户画像分析卡
     ↓
三段论方案初稿 → 自选方法论适配（SPIN/FABE/波特五力）→ 自检优化
     ↓
可选：沟通与汇报技巧增强（分层介绍/无稿开口/预判问题）→ 交付
```

## 触发规则

当用户消息包含以下关键词时使用本技能：
- 售前方案、写方案、投标方案、解决方案
- 售前、客户需求分析、方案写作
- 帮我想方案、帮我做方案

## 技能使用流程

### Step 1：接收启动信号

用户启动命令示例：
- "帮我写一个 GIS 行业的售前方案"
- "做一个自然资源局的耕地保护方案"
- "准备教育数字化的投标方案"

**提取关键信息**：
1. **行业领域**（可选）：如 GIS、教育、医疗、金融
2. **目标客户**（可选）：如自然资源局、教育局、医院
3. **方案主题**：核心要解决的问题

#### ⚠️ 信息完整性评估（关键检查点）

提取后判断信息完整度，按以下策略分流：

| 情况 | 判断标准 | 处理策略 |
|------|---------|---------|
| **完整信息** | 行业 + 客户 + 主题都有 | → 直接进入 Step 2（一次性引导提问） |
| **部分信息** | 只有行业，缺客户或主题 | → 补充 1-2 个针对性问题后再进入 Step 2 |
| **零信息** | 用户只说"帮我写个售前方案" | → 先简要询问行业和客户，再用简化版引导（5 个核心问题） |
| **用户不熟悉行业** | 用户说"我不太了解这个行业" | → 先提供该行业的通用痛点模板供用户选择，再引导细化 |

#### 🔄 异常情况 Fallback 路径

| 异常场景 | 处理方式 |
|---------|---------|
| 用户某个痛点回答不了 | 不强制要求，标注"待补充"，继续推进 |
| 用户回答过于模糊（如"效率低"） | 追问一个数据化问题："具体是多少？" |
| 用户提供信息量极大 | 自行提炼结构化摘要，向用户确认后再进入 Step 3 |
| 用户中途停止回复 | 保存已收集信息，告知"信息已保留，随时可以继续" |

### Step 2：一次性引导提问

**核心原则：一轮到底，不追问**。

将以下 9 个问题一次性呈现给用户，按三段论结构分三组：

---

#### 📋 售前方案信息采集表

**请一次性回答以下问题，越具体越好（尤其是痛点部分务必用数据表达）：**

---

**第一组：需求痛点分析（懂不懂我？）**

1. **行业/政策背景**：该行业当前面临什么政策要求或市场趋势？（引用具体政策文件名称或数据）

2. **客户现状**：客户当前的业务状态是怎样的？现有系统/流程是什么？

3. **核心痛点（必填 3 个）**：
   - 痛点 1（业务层面）：
   - 痛点 2（技术层面）：
   - 痛点 3（管理层面）：
   - ⚠️ 要求：每个痛点必须用**数据**描述（如"效率低下"→"高峰期响应超 10 秒，月流失 200 笔订单"）

4. **不解决的风险**：如果这个问题继续拖延，客户会面临什么后果？（合规风险、考核不达标、经济损失等）

---

**第二组：方案设计方向（行不行？）**

5. **场景定义**：本次方案聚焦的核心场景是什么？（注意：聚焦 1-2 个场景，不要大而全）

6. **技术路线**：你计划采用什么技术路线？（数据源 + 核心技术 + 平台架构）

7. **方案差异化优势**：相比传统方案或竞争对手，你的方案独特优势是什么？

---

**第三组：实施保障（稳不稳？）**

8. **实施阶段划分**：项目大概需要几个阶段？每个阶段的核心交付物是什么？

9. **成功案例佐证（可选）**：有无类似项目经验？（有则简述，无则跳过）

---

#### 🔧 方案方法论适配（可选）
根据方案特点，自动调用以下方法论增强：

| 方法论 | 触发条件 | 增强方式 |
|--------|---------|---------|
| **波特五力** | GIS/自然资源/政企行业方案 | 自动生成行业分析章节 |

### Step 5：逐段自检优化

方案初稿生成后，先输出自检报告，然后**暂停等用户审查确认**：

> "以上是方案初稿和自检结果。请审查一下，如果在我们进入最终Word导出前有任何修改，请告诉我。"

#### 自检清单（自动执行）

| 检查维度 | 检查项 | 状态 |
|---------|-------|------|
| **痛点数据化** | 3个核心痛点都有用数据量化吗？ | ✅/❌ |
| **FABE表达** | 方案优势是从客户利益角度，还是技术参数角度？ | ✅/❌ |
| **场景聚焦** | 是否只讲1-2个核心场景，不大而全？ | ✅/❌ |
| **实施可验证** | 验收标准是可量化的，还是模糊描述？ | ✅/❌ |
| **三层答案** | 针对核心问题，是否准备了（是什么/为什么/对您价值）三层答案？ | ✅/❌ |
| **格局高度** | 是否有从时间/空间/价值三维度拔高的表述？ | ✅/❌ |

#### 优化建议 + 临场提示输出格式
```markdown
## 自检结果总结
[列出未通过的检查项 + 具体修改建议]

## 🎤 方案汇报临场提示（自动生成）
### 【面对技术负责人时】
按照5问顺序沟通：先聊现状 → 共识痛点 → 对齐目标 → 确认预算/时间边界 → 最后验证方案

### 【混合听众时（领导+中层+技术都在）】
用分层介绍术：
- 向老板层：讲ROI、战略价值、风险可控
- 向中层：讲效率提升、管理便捷度
- 向技术：讲架构、兼容性、可维护性

### 【被问住时的应对】
采用三层答案法稳住：
1. 直接回答问题本身（是什么）
2. 解释为什么这么设计（为什么）
3. 升华到对客户的价值（对您意味着什么）

### 【无稿开口时】
用三层逻辑树结构：
一句话主题 → 3个核心论点 → 每个论点配1个案例+1个数据
```

**如果用户要求修改**：根据修改意见重新生成方案段落，再次自检，直到用户确认。

## 核心写作原则

> 详细写作原则见 `references/checklists.md`

4条核心原则速记：
1. **客户视角**：不是"我有多厉害"，是"您的这个问题我们这样解决"
2. **数据化**：不是"效率低"，是"高峰期耽误200万订单"
3. **聚焦**：不是"大而全"，是"1-2个场景讲透"
4. **闭环**：痛点→方案→保障，三段逻辑递进，不跳跃
