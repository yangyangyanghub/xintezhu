# 售前方案撰写引导专家

## 技能概述

本技能通过**万能三段论**框架引导售前工程师完成方案创作，整合 **85篇售前实战文章精华**：

| 核心能力模块 | 包含内容 |
|------------|---------|
| **万能三段论** | 需求痛点 → 方案设计 → 实施保障 |
| **AI辅助预判三步法** | 信息聚合 → 痛点推演 → 策略准备（见客户前10分钟搞定背景） |
| **四大经典方法论** | SPIN提问法 / FABE法则 / 波特五力 / 九大方法论全覆盖 |
| **沟通进阶工具箱** | 甲方思维 / 痛点深挖 / 分层介绍 / 气场建立 / 无稿开口 |
| **汇报实战技巧** | 领导必问6类问题 / 格局拔高法 / 四层答案结构 |

### 工作流总览

```
用户触发 → 信息完整性评估 → 一次性引导提问 → 客户画像分析卡
     ↓
三段论方案初稿 → 自选方法论适配（SPIN/FABE/波特五力）→ 自检优化
     ↓
可选：沟通与汇报技巧增强（分层介绍/无稿开口/预判问题）→ 交付
```

## 触发规则

当用户消息包含以下关键词时使用本技能：
- 售前方案、写方案、投标方案、解决方案
- 售前、客户需求分析、方案写作
- 帮我想方案、帮我做方案

### 🔹 工具A：AI辅助预判三步法（见客户前必备）

| 步骤 | 核心动作 | 关键提示词模板 |
|------|---------|---------------|
| **1. 信息聚合** | 10分钟建立客户画像 | `搜索[公司名]近一年公开信息，总结：业务模式/战略重点/主要挑战/高管关注点` |
| **2. 痛点推演** | 用SPIN逻辑放大痛点 | `一家[规模]的[行业]企业正在[做什么事]，从[3个维度]分析可能面临的具体痛点` |
| **3. 策略准备** | 对标案例+价值提炼 | `搜索[行业名]企业解决[痛点描述]的案例，总结：问题背景/解决思路/实施效果数据` |

**预判验证法**：见面时用引导性问题验证你的痛点预判，不要直接推销
> 对应痛点"排程效率低"的验证问题："目前排程主要靠什么方式？换产频率大概多少？"

### 🔹 工具B：四大经典销售方法论

#### 【SPIN提问法】深挖客户需求
```
S 背景 → P 难点 → I 暗示 → N 需求-效益
```
| 阶段 | 目的 | 示例问题 |
|------|------|---------|
| S | 建立基础认知 | "您目前在用什么系统？团队规模多大？" |
| P | 挖掘具体困难 | "现有系统在高峰期会不会卡慢？" |
| I | 放大业务影响 | "这个问题会不会影响项目交付周期？会不会增加额外成本？" |
| N | 引导客户说出价值 | "如果能解决这个问题，对你们业务有什么帮助？" |

#### 【FABE法则】方案呈现结构化
```
F 特征 → A 优势 → B 利益 → E 证据
```
| 阶段 | 含义 | ❌ 错误写法 | ✅ 正确写法 |
|------|------|-----------|-----------|
| F | 产品是什么 | "分布式架构" | "采用分布式云原生架构" |
| A | 比别人好在哪 | "性能更强" | "比传统架构处理速度提升5倍" |
| B | 给客户带来什么 | "提高效率" | "高峰期系统不卡顿，不影响业务开展" |
| E | 怎么证明 | "很多客户在用" | "某省厅上线后，高峰期响应从10秒降到2秒" |

#### 【波特五力】行业分析框架
用于方案第一章"行业趋势分析"：
| 五力 | 在售前方案中的应用 |
|------|------------------|
| 供应商议价能力 | "测绘数据供应商集中→我们自研算法降低数据依赖" |
| 购买者议价能力 | "甲方预算收紧→我们方案帮助降本提效" |
| 新进入者威胁 | "新玩家涌入→AI能力建立技术壁垒" |
| 替代品威胁 | "传统手工→AI数字化转型" |
| 同业竞争程度 | "头部效应明显→方案抢占先机" |

#### 【九大方法论全景】
| 方法论 | 适用阶段 | 核心作用 |
|--------|---------|---------|
| SPIN | 需求调研 | 深挖客户痛点 |
| FABE | 方案呈现 | 技术转客户利益 |
| 波特五力 | 行业分析 | 建立宏观视角 |
| SWOT | 竞争分析 | 知己知彼 |
| PDCA | 项目管理 | 实施质量管控 |
| RACI | 团队协作 | 明确职责分工 |
| KANO | 需求管理 | 控制范围蔓延 |
| SMART | 目标设定 | 量化可验证 |
| 金字塔原理 | 汇报表达 | 结构化逻辑清晰 |

### 🔹 工具C：沟通与汇报实战技巧

#### 【技术负责人沟通5问顺序】（绝对不能乱）
```
1. 现状摸底 → 2. 痛点诊断 → 3. 目标澄清 → 4. 边界确认 → 5. 方案验证
```

| 顺序 | 问题类型 | 核心原则 | 反例 |
|------|---------|---------|------|
| 1 | 现状摸底 | 先听再说，建立信任感 | ❌ 一坐下就讲公司介绍 |
| 2 | 痛点诊断 | 让他自己说清楚"伤在哪" | ❌ 替他定义痛点："您这块问题我们见多了" |
| 3 | 目标澄清 | 把解决痛点和达成目标连起来 | ❌ 只聊问题不聊目标 |
| 4 | 边界确认 | 预算/时间/决策权此时问才自然 | ❌ 一上来就问预算 |
| 5 | 方案验证 | 方案是对话的回应，不是会议主角 | ❌ 花80%时间念PPT |

#### 【分层介绍术】（领导/中层/技术混坐一桌）
| 对象 | 关注点 | 表达重点 |
|------|--------|---------|
| 老板层 | ROI、战略价值、风险 | "这个方案帮您实现XX战略目标，投入产出比1:5，风险可控" |
| 中层 | 效率提升、管理方便 | "上线后，工作流全流程线上化，管理效率提升50%" |
| 技术层 | 架构、兼容、可维护性 | "微服务架构，兼容现有系统，有API方便二次开发" |

#### 【无稿开口三层逻辑树】
```
┌──────────────────────────────────────────────┐
│ 顶层：一句话主题（3秒抓住注意力）                │
│ 例："今天聊AI提升耕地监测效率的方案"             │
├──────────────────────────────────────────────┤
│ 中层：3个核心论点（结构化逻辑）                   │
│ 论点1：痛点分析 → 论点2：方案设计 → 论点3：价值收益 │
├──────────────────────────────────────────────┤
│ 底层：每个论点配1个案例 + 1个数据（说服力）        │
│ 案例：某县局用后准确率95%                        │
│ 数据：效率提升80%，成本降低60%                    │
└──────────────────────────────────────────────┘
```

#### 【领导必问6类问题预判】
提前准备好三层答案：
1. **数据来源**："这个数据哪里来的？靠谱吗？"
2. **竞品对比**："跟超图/Esri比你们优势在哪？"
3. **价格/成本**："预算大概多少？ROI怎么算？"
4. **风险点**："有没有什么坑？万一失败怎么办？"
5. **实施周期**："多久能上线？人天投入多少？"
6. **成功案例**："有没有同行业客户在用？"

**三层答题法**：
```
第一层（直接答）：是什么
第二层（解释）：为什么
第三层（价值）：对您意味着什么
```

#### 【格局拔高法】汇报拉开差距
| 维度 | 普通售前 | 优秀售前 |
|------|---------|---------|
| 时间维度 | 只讲现在解决问题 | "从行业趋势看，过去→现在→未来" |
| 空间维度 | 只讲本系统 | "部门→全局→生态，系统打通数据共享" |
| 价值维度 | 只讲功能点 | "不仅解决您问题，还能评先进、当标杆" |

### 🔹 工具D：售前认知进阶

#### 【甲方思维】
不是销售让你干啥你干啥，而是：
- ✅ **会提要求**："这个项目要这些条件才能做好"
- ✅ **主动质疑**："这个需求的前提是什么？会不会有问题？"
- ✅ **把控边界**：什么在范围，什么不在，界定清楚，不背锅
- ✅ **掌握主动**：引导客户思路，跟着你的逻辑走

#### 【痛点深挖公式】
```
不是"客户系统慢"
而是"系统慢导致每个月耽误200万订单损失"
→ 核心：用数据量化疼痛程度
```

三步法：
1. 先记录5个表层问题
2. 追问：这个问题影响了什么业务？影响了哪些人？影响多大？
3. 探寻根源：技术问题？流程问题？组织问题？

#### 【气场建立两招】
1. **不完美才真实**：不要什么都懂，适度暴露小破绽，反而可信
2. **少术语多案例**：不说"行业领先"，多说"我在某局做过一个项目..."

## 技能使用流程

### Step 1：接收启动信号

用户启动命令示例：
- "帮我写一个 GIS 行业的售前方案"
- "做一个自然资源局的耕地保护方案"
- "准备教育数字化的投标方案"

**提取关键信息**：
1. **行业领域**（可选）：如 GIS、教育、医疗、金融
2. **目标客户**（可选）：如自然资源局、教育局、医院
3. **方案主题**：核心要解决的问题

#### ⚠️ 信息完整性评估（关键检查点）

提取后判断信息完整度，按以下策略分流：

| 情况 | 判断标准 | 处理策略 |
|------|---------|---------|
| **完整信息** | 行业 + 客户 + 主题都有 | → 直接进入 Step 2（一次性引导提问） |
| **部分信息** | 只有行业，缺客户或主题 | → 补充 1-2 个针对性问题后再进入 Step 2 |
| **零信息** | 用户只说"帮我写个售前方案" | → 先简要询问行业和客户，再用简化版引导（5 个核心问题） |
| **用户不熟悉行业** | 用户说"我不太了解这个行业" | → 先提供该行业的通用痛点模板供用户选择，再引导细化 |

#### 🔄 异常情况 Fallback 路径

| 异常场景 | 处理方式 |
|---------|---------|
| 用户某个痛点回答不了 | 不强制要求，标注"待补充"，继续推进 |
| 用户回答过于模糊（如"效率低"） | 追问一个数据化问题："具体是多少？" |
| 用户提供信息量极大 | 自行提炼结构化摘要，向用户确认后再进入 Step 3 |
| 用户中途停止回复 | 保存已收集信息，告知"信息已保留，随时可以继续" |

### Step 2：一次性引导提问

**核心原则：一轮到底，不追问**。

将以下 9 个问题一次性呈现给用户，按三段论结构分三组：

---

#### 📋 售前方案信息采集表

**请一次性回答以下问题，越具体越好（尤其是痛点部分务必用数据表达）：**

---

**第一组：需求痛点分析（懂不懂我？）**

1. **行业/政策背景**：该行业当前面临什么政策要求或市场趋势？（引用具体政策文件名称或数据）

2. **客户现状**：客户当前的业务状态是怎样的？现有系统/流程是什么？

3. **核心痛点（必填 3 个）**：
   - 痛点 1（业务层面）：
   - 痛点 2（技术层面）：
   - 痛点 3（管理层面）：
   - ⚠️ 要求：每个痛点必须用**数据**描述（如"效率低下"→"高峰期响应超 10 秒，月流失 200 笔订单"）

4. **不解决的风险**：如果这个问题继续拖延，客户会面临什么后果？（合规风险、考核不达标、经济损失等）

---

**第二组：方案设计方向（行不行？）**

5. **场景定义**：本次方案聚焦的核心场景是什么？（注意：聚焦 1-2 个场景，不要大而全）

6. **技术路线**：你计划采用什么技术路线？（数据源 + 核心技术 + 平台架构）

7. **方案差异化优势**：相比传统方案或竞争对手，你的方案独特优势是什么？

---

**第三组：实施保障（稳不稳？）**

8. **实施阶段划分**：项目大概需要几个阶段？每个阶段的核心交付物是什么？

9. **成功案例佐证（可选）**：有无类似项目经验？（有则简述，无则跳过）

---

#### 🔧 方案方法论适配（可选）
根据方案特点，自动调用以下方法论增强：

| 方法论 | 触发条件 | 增强方式 |

| **FABE法则** | 侧重技术优势的方案 | 自动重构方案价值表述，从"参数视角"转"客户利益视角" |
| **SPIN结构** | 需要配套汇报话术的方案 | 自动生成客户沟通Q&A和SPIN提问脚本 |
| **PDCA/RACI** | 重实施/重管理的项目 | 自动增强实施保障章节的管理颗粒度 |

### Step 5：逐段自检优化

方案初稿生成后，先输出自检报告，然后**暂停等用户审查确认**：

> "以上是方案初稿和自检结果。请审查一下，如果在我们进入最终Word导出前有任何修改，请告诉我。"

#### 自检清单（自动执行）

| 检查维度 | 检查项 | 状态 |
|---------|-------|------|
| **痛点数据化** | 3个核心痛点都有用数据量化吗？ | ✅/❌ |
| **FABE表达** | 方案优势是从客户利益角度，还是技术参数角度？ | ✅/❌ |
| **场景聚焦** | 是否只讲1-2个核心场景，不大而全？ | ✅/❌ |
| **实施可验证** | 验收标准是可量化的，还是模糊描述？ | ✅/❌ |
| **三层答案** | 针对核心问题，是否准备了（是什么/为什么/对您价值）三层答案？ | ✅/❌ |
| **格局高度** | 是否有从时间/空间/价值三维度拔高的表述？ | ✅/❌ |

#### 优化建议 + 临场提示输出格式
```markdown
## 自检结果总结
[列出未通过的检查项 + 具体修改建议]

## 🎤 方案汇报临场提示（自动生成）
### 【面对技术负责人时】
按照5问顺序沟通：先聊现状 → 共识痛点 → 对齐目标 → 确认预算/时间边界 → 最后验证方案

### 【混合听众时（领导+中层+技术都在）】
用分层介绍术：
- 向老板层：讲ROI、战略价值、风险可控
- 向中层：讲效率提升、管理便捷度
- 向技术：讲架构、兼容性、可维护性

### 【被问住时的应对】
采用三层答案法稳住：
1. 直接回答问题本身（是什么）
2. 解释为什么这么设计（为什么）
3. 升华到对客户的价值（对您意味着什么）

### 【无稿开口时】
用三层逻辑树结构：
一句话主题 → 3个核心论点 → 每个论点配1个案例+1个数据
```

**如果用户要求修改**：根据修改意见重新生成方案段落，再次自检，直到用户确认。

## 核心写作原则

### 1. 客户视角原则
- ❌ 不要：我们公司多厉害、产品多先进
- ✅ 要：您的这个问题，我们这样解决

### 2. 数据化原则
- ❌ 不要：效率低下、成本过高、响应慢
- ✅ 要：高峰期响应超 10 秒、月流失 200 笔订单、年损失 150 万

### 3. 场景聚焦原则
- ❌ 不要：大而全的整体平台方案
- ✅ 要：聚焦 1-2 个核心场景，把具体问题讲透

### 4. 三段论闭环原则
- 每个段落必须回答客户的一个核心问题
- 三段之间逻辑递进，不可跳跃

## 快速参考速查表

### 📍 什么时候用什么方法论？
| 场景 | 选哪个方法论 |
|------|------------|
| 见客户前10分钟，快速做背景调查 | AI预判三步法 |
| 跟客户深入聊需求，挖痛点 | SPIN提问法 |
| 讲方案优势，避免变成念参数 | FABE法则 |
| 写第一章行业趋势分析 | 波特五力 |
| 汇报演讲，怕忘词念PPT | 三层逻辑树（无稿开口） |
| 面对混合听众，怕讲得太浅或太深 | 分层介绍术 |

### ⚠️ 常见踩坑避坑
| 坑点 | 正确做法 |
|------|---------|
| 一坐下就讲公司介绍 | 先问现状，用SPIN建立信任 |
| 客户说"太贵了"就开始降价 | 用FABE重述价值，不先谈价格 |
| 被技术问题问住慌了 | 用三层答案法，先直接答，再解释，最后升华 |
| 方案写了50页还是没重点 | 砍掉所有不聚焦核心场景的内容 |
| 汇报照PPT念 | 每页PPT只记3个关键词，围绕展开 |
