# RSS 新闻收集器

从 RSS 源收集指定日期新闻，智能评分后存储。

## 功能

- 多线程并行抓取：10线程并发，3分钟完成167个源
- 断点续传：中断后自动从上次进度继续
- 详情抓取：深入详情页获取完整内容
- 智能合并：合并相同新闻，保留详情最丰富的版本
- 自动摘要：一句话总结 + 3-5条要点
- 5分制评分：过滤低价值内容
- 智能分类：AI/遥感/无人机/测绘地理信息/政策

## 工作流程

### **第一步：收集新闻**
```bash
# 基本用法
python .opencode/skill/rss-news/scripts/collect.py --date=2026-03-10

# 自定义并发数和超时
python .opencode/skill/rss-news/scripts/collect.py --date=2026-03-10 --workers=20 --timeout=5

# 强制重新抓取所有源
python .opencode/skill/rss-news/scripts/collect.py --date=2026-03-10 --force
```

**参数说明**：
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--date` | today | 目标日期，支持 YYYY-MM-DD 或 today |
| `--workers` | 10 | 并发线程数 |
| `--timeout` | 8 | 请求超时秒数 |
| `--force` | false | 强制重新抓取，清除所有进度 |

### **第二步：评分**
```bash
python .opencode/skill/rss-news/scripts/rate.py --date=2026-03-10
```

### **第三步：存储**

输出到 `dailynews/[日期]/news.md`。

### **第五步：生成分类图片（可选）**

使用 `image-service` 技能为每个分类生成黑板报风格的图片，展示精选新闻内容。

**输出目录**：`dailynews/[日期]/images/`

#### ⚠️ 中文文字渲染规范（必读）

**问题**：部分模型（如 DashScope qwen-image）对大量中文小字渲染不稳定，易产生乱码。

**解决方案**：
1. **布局简化**：只使用大标题，避免正文小字
2. **文字清晰后缀**：prompt 末尾必须追加文字渲染要求
3. **生成后校验**：用 OCR 或视觉检查确认文字清晰

#### 文字清晰后缀（必须添加）

**每个生图 prompt 必须追加以下后缀**：

```
【文字渲染要求】
- 所有中文文字必须清晰可读，笔画完整，无模糊、无乱码、无伪文字
- 文字边缘锐利，呈现印刷级清晰度，彻底消除压缩噪点与边缘溢色
- 字体风格统一，字距适中，排版规整
- 严禁出现无法阅读的乱码字符或残缺笔画
```

#### 推荐布局模板（避免乱码）

**简化版模板**（大标题 + 新闻标题列表，无正文小字）：

```
黑板报风格，粉笔字效果，深绿色黑板背景。
顶部大标题「{分类名}」，右上角日期「YYYY-MM-DD」。

画面分为 2-4 个大方块，每个方块内用大号粉笔字写一条新闻标题：
- 方块1：「{新闻标题1}」
- 方块2：「{新闻标题2}」
- 方块3：「{新闻标题3}」
- 方块4：「{新闻标题4}」

每个方块用白色边框分隔，文字用白色粉笔效果，字号要大，清晰醒目。
底部右下角写小字「AI生成」。

【文字渲染要求】
- 所有中文文字必须清晰可读，笔画完整，无模糊、无乱码、无伪文字
- 文字边缘锐利，呈现印刷级清晰度
- 字体风格统一，排版规整
```

#### 生成后校验流程

**步骤1：视觉检查**
```bash
# 使用 look_at 工具检查图片
# 或人工查看图片确认文字清晰
```

**步骤2：OCR 校验（可选）**
```bash
cd .opencode/skill/image-service
python core/image_to_text.py -m ocr "dailynews/{日期}/images/{分类名}.png"
```

**步骤3：乱码处理**

如果发现乱码，按以下优先级处理：

| 优先级 | 方案 | 适用场景 |
|--------|------|----------|
| 1 | 重新生成，使用简化布局 | 大量正文小字乱码 |
| 2 | 图生图修复，指定清晰文字 | 局部文字乱码 |
| 3 | 换用智谱 API（GLM-Image） | 文字渲染不稳定 |

**图生图修复 prompt**：
```
执行语意级图像重构。针对图中模糊或乱码的文字区域进行修复：
1. 保持原图的版面配置、物体座标、配色风格完全不变
2. 将模糊文字修复为清晰的简体中文：{预期文字内容}
3. 文字笔画必须呈现印刷级清晰度，边缘锐利，无压缩噪点
4. 严禁产生无法阅读的伪文字或乱码
直接输出修复后的图像。
```

#### 生成命令示例

**简化版（推荐）**：
```bash
cd .opencode/skill/image-service
python core/text_to_image.py "黑板报风格，粉笔字效果，深绿色黑板背景。顶部大标题「AI」，右上角日期「2026-03-30」。画面分为四个方块：左上「杨立昆批评LLM发展」，右上「多模态论文造假检测」，左下「开源智能体中医问诊」，右下「JetBrains停止更新」。底部右下角「AI生成」。【文字渲染要求】所有中文文字必须清晰可读，笔画完整，无模糊、无乱码、无伪文字。文字边缘锐利，呈现印刷级清晰度。" -r 16:9 -o "../../dailynews/2026-03-30/images/AI.png"
```

**注意事项**：
- API 有频率限制（429错误），批量生成时需间隔 5-10 秒
- 每张图约需 30-60 秒生成
- 生成失败时等待 60 秒后重试
- **必须添加文字清晰后缀**
- **生成后必须校验文字清晰度**
- 依赖 `image-service` 技能的 API 配置
- 智谱 GLM-Image 文字渲染更稳定，DashScope qwen-image 小字易乱码
