# RSS 新闻收集器

从 RSS 源收集指定日期新闻，智能评分后存储。

## 功能

- 多线程并行抓取：10线程并发，3分钟完成167个源
- 断点续传：中断后自动从上次进度继续
- 详情抓取：深入详情页获取完整内容
- 智能合并：合并相同新闻，保留详情最丰富的版本
- 自动摘要：一句话总结 + 3-5条要点
- 5分制评分：过滤低价值内容
- 智能分类：AI/遥感/无人机/测绘地理信息/政策

## 工作流程

### **第一步：收集新闻**
```bash
# 基本用法
python .opencode/skill/rss-news/scripts/collect.py --date=2026-03-10

# 自定义并发数和超时
python .opencode/skill/rss-news/scripts/collect.py --date=2026-03-10 --workers=20 --timeout=5

# 强制重新抓取所有源
python .opencode/skill/rss-news/scripts/collect.py --date=2026-03-10 --force
```

**参数说明**：
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--date` | today | 目标日期，支持 YYYY-MM-DD 或 today |
| `--workers` | 10 | 并发线程数 |
| `--timeout` | 8 | 请求超时秒数 |
| `--force` | false | 强制重新抓取，清除所有进度 |

### **第二步：评分**
```bash
python .opencode/skill/rss-news/scripts/rate.py --date=2026-03-10
```

### **第三步：存储**

输出到 `dailynews/[日期]/news.md`。

### **第五步：生成分类图片（可选）**

使用 `image-service` 技能为每个分类生成黑板报风格的图片，展示精选新闻内容。

**输出目录**：`dailynews/[日期]/images/`

**生成命令**：
```bash
cd .opencode/skill/image-service
python core/text_to_image.py "黑板报风格，黑色背景，粉笔字效果，标题「{分类名}」，日期「YYYY-MM-DD」，精选新闻：{精选内容}，装饰性边框和{主题}图案" -r 3:4 -o dailynews/{日期}/images/{分类名}.png
```

**分类对应模板**：

| 分类 | 标题 | 主题图案 | 精选内容示例 |
|------|------|---------|-------------|
| 政策/通知 | 政策/通知 | 公文、印章 | 坐标系转换、规划纲要、招聘公告 |
| AI | AI人工智能 | 芯片、机器人 | 大模型发布、算法突破、工具更新 |
| 遥感 | 遥感 | 卫星、雷达 | 激光雷达、卫星影像、遥感模型 |
| 无人机 | 无人机 | 无人机 | 路径规划、飞行算法、应用案例 |
| 测绘地理信息 | 测绘地理信息 | 地图、测量仪 | ArcGIS、坐标系、测绘项目 |
| 其他 | 行业动态 | 火箭、齿轮 | 载人航天、数据资源、技术工具 |

**提示词模板**：
```
黑板报风格，黑色背景，粉笔字效果，标题「{分类名}」，日期「{日期}」，精选新闻：{从daily-brief.md提取该分类1-2条精选内容，保留关键数据}，装饰性边框和{主题图案}图案
```

**注意事项**：
- API 有频率限制（429错误），批量生成时需间隔 5-10 秒
- 每张图约需 30-60 秒生成
- 生成失败时等待 60 秒后重试
- 依赖 `image-service` 技能的 API 配置

| 分数 | 说明 |
|------|------|
| 5分 | 极高质量，强烈推荐 |

## 配置

- RSS源：`references/feeds.opml`（167个源）
- 分类规则：`references/categories.json`
- scripts/collect.py：收集新闻、智能合并、自动摘要
- scripts/rate.py：评分及分类
