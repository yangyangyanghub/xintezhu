## 1. 选题交集模型

空间智研社的好选题，不是单纯的 GIS 技术点，也不是泛 AI 热点。

一个好选题 = **空间技术专业性 + 读者现实痛点 + 当下时间节点**。

| 维度 | 含义 | 判断问题 |
|---|---|---|
| 空间技术专业性 | GIS、测绘、遥感、WebGIS、GeoAI、空间智能 | 这个选题是否属于「从地理信息到空间智能」主线？ |
| 读者现实痛点 | 工具不会用、数据处理慢、系统难落地、选型困难 | 读者看完是否能解决一个具体问题？ |
| 当下时间节点 | 新政策、新工具、新模型、新项目、新招投标窗口 | 现在写是否比半年后写更有价值？ |

三项都满足，优先写。满足两项，可以写。只满足一项，先改角度。

## 2. 空间智能选题地图

「从地理信息到空间智能」要体现在选题层，而不是只写在简介里。

### 四层主线

| 层级 | 说明 | 常见选题 |
|---|---|---|
| 地理信息 | 数据、格式、坐标、制图、空间分析 | Shapefile、GeoJSON、QGIS、ArcGIS、GeoPandas、PostGIS |
| 空间理解 | 对象、属性、空间关系、时空变化 | 地物语义、邻近/包含/连接/叠置、变化检测 |
| 空间推理 | 规则、模型、上下文、预测判断 | 用途管制、规划约束、风险分级、学位供需、灾害预警 |
| 空间行动 | 查询、分析、制图、预警、治理建议 | GeoAgent、自然语言 GIS、自动制图、巡检派单、报告生成 |

写作时尽量推动选题从第一层向后走一步。

普通教程也可以这样升级：

- 不只写「怎么转换坐标系」，还写「坐标系错误会怎样影响空间叠加分析」。
- 不只写「怎么用 QGIS 做缓冲区」，还写「缓冲区如何服务选址、巡检和风险识别」。
- 不只写「自然语言 GIS 怎么用」，还写「它如何把一句话拆成空间查询、过滤、分析和制图任务」。

## 3. 长期栏目方向

### 3.1 GIS 工具到空间智能体

核心问题：传统 GIS 依赖专家点菜单、写脚本，空间智能体能否规划任务、调用工具、审查结果？

可写标题：

- 《GIS 的下一步不是更复杂的软件，而是会思考的空间智能体》
- 《从 QGIS 插件到 GeoAgent：AI 如何接管空间分析工作流》
- 《为什么一个大模型做不好 GIS？空间智能体需要分层架构》

### 3.2 自然语言 GIS

核心问题：自然语言 GIS 解决的是非专业人员如何操作 GIS，但它不只是聊天入口。

可写标题：

- 《自然语言 GIS：从「问地图」到「让地图做事」》
- 《会聊天的地图只是开始，真正重要的是空间推理》
- 《GeoAI Universal Platform 实测：一句话完成空间查询靠谱吗》

### 3.3 遥感 AI 智能体

核心问题：遥感 AI 从地物识别走向任务规划、多工具协同、报告生成。

可写标题：

- 《让卫星影像开口说话：遥感智能体的真正价值》
- 《遥感 AI 的下一步：不只是看见地物，而是理解变化》
- 《从地物提取到灾害预警：遥感智能体能走多远》

### 3.4 地理基础模型 GeoFM

核心问题：GeoFM / Earth AI / TerraMind 等正在成为空间数据科学基础设施。

可写标题：

- 《地理基础模型来了：空间数据科学的 GPT 时刻》
- 《GeoFM 会成为下一代空间智能基础设施吗》
- 《为什么遥感基础模型不是普通视觉模型的换皮》

### 3.5 空间推理与可靠性

核心问题：空间智能不是会回答问题，而是能理解空间关系、业务规则和结果边界。

可写标题：

- 《空间智能真正难的地方：不是看图，而是理解空间关系》
- 《GeoAgent 到底该函数调用，还是代码生成》
- 《空间智能体怎么评测？只会答题还不够》

### 3.6 空间治理与行业落地

核心问题：空间智能最终要落到治理、规划、巡检、教育、灾害、资源管理。

可写标题：

- 《空间智能最终要落到治理决策，而不是技术展示》
- 《从学区划片到学位供需：空间数据如何进入教育决策》
- 《AI+GIS 在国土空间治理里，真正能帮上的忙是什么》

## 4. 内容类型与栏目公式

| 类型 | 公式 | 示例 |
|---|---|---|
| 技术教程 | {时间}分钟{动作}：{工具名}{技能}速成 | 30 分钟学会 Cesium 三维地球开发 |
| 开源工具 | {任务}的{语言/工具}脚本开源（附避坑指南） | 批量处理 Shapefile 的 Python 脚本开源（附坐标系避坑） |
| 工具测评 | {产品A} vs {产品B}：{维度}全面对比 | QGIS vs ArcGIS Pro：开源与商业 GIS 怎么选 |
| 案例拆解 | 我们如何用{技术}解决{业务问题} | 我们如何用 GeoPandas 梳理学区划片数据 |
| 行业快报 | {年份}年{部门}最新{政策}：{受众}必须关注的{N}个变化 | 2026 年自然资源数据新规：GIS 开发者必须关注的 3 个变化 |
| 空间智能 | 从{传统 GIS 任务}到{智能能力}：{对象/关系/变化/决策}如何被 AI 理解 | 从地图查询到空间推理：自然语言 GIS 智能体到底能做什么 |
| 方法论 | 从{经历}到{判断}：我对{主题}的{N}个理解 | 从 Skill 培训到项目落地：我对 AI 工作流的 5 个判断 |

## 5. 选题来源

选题来源要分层。外部平台负责发现趋势，学术论文负责验证深度，真实项目负责落地转化，知识库负责长期复用。

### 5.1 一手项目源

优先级最高，因为它决定文章有没有真实感。

- 数据治理、学区划片、制图脚本、系统接入、AI 培训、售前方案
- 项目会议、客户需求、日报记录、待办清单、系统故障、交付复盘
- 适合转化为案例拆解、方法论总结、技术教程

判断标准：能不能说清「真实问题 → 解决过程 → 结果变化」。

### 5.2 内部知识库源

用于沉淀长期选题，避免写一次就丢。

- `myk/技术文章/`
- `myk/wiki/`
- `myk/调研笔记/`
- `myk/项目文档/`
- `daily-report/`

适合转化为系列文章、知识库二次创作、专题栏目。

### 5.3 中文行业内容源

用于观察国内 GIS / 测绘 / AI 应用圈正在讨论什么。

- 微信公众号：GIS、测绘、自然资源、遥感、AI Agent、数字政府相关账号
- 知乎：GIS 开发、遥感算法、测绘工程、空间规划问题
- B站：QGIS、ArcGIS、Cesium、遥感解译、Python GIS 教程
- 小红书：低门槛工具使用、学生学习痛点、AI 制图玩法
- 即刻/微博：行业讨论、工具更新、AI 产品热点

判断标准：同一话题如果在多个中文平台出现，说明有传播势能。

### 5.4 海外技术社区

用于捕捉 GeoAI、Agent、开源项目和工具生态的一手信号。

- X / Twitter：GeoAI 学者、开源作者、GIS/AI 公司动态
- Reddit：r/gis、r/remotesensing、r/MachineLearning、r/LocalLLaMA、r/artificial
- Hacker News：开源工具、技术架构、开发者讨论
- GitHub Trending：GeoAI、QGIS 插件、MCP、Agent、遥感模型项目
- Product Hunt：地图工具、AI 工具、数据可视化产品

判断标准：海外社区适合发现「还没进入中文圈」的新工具、新论文、新产品。

### 5.5 学术研究源

用于支撑空间智能、GeoAgent、遥感基础模型等深度内容。

- arXiv：GeoAI、GeoAgent、Remote Sensing Agent、GIS Automation、Spatial Reasoning
- Google Scholar / Semantic Scholar：引用链、代表性学者、相关论文扩展
- Papers With Code：模型、数据集、benchmark、开源实现
- IEEE Xplore / ACM Digital Library：工程与系统论文
- ScienceDirect / Springer / Nature：综述、基础模型、空间智能理论
- ISPRS、TGIS、IJGIS、Remote Sensing of Environment、Computers & Geosciences 等 GIS/遥感期刊

判断标准：学术源负责验证「这个趋势是否真的成立」，不要只靠社媒热度下结论。

### 5.6 官方政策与行业源

用于行业快报、政策解读、项目机会类内容。

- 自然资源部、住建部、国家数据局
- 各省自然资源厅、测绘地理信息主管部门
- 中国地理信息产业协会、测绘学会等行业组织
- 政府采购网、公共资源交易平台、招投标平台
- 标准规范：OGC、ISO、国家/行业标准

判断标准：官方源负责确定政策边界和行业机会，不能只转述，要解释对 GIS 开发、测绘企业和政企项目有什么影响。

### 5.7 选题转化链路

完整链路是：

```text
外部趋势 → 学术验证 → 项目落地 → 知识沉淀 → 公众号表达
```

写作时尽量说明选题来自哪一层：

- 外部趋势发现一个方向
- 学术研究验证它不是噱头
- 项目现场证明它能落地
- 知识库沉淀可复用框架
- 公众号把它表达成读者能理解、能行动的内容

## 6. SEO/GEO 关键词策略

SEO/GEO 关键词策略的目标不是制造流量感，而是让文章被准确搜索、准确理解、准确引用。

- SEO 解决「用户搜得到」
- GEO 解决「AI 能理解、能摘录、能引用」
- 空间智研社的关键词必须服务「从地理信息到空间智能」主线

### 6.1 关键词矩阵

动笔前先列关键词矩阵。不要只列热门词，要按文章结构分层。

```markdown
主题：

主关键词：
- 用户最可能搜索的核心词

长尾关键词：
- 是什么
- 怎么做
- 有什么用
- 和什么区别
- 适合什么场景

语义相关词：
- 同领域概念
- 上下游技术
- 相关工具
- 相关业务动作

场景关键词：
- 行业
- 项目类型
- 业务流程
- 使用对象

AI 可引用句：
- 定义句
- 区别句
- 判断句
- 总结句
```

示例：

```markdown
主题：自然语言 GIS 智能体

主关键词：
- 自然语言 GIS
- GeoAgent

长尾关键词：
- 自然语言 GIS 是什么
- GeoAgent 怎么做空间分析
- 自然语言 GIS 和传统 WebGIS 有什么区别

语义相关词：
- AI GIS
- 空间智能
- 空间对象
- 空间关系
- 空间推理
- 工具编排

场景关键词：
- 国土空间规划
- 城市更新
- 自动制图
- 用地分析
- 报告生成

AI 可引用句：
- 自然语言 GIS 的价值，不是让地图多一个聊天框，而是让系统理解空间任务、拆解分析步骤，并对结果做基本审查。
```

### 6.2 标题公式

标题要同时满足「人愿意点」和「机器能识别主题」。

| 公式 | 示例 |
|---|---|
| 主关键词 + 趋势判断 | 自然语言 GIS 正在重构空间分析工作流 |
| 主关键词 + 场景价值 | AI GIS 如何帮助规划师完成用地分析和自动制图？ |
| 主关键词 + 对比关系 | 自然语言 GIS 和传统 WebGIS 到底有什么不同？ |
| 主关键词 + 方法路径 | 从 GeoAgent 到空间智能：AI GIS 工作流怎么搭起来？ |
| 工具词 + 任务结果 | 用 GeoPandas 批量处理 Shapefile：从数据清洗到自动制图 |

避免标题变成关键词清单。标题里出现 1 个主关键词 + 1 个场景/价值词通常就够了。

### 6.3 正文嵌入位置

| 位置 | 放什么 | 写法 |
|---|---|---|
| 标题 | 主关键词 | 自然出现，不堆叠 |
| 摘要/开头 | 主关键词 + 痛点词 | 100 字内说清主题、对象、问题 |
| 小标题 | 长尾关键词 | 尽量写成用户会搜索的问题 |
| 正文 | 语义相关词 | 用同主题词群替代重复主关键词 |
| 案例 | 场景关键词 | 绑定行业、项目、业务动作 |
| 结尾 | 判断句/总结句 | 方便 AI 搜索直接摘录 |

### 6.4 GEO 友好句式

AI 搜索更容易引用结构清楚、边界明确的句子。写作时优先加入这四类句子。

**定义句**：

```markdown
【关键词】是指……，它主要解决……问题，常用于……场景。
```

示例：

> GeoAgent 是一种面向地理空间任务的智能体，它能够理解自然语言问题，拆解空间分析流程，并调用 GIS 工具完成查询、分析、制图和报告生成。

**区别句**：

```markdown
【关键词 A】和【关键词 B】的区别在于……
```

示例：

> 自然语言 GIS 和传统 WebGIS 的区别在于，传统 WebGIS 依赖用户点击菜单和配置参数，而自然语言 GIS 更强调对空间意图、空间对象和空间关系的理解。

**场景句**：

```markdown
在【行业/项目场景】中，【关键词】可以用于……
```

示例：

> 在城市更新项目中，自然语言 GIS 可以用于查询低效用地、识别公共服务短板、分析交通可达性，并自动生成现状分析图和初步研判报告。

**判断句**：

```markdown
真正有价值的【关键词】不是……，而是……
```

示例：

> 真正有价值的 AI GIS 不是把聊天框接到地图上，而是让系统能够理解空间对象、空间关系和业务约束，并把分析结果转化为可执行的空间决策建议。

### 6.5 发布前关键词检查

- 标题是否自然包含主关键词？
- 开头 100 字是否出现主题、对象和痛点？
- 小标题是否覆盖 2-3 个长尾问题？
- 正文是否有同主题词群，而不是重复同一个词？
- 是否有定义句、区别句、场景句或判断句？
- 结尾是否有一句能被 AI 搜索直接摘录的总结？
- 关键词是否都和文章主题有关，没有蹭无关热点？

## 7. HKR 质检法

动笔前必须检查：

- **H (Happy)**：标题和开头有没有好奇心，读者会不会点开？
- **K (Knowledge)**：读完有没有新工具、新方法、新判断？
- **R (Resonance)**：是否戳中「这个坑我也踩过」「这个需求我也有」？

三项都满足，优先写。两项满足，可以写。一项满足，改角度。零项，放弃。

## 8. 创意案例工作法

案例要能独立成立，不只是说明一个观点。

四步结构：

1. **亮出挑战**：要解决什么 GIS/AI/项目问题？
2. **展示脑洞**：为什么想到这个工具、脚本或工作流？
3. **秀出过程**：关键步骤、截图、代码、参数、Prompt 或架构图。
4. **引爆结果**：最终节省了什么、生成了什么、解决了什么。

好的案例是「痛点 + 方法 + 结果」。如果只有方法，没有痛点和结果，文章就像说明书。

- [ ] 是否接回「从地理信息到空间智能」？
- [ ] 是否明确写给哪类读者？
- [ ] HKR 是否至少满足两项？
- [ ] 是否有真实项目、数据、工具或知识库支撑？
- [ ] 是否有明确标题公式？
- [ ] 是否能给读者一个行动建议？
