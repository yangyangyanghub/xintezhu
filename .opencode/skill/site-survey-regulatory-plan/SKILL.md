## 🔄 四阶段完整工作流

### 阶段一：调研前准备（第1天）

#### 1.1 资料索要清单生成

**使用模板**：`@path prompts/01-preparation/资料索要清单.md`

**执行步骤**：
1. 向用户确认地块基本信息：面积、性质、位置、周边情况
2. 自动生成分单位资料索要清单
3. 生成给规划局、市政各公司、村委会的对接函模板
4. 输出：《调研前准备资料清单.docx》结构

#### 1.2 分组分工与路线规划

**使用模板**：`@path prompts/01-preparation/分组分工与路线规划.md`

**执行步骤**：
1. 根据地块大小和复杂程度，推荐分组方案
2. 自动生成每日调研路线（按区域分片，不走回头路）
3. 生成每个小组的详细任务清单
4. 生成每个小组需要携带的仪器工具清单
5. 输出：《调研团队分工与每日时间表》

#### 1.3 调研表格模板生成

**使用模板**：`@path templates/空白调研表格模板.xlsx`

**生成以下表格**：
1. 用地边界实测记录表
2. 建构筑物调查表（保留/拆除分类）
3. 市政管线实测表（给水/排水/电力/燃气/弱电）
4. 道路与交通调查表
5. 访谈记录表
6. 每日问题汇总表

**要求**：所有表格留足手写空间，适合现场打印使用。

### 阶段二：外业现场调研（第2-3天，AI实时辅助）

#### 2.1 现场操作标准SOP

**每到一个测点的标准动作**：
```
1. 打开「GPS水印相机」拍照（自动叠加时间、经纬度、高程、地点、调研人）
2. 用RTK/手机测坐标，录入在线表格
3. 有需要访谈的，开「讯飞听见」录音，实时转文字
4. 发现问题，立即在表格里标记颜色：红色=严重，橙色=中等，黄色=一般
5. 离开前检查：照片拍了吗？数据录了吗？还有遗漏吗？
```

**每日晚上AI整理流程**：
```
1. 所有人把当天照片传到共享文件夹对应分类
2. AI自动：
   - 重命名所有照片（格式：编号-地点-内容-年月日.jpg）
   - 识别照片中的文字（如管线铭牌、门牌号），提取到表格
   - 汇总当天所有表格数据，自动统计
   - 生成《当日调研简报》和《第二天补测点清单》
3. 团队15分钟短会：过一遍简报，安排第二天工作
```

#### 2.2 现场问题AI辅助判断

**使用模板**：`@path prompts/02-fieldwork/现场问题判断prompt.md`

**调用方式**：用户在现场发现问题时，拍照+文字描述，AI立即返回：
1. 问题严重等级（🔴严重/🟠中等/🟡轻微）
2. 需要补充测量或调查的信息
3. 对后续规划设计可能的影响
4. 相关规范标准提示

#### 2.3 六大专业组调研要点

**【土地与权属组】**
- 红线拐点坐标实测、与土地证比对
- 现场钉桩标记、与周边地物关系拍照
- 核实飞地、插花地、代征绿地/道路
- 访谈村委会、权属人，记录争议点

**【建构筑物组】**
- 保留建筑：编号、层数、高度、结构、建成年代、质量评估
- 拆除建筑：分类统计（住宅/厂房/商业）、面积估算
- 特殊建筑：化工厂、加油站、变电站等污染/安全隐患
- 围墙、挡土墙、水井、化粪池等附属设施

**【市政管线组】**
- 给水：管道位置、管径、材质、埋深、水压、接口位置
- 排水：雨水/污水管道、管径、坡度、埋深、出水口标高
- 电力：周边变电站、高压线路、杆位、变压器
- 燃气：管道位置、管径、压力、接口、供气能力
- 弱电：电信、移动、联通、广电线路走向
- **所有管线必须现场实测坐标，严禁仅依赖竣工图**

**【交通组】**
- 周边道路等级、红线宽度、断面形式、路面材质
- 交叉口形式、信号灯配置、渠化设计
- 实测早晚高峰小时交通流量
- 地块出入口可行性分析
- 公交站点、地铁站距离与步行时间

**【自然生态组】**
- 地形标高实测（最高点、最低点、坡度变化点）
- 坡度分析图（<5%、5%-15%、15%-25%、>25%）
- 水体位置、面积、水深、水质、常水位/洪水位
- 现状植被类型、覆盖率，古树名木记录
- 确认是否涉及自然保护区、饮用水源保护区

**【人文与施工条件组】**
- 文物保护单位、历史建筑位置与保护范围
- 周边社区人口、年龄结构、公共服务设施
- 施工便道、材料堆场、弃土场选址评估
- 施工水电接驳条件
- 周边敏感点（学校、医院、居民区对噪音扬尘的敏感度）

### 阶段三：内业成果生成（第4天，AI主力输出）

#### 3.1 调研报告自动生成

**使用模板**：`@path prompts/03-officework/调研报告生成prompt.md`

**报告完整结构（必须严格遵循）**：
```
# 修建性详细规划现场调研报告

## 一、项目概况
  1.1 项目背景与调研范围
  1.2 调研时间与团队组成
  1.3 调研方法与技术路线

## 二、土地与权属现状
  2.1 用地边界实测情况（与控规红线误差说明）
  2.2 土地利用现状（分类统计面积）
  2.3 权属情况与纠纷排查
  2.4 征地拆迁量统计与分类

## 三、建构筑物现状
  3.1 保留建筑评估（质量等级、改造可行性）
  3.2 拆除建筑分类统计
  3.3 特殊建筑说明（文保、污染、安全隐患）

## 四、市政基础设施现状
  4.1 给水系统现状与容量分析
  4.2 排水系统现状（雨污分流、防洪标准）
  4.3 电力系统现状与负荷预测
  4.4 燃气与供热系统
  4.5 管线综合问题与交叉冲突点

## 五、交通系统现状
  5.1 周边道路条件分析
  5.2 地块出入口可行性评估
  5.3 公共交通情况
  5.4 停车现状与需求预判

## 六、自然生态与景观资源
  6.1 地形地貌与坡度分析
  6.2 水文水资源情况
  6.3 植被与古树名木保护
  6.4 生态敏感区评价

## 七、施工条件评估
  7.1 施工进场条件（便道、堆场、弃土场）
  7.2 施工水电接驳条件
  7.3 周边敏感点与施工限制
  7.4 气候与地质条件

## 八、特殊限制因素（一票否决项核查）
  - 机场净空限制
  - 铁路/高速公路控制线
  - 输油输气管道安全距离
  - 军事设施管理范围
  - 生态红线与环保要求

## 九、存在问题与建议（核心！）
  9.1 按严重等级分类列出所有问题
  9.2 每个问题的解决建议与责任单位
  9.3 需要进一步专项调查的事项清单

## 十、附录
  10.1 现场照片集（按类别编号）
  10.2 访谈记录摘要
  10.3 各类统计汇总表
```

**AI生成要求**：
- 所有数据用表格呈现，便于阅读
- 问题部分要有具体位置和影响分析，不要空话
- 建议部分要有可操作性，不要泛泛而谈
- 字数控制在1.5-2万字之间

#### 3.2 统计表格自动生成

**使用模板**：`@path prompts/03-officework/统计表格生成prompt.md`

**自动生成以下表格**：
1. 《拆迁建筑分类统计表》（按类型、结构分类，建议拆除时间）
2. 《保留建筑质量评估表》（质量等级、改造费用估算、使用建议）
3. 《市政管线容量校核表》（给水、排水、电力、燃气现状容量、项目需求、余量分析、扩容建议）
4. 《风险分级清单》（🔴一级风险、🟠二级风险、🟡三级风险，每个风险包含：位置、描述、影响、建议措施、责任单位）

#### 3.3 CAD现状图AI辅助绘制

**使用模板**：`@path prompts/03-officework/CAD绘图辅助prompt.md`

**AI辅助绘图要点**：
1. 将RTK测点坐标自动导入CAD
2. 自动分层：边界线、建筑、道路、给水、排水、电力、燃气、标注
3. 每个点自动标注：点号、关键参数（管径、建筑层数）
4. 同类点自动连线，形成管线或边界
5. 输出标准CAD格式，比例1:500或1:1000

## ⚠️ 常见问题与避坑指南

### 问题1：现场发现有重要管线，但竣工图上没有，怎么办？
> 立即用管线探测仪精确定位，拍照留GPS坐标，联系权属单位确认，报告中列为🔴一级风险，设计时必须避让。

### 问题2：村民阻扰调研、不配合访谈怎么办？
> 先找村委会干部陪同，不要硬来。说明调研目的是为了大家好，敏感问题暂时回避，后续通过官方渠道解决。

### 问题3：AI生成的报告太笼统，不够项目-specific？
> 喂给AI的原始数据越详细越好，不要只给摘要。把所有表格、照片描述、甚至录音转文字都喂进去。

### 问题4：CAD自动绘图错误多？
> AI画完后人工必须抽查10%的关键点坐标。CAD是辅助，最终责任人还是设计师。

### 问题5：关键节点没有甲方签字确认？
> 所有争议点、边界点、特殊情况，现场就找甲方和村委会签字确认，AI自动生成《现场确认单》，打印出来就签，不要等回来补。

## 🎯 使用提示

1. **不要跳过任何阶段**：四阶段流程是经验总结，每一步都有其必要性
2. **原始数据越详细越好**：AI生成质量取决于输入数据的质量
3. **人工审核必须做**：AI是工具，最终质量责任在人
4. **照片命名必须规范**：这是可追溯性的核心保证
5. **风险分级是关键**：甲方最关心的就是问题和风险，这部分一定要做扎实

**记住：我们的目标不是"完成调研"，而是"发现所有潜在风险，给后续设计和施工扫清地雷"。**
