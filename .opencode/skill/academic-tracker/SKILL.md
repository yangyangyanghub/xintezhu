# 学术追踪器搭建技能

## 概述

本技能指导用户使用 AI 编程平台（推荐 Coze/扣子）搭建一个自动化学术追踪服务，流程为：
**RSS 抓取 → AI 相关性分析 → 报告生成 → 邮件/消息推送**

核心价值：从海量文献中自动筛选与研究方向强相关的文章，每日推送简报，手机即可查看。

## 搭建流程（4 步）

### 第一步：明确研究方向与信源

首先询问用户：

1. **你的核心研究方向是什么？**（越具体越好，例如"人工智能伦理中的价值观风控"或"遥感 AI 与地理信息结合"）
2. **你想追踪哪些期刊/信息源？**（如果用户不确定，提供下方「预置信源参考」）
3. **你希望多久收到一次推送？**（默认 12 小时/每日）
4. **推送到哪里？**（邮件/飞书/企业微信）

### 第二步：生成 Coze Prompt

根据用户输入，使用以下模板生成 Coze 工作流 Prompt，复制进去后 AI 编程平台会自动搭建工作流：

```
我想做一个这样的学术追踪服务给自己用。

整体流程：
1. 我需要通过 RSS 订阅方式，抓取学术期刊/信息源的最新文章标题、摘要、链接。
2. 利用 AI 对抓取的文章进行深度分析，找出与我的研究方向强相关的文献，并打分。
3. 将每日/每半日报使用邮件发送给我。

我的研究方向描述："[用户的研究方向描述，例如：我主要关注地理信息系统（GIS）、空间智能（Spatial Intelligence）、遥感 AI、低空经济与无人机巡检、测绘数字化转型等方向。重点关注 AI 在遥感图像解译中的应用、空间智能体、大模型与 GIS 结合、自然资源信息化与数字化转型等。]"

我的 RSS 列表如下：
[用户选择的 RSS 源，每行一个]

核心节点：
1. **RSS 抓取节点**
   - 输入：RSS 订阅源 URL 列表、研究方向提示词
   - 功能：定时从期刊/信息源 RSS 抓取最新文章标题、摘要、链接、作者、发表日期
   - 输出：抓取的文章列表

2. **内容解析与存储节点**
   - 输入：文章列表
   - 功能：解析文章摘要，可选择下载全文，存入数据库
   - 输出：结构化的文章数据

3. **AI 相关性分析节点**（Agent 节点）
   - 输入：文章列表 + 研究方向描述
   - 功能：分析每篇文章与研究方向的相关度，打分（1-10 分），筛选高于阈值（默认 7 分）的文献
   - 输出：按相关性排序的文献列表，附带分析说明

4. **报告生成节点**（Agent 节点）
   - 输入：筛选后的文献列表
   - 功能：生成每日/每半日分析报告，总结新发现、热点趋势、与用户研究方向的具体关联
   - 输出：Markdown 格式报告

5. **推送节点**
   - 输入：报告内容
   - 功能：通过邮件/消息推送报告，设定推送频率，重复文献不推送
   - 输出：推送结果
```

### 第三步：指导用户完成部署

按照以下顺序引导用户操作：

1. **注册平台**：前往 [Coze](https://www.coze.cn/) 注册（约 2 分钟）
2. **创建项目**：左侧点击"扣子编程"→ 新建项目 → 选择"工作流"
3. **粘贴 Prompt**：将上面生成的 Prompt 复制进去
4. **等待生成**：AI 会自动搭建工作流并测试通过（3-5 分钟）
5. **部署**：点击右上角"部署"→"开始部署"（2 分钟）
6. **配置推送**：
   - **邮件推送**：配置 SMTP 发件邮箱（如 163 邮箱授权码）和收件邮箱
   - **飞书/企微**：配置 webhook 或集成相应插件
7. **输入研究方向**：告诉 AI 你的研究方向描述，帮助它理解筛选标准
8. **完成**：等邮件/消息即可

### 第四步：优化建议

跑通基础版后，推荐用户逐步升级：

| 优化方向 | 说明 |
|---------|------|
| 增加信源 | 从 10 个扩展到 30-50 个期刊/源 |
| 全文分析 | 让 AI 自动下载 PDF 做深度分析 |
| 多方向追踪 | 创建副本，分别追踪不同研究方向 |
| 长时记忆 | 先用 OpenClaw 建长时记忆 Agent 读几千篇文献，再做深度简报 |
| 交叉评审 | 使用不同 AI 模型进行交叉评审推荐 |

## 预置信源参考

当用户不知道用什么信源时，主动提供以下预置清单。完整清单见 `references/{domain}.md`。

### 按领域快速选择

| 领域 | 参考文件 |
|------|---------|
| 地理信息/测绘/遥感/AI | `references/geospatial.md` |
| AI/计算机/大模型（通用） | 使用 arXiv + Google Scholar + 顶会 |
| 用户自定义领域 | 引导用户按下方「添加信源方法」自行收集 |

### 添加信源通用方法

1. **知网中文期刊**：访问 https://navi.cnki.net/knavi/journals/index → 筛选期刊 → 点击期刊介绍右上角"RSS 订阅" → 复制地址
2. **arXiv**：https://export.arxiv.org/rss/{分类}，如 cs.CV=计算机视觉，cs.AI=人工智能
3. **Springer**：https://link.springer.com/search.rss?query=关键词&sortBy=relevance
4. **Taylor & Francis**：https://www.tandfonline.com/feed/rss/期刊代码

## 故障排查指南

| 问题 | 解决方案 |
|------|---------|
| 抓取不到足够文献 | 扩大抓取数量到 1000，下调关联度要求到 5-6 分 |
| 邮件排版有问题 | 告诉 AI"手机上看邮件排版有点问题，请按作者、摘要、链接、核心问题的格式排版" |
| Springe RSS 地址报错 | 某些 RSS 地址被屏蔽，但交给 Coze 处理通常没问题 |
| 工作流报错中止 | 在聊天框打"?"让 AI 自检，或暂停后说"请检查可能的错误" |
| 需要加附件 | 告诉 AI"把每天的简报同时整理一个 md 文件作为邮件附件发送" |
