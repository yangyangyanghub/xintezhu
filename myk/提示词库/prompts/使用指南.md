# 提示词库使用指南

## 快速开始

### 1. 添加新提示词

在 `myk/提示词库/prompts/` 目录下创建新的 Markdown 文件：

```markdown
---
title: 提示词名称
category: 分类名称
description: 简短描述
tags: [标签1, 标签2, 标签3]
cover: myk/提示词库/assets/封面图.png
---

# 提示词标题

## 提示词内容

```
你的提示词内容
```

## 参数建议
...
```

### 2. 必填字段

| 字段 | 说明 | 示例 |
|------|------|------|
| `title` | 提示词名称 | 极简海报设计 |
| `category` | 分类 | 海报设计/图标设计/信息图 |
| `description` | 简短描述 | 生成简洁大气的商业海报 |
| `tags` | 标签数组 | [海报, 极简, 商业] |
| `cover` | 封面图路径 | myk/提示词库/assets/xxx.png |

### 3. 可选字段

- `style`: 风格描述
- `use_case`: 适用场景
- `model`: 推荐模型
- `version`: 版本号

## 分类建议

当前分类体系：

- **海报设计** - 各类海报设计提示词
- **图标设计** - App图标、功能图标
- **信息图** - 数据可视化、流程图
- **插画** - 艺术插画、角色设计
- **摄影** - 摄影风格、构图参考
- **代码** - 编程相关提示词
- **文案** - 营销文案、创意写作

## 封面图规范

1. **尺寸**：建议 3:4 或 1:1 比例
2. **格式**：PNG 或 JPG
3. **命名**：`{提示词名称}_cover.png`
4. **存放**：`myk/提示词库/assets/`

## 最佳实践

### 提示词编写

1. **结构化**：使用清晰的段落和标记
2. **参数化**：用 `[变量]` 标记可替换内容
3. **示例化**：提供具体示例帮助理解
4. **版本化**：重大更新时更新版本号

### 标签使用

- 使用 3-5 个相关标签
- 标签要具体（如用 "极简" 而非 "设计"）
- 保持标签一致性

### 定期维护

- 每月检查过时提示词
- 更新效果不佳的提示词
- 收集使用反馈
