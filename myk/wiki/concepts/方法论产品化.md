# 方法论产品化

> 知识从「可读」走向「可执行」：书籍、经验、咨询框架被拆分为可调用、可分阶段、可组合、可复用的技能系统。

---

## 定义

方法论产品化是指将专家经验、书籍内容、咨询框架等知识形态转换为结构化、可被 AI Agent 理解和执行的技能（Skills）的过程。

与传统知识载体（书籍、文章、课程、一次咨询）相比，产品化的方法论具备以下特征：
- **可调用**：通过 `/skill-name` 命令即时激活
- **可分阶段**：按步骤拆解，不一次性倾倒
- **可组合**：多个 skill 串联形成完整工作流
- **可复用**：同一方法论可反复应用于不同场景

---

## 演进路径

```
书籍/经验 → 博客/文章 → Prompt → Skill 模块 → Agent 操作系统
  (可读)    (可搜索)   (可用)   (可组合)     (可自治)
```

---

## 典型样本：slavingia/skills

Sahil Lavingia 将自己的书《The Minimalist Entrepreneur》从传统出版物转换为 10 个可执行的认知型技能模块：

**传统形态**：书只能从头到尾读，读者自行提取应用。
**产品化形态**：每个创业阶段对应一个 skill，Agent 按当前阶段自动加载相应决策框架。

| 阶段 | Skill | 传统书对应章节 |
|------|-------|---------------|
| 找社区 | find-community | 第2章：选对社区 |
| 验证需求 | validate-idea | 第3章：验证想法 |
| 做MVP | mvp | 第4章：构建最小产品 |
| 第一批客户 | first-customers | 第6章：获得首批客户 |
| 定价 | pricing | 第7章：定价策略 |
| 持续增长 | grow-sustainably | 第9章：可持续增长 |

---

## 技能结构模板

每个产品化的方法论 skill 通常包含四层：

```yaml
1. 哲学原则：你的世界观是什么？
2. 决策框架：遇到 X 时怎么判断？
3. 行为约束：哪些事绝对不要做？
4. 输出契约：最终给用户什么结果？
```

## 市场信号

- **slavingia/skills 热度**：创建不足一个月 8,000+ stars，817 forks
- **社区扩展**：已出现 build-in-public、delegate 等新 skill 提案
- **竞品涌现**：其他专家/书籍开始以相同模式产品化自己的方法论

---

## 与认知型技能的关系

方法论产品化是**手段**，认知型技能是**产物形态**。
通过产品化过程，方法论变成了 Agent 可调用的认知层模块。
