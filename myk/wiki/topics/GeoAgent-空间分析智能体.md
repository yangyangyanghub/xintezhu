# GeoAgent 空间分析智能体

> 面向自主空间分析的基于大语言模型的分层多智能体架构

---

## 核心定位

GeoAgent 是一个**自主空间分析系统**，通过模拟人类专家团队的协作模式，实现了从"工具的有序调用"到"具备自我验证与纠错机制的多智能体协同"的范式跃迁。

**核心价值**：
- 将任务规划、流程执行与结果审查统一纳入自主流程
- 有效克服长逻辑链条下的错误累积问题
- 生成更符合真实地理分析逻辑的可靠结果

## 架构设计：三层分层结构

```
┌─────────────────────────────────────────┐
│           规划层 (Planning Layer)         │
│  规划者 → 解析指令、评估难度、拆解子任务     │
├─────────────────────────────────────────┤
│           执行层 (Execution Layer)        │
│  管理者 → 协调分析师、制图师、报告员        │
├─────────────────────────────────────────┤
│           审查层 (Review Layer)           │
│  分析审查员 + 图像审查员 + 报告审查员       │
│  → 自主验证、纠错、闭环反馈                │
└─────────────────────────────────────────┘
```

### 规划层（Planning Layer）

- **核心角色**：规划者智能体
- **职责**：解析自然语言指令与空间数据，评估任务难度与类别，拆解子任务，生成标准化执行计划
- **价值**：避免执行层陷入逻辑混乱，确保步骤有序

### 执行层（Execution Layer）

- **核心角色**：管理者智能体
- **协调对象**：分析师、制图师、报告员三类专业执行者
- **工作流**：数据理解→代码生成→代码检查→代码调试（四步混合节点）

### 审查层（Review Layer）

- **核心角色**：分析审查员、图像审查员、报告审查员
- **职责**：
  - 分析审查员：审查代码逻辑
  - 图像审查员：借助多模态 LLM 审查制图规范
  - 报告审查员：审查报告格式与逻辑
- **机制**：若结果不达标，生成修改反馈并触发修订，形成闭环自我验证循环

## 工具设计：三大核心工具集

### 系统交互工具

- 文件读写、工作区目录列表、代码执行
- 连接 LLM 与外部计算环境的基础桥梁

### 数据理解工具（核心创新）

- **理念**：先理解数据再进行推理
- **功能**：主动提取空间范围、坐标参考系统、属性分布特征
- **价值**：从"被动生成"转向"主动决策"

### 领域知识检索工具

- **技术**：检索增强生成（RAG）
- **功能**：获取 GeoPandas、Rasterio 等库的最新函数定义与参数规范
- **价值**：解决大模型地理专业知识时效滞后问题

## 实验结果

### 测试集

- 147 项真实空间分析任务
- 分类：85 基础 + 39 中级 + 23 高级

### 性能对比

| 配置 | 整体成功率 | 高级任务成功率 |
|------|-----------|--------------|
| **GeoAgent (DeepSeek-V3)** | **95.24%** | **95.65%** |
| GeoAgent (GPT-4o) | 94.56% | - |
| GeoJsonAgent (多智能体基线) | 81.63% | - |
| GPT-4o (单智能体) | 70.75% | 47.83% |

### 消融实验（层级贡献）

| 配置 | 成功率 |
|------|--------|
| 完整框架 (GPT-4o) | 94.56% |
| 移除规划层 | 59.18% |
| 移除审查层 | 69.39% |
| 仅执行层 | - |

**结论**：规划层和审查层缺一不可

### 资源消耗

- 执行层构成资源消耗主体（符合核心处理单元定位）
- 规划层和审查层开销明显更低，但发挥关键作用，具有高"性价比"

## 应用案例：城市设施选址

**任务**：多约束城市垃圾填埋场选址

**子任务拆解**：
1. 数据预处理与验证
2. 土地利用数据更新
3. 多准则选址分析（5 项空间约束）
4. 人口影响评估

**空间约束**：
- 距道路 ≤ 200 米
- 距河流 ≥ 150 米
- 距居民区 800-1500 米
- 位于未分配用地上
- 设施间距 > 2000 米

**结果**：精准筛选出两个最优选址，并自动计算影响范围内总人口数量

## 核心创新点

1. **分层多智能体协作范式** — 模拟人类专家团队的"分工"与"评审"机制
2. **自我验证机制** — 审查层实现自主质量控制，防止错误传播
3. **环境感知工具集** — 数据理解+领域知识检索，实现"先理解后生成"
4. **消融实验验证** — 深度揭示层级组件在不同任务复杂度下的核心价值

## 技术启示

1. **长逻辑链条任务**需要分层规划，避免错误累积
2. **质量控制**不能依赖单一执行者，需要独立审查层
3. **领域知识**需要通过 RAG 动态获取，而非静态硬编码
4. **数据理解**是空间分析的前提，"先理解后生成"模式更可靠
