# AI 一人公司模式

> 2026 年最热的 AI 创业范式：一个人 + N 个 AI Agent = 运营一家完整公司。

---

## 核心逻辑

| 传统公司 | AI 一人公司 |
|---------|------------|
| 招聘人类员工 | 配置 AI Agent 角色 |
| 管理团队协作 | Agent 间通过协议协作 |
| 员工培训 + 离职风险 | Skill 即能力沉淀，零离职成本 |
| 朝九晚五 | 7×24 小时无休 |
| 人力成本递增 | 边际成本趋近于 Token 费用 |

## 方法论维度：slavingia/skills 创业操作系统

> 2026-04-19 新增：从 Agent 蜂群协作到创业决策框架

### slavingia/skills 项目

Gumroad 创始人 Sahil Lavingia 将自己的创业方法论《The Minimalist Entrepreneur》拆分为 10 个可执行 AI 技能模块，创建不足一个月突破 8,000 stars。

**核心路径**：Community -> Validate -> Build -> Processize -> Sell -> Price -> Market -> Grow -> Culture -> Review

**统一设计模式**：每个 SKILL.md 采用「原则 -> 框架 -> 约束 -> 输出」四层结构：
- find-community：先找社区，不先找 idea
- validate-idea：验证靠 selling，不靠 building
- mvp：Build as little as possible
- processize：Processize before you productize（最高辨识度模块，把抽象产品思维落成 SOP）
- pricing：Charge something. Always.
- grow-sustainably：Profitability is a superpower
- minimalist-review：全书方法论变成通用决策审查器

**生态定位**：位于技能生态上层（认知/策略型），解决「哪些活值得干、按什么顺序干」，与 OpenCode Skills（执行型）互补而非竞争。

### 一人公司成功四要素

1. **痛点明确**：高频、标准化、可交付
2. **分发能力**：被看到 > 被做到（AI 降低生产门槛但未降低分发门槛）
3. **信任建立**：direct-to-fan、build in public、email owned land
4. **护城河设计**：避免只做模型壳产品，防范模型厂商能力吞并

## Skill 组合清单（创业必备）

### 基础能力层
1. **写作 Skill**：花叔 20 个 AI 创作 Skills
2. **图表 Skill**：5 个图表 Skill，覆盖所有场景
3. **数据分析 Skill**：流程封装为 AI Skill
4. **网页抓取 Skill**：Web Access 拉满联网能力

### 扩展能力层
5. **原型设计 Skill**：3 个效果爆炸的原型 Skill
6. **UI 优化 Skill**：嫌 AI 界面太丑？装上开源插件秒变设计师
7. **API 文档 Skill**：AI 帮忙设计 API 文档
8. **前端设计 Skill**：照着 Apple/Stripe 的 Markdown 做界面

### 自动化层
9. **记忆管理**：claude-mem 自动记忆系统
10. **代码生成**：OpenSpec/GStack Spec 驱动发
11. **测试自动化**：Apifox CLI + Claude Skills

## 风险与挑战

| 风险 | 说明 | 缓解策略 |
|------|------|---------|
| Agent 一致性 | AI 可能"偷懒" | Google 专 Agent Skills 专治 AI 偷懒 |
