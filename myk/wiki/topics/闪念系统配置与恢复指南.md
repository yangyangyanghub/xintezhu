# ⚡ 闪念系统 (Flash Thought System)

> **文档用途**：用于在系统损坏、换新设备或重装插件后，快速恢复"碎片灵感记录系统"（类 flomo）的完整功能。

---

## ️ 系统架构

**核心逻辑**：`灵感` → `快速捕获` → `暂存 (Inbox)` → `定期回顾` → `沉淀到 Wiki`


```
[Alt+N] 弹窗输入 → 自动生成文件 → myk/闪念/inbox/
                                       ↓
                                   每周回顾
                                       ↓
                         ┌─────────────┴──────────────┐
                         ↓                            ↓
                   有价值 → 沉淀到 Wiki           无价值 → 删除
                   移至 processed/
```

---

## 📦 依赖插件

必须安装以下两个 Obsidian 插件：

| 插件名称 | 用途 | 关键设置 |
|---------|------|----------|
| **Templater** | 执行 JS 脚本，调用弹窗系统 | 指定 Scripts Folder |
| **Dataview** | 自动生成看板、统计表 | 默认设置即可 |

---

## ️ 恢复步骤

### 1. 创建目录结构

在 Obsidian Vault 根目录下创建以下文件夹（不存在时）：

```text
00 templates/
  └── Templater/       # 存放 JS 脚本
myk/
  └── 闪念/
      ├── inbox/       # 待处理
      └── processed/   # 已沉淀
```

### 2. 配置 Templater

打开 `设置 -> Templater`，确保以下两项**都已填写**且指向正确的文件夹：

⚠️ **注意**：两个设置项必须都包含 `Templater` 文件夹路径。

| 设置项 (英文) | 推荐值 (中文) | 必填? |
|---------------|--------------|-------|
| **Template folder location** | `00 templates/Templater` | ✅ 是 |

### 3. 创建核心文件

将以下代码复制到对应路径。

#### A. 脚本：AHA 闪念捕获.js
**路径**：`00 templates/Templater/AHA 闪念捕获.js`
**作用**：Alt+N 的逻辑核心，弹窗输入 -> 写入内容 -> 自动打标签。

```javascript
/**
 * AHA 闪念捕获 - 用户脚本
 * 弹窗输入灵感 → 自动创建闪念卡片到 inbox
 */

// ⚠️ 注意：参数必须直接接收 tp，不能包裹在 params 中
module.exports = async (tp) => {
  // 获取今天时间
  const today = tp.date.now("YYYY-MM-DD");
  const nowTime = tp.date.now("HH-mm-ss");
  const nowDisplay = tp.date.now("HH:mm");
  
  // 1. 弹窗输入闪念内容
  const content = await tp.system.prompt(" 记录一闪而过的灵感：");
  
  if (!content || content.trim() === "") {
    return;
  }
  
  // 2. 询问标签（可选，默认 #flash）
  const tags = await tp.system.prompt("🏷️ 标签（可选，空格分隔）", "#flash") || "#flash";
  
  // 3. 创建文件路径
  const fileName = `myk/闪念/inbox/${today}-${nowTime}`;
  
  // 4. 生成文件内容
  const fileContent = `---
created: ${today} ${nowDisplay}
tags:
${tags.split(" ").map(t => `  - ${t.startsWith("#") ? t : "#" + t}`).join("\n")}
type: flash
status: inbox
---

## 💡 闪念

${content}

---
## 📝 关联
- 来源：
- 关联笔记：

---
## 🔄 批注（回顾时追加）
- ${today}：
`;
  
  // 5. 创建新文件并打开
  const file = await tp.file.create_new(fileContent, fileName, true);
  
  new Notice("✅ 闪念已保存！");
};
```

## 🐛 故障排查 (Troubleshooting)

### 1. 报错：Template parsing error / Cannot read properties of undefined (reading 'date')
* **原因**：JS 脚本接收参数方式错误。旧版本脚本使用 `module.exports = async (params)`，新版本 Templater 直接传 `tp`。
* **修复**：确保 `AHA 闪念捕获.js` 的函数签名是 `module.exports = async (tp) => { ... }`。参考上方脚本代码。

### 2. 报错：User Scripts folder not set
* **原因**：Templater 设置中未指定脚本文件夹。
* **修复**：进入 Templater 设置，填好 `User Scripts folder` (通常是 `00 templates/Templater`)。
* **注意**：设置后可能需要 **重启 Obsidian** 才能生效。

### 3. 弹窗不出现，只创建空文件
* **原因**：快捷键绑定到了 QuickAdd 的 Capture 模式，而不是 Templater 模板。
* **修复**：检查快捷键是否绑定到了 `Templater_AHA_闪念.md` 模板。

### 4. Dataview 不显示内容
* **原因**：
  1. 标签或字段名写错了 (必须是 `type: flash` 和 `status: inbox`)。
  2. Dataview 插件被禁用。
  3. 索引未刷新 (设置里点 "Force refresh")。
* **修复**：检查 Frontmatter 格式是否正确。

## 💡 进阶优化建议

* **Git 备份**：这些模板和脚本文件都应纳入 Git 版本控制（排除隐私文件即可）。
* **定期清理**：每周日回顾 `inbox`，把有价值的移动到 `processed` 并更新 `status: processed`，没用的定期删除。
* **双链关联**：在闪念的"关联笔记"部分多用 `[[双链]]` 连接到 Wiki 的其他知识点，形成知识网络。
