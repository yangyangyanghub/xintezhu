# AI 记忆系统全景对比

> AI Agent 的记忆能力决定了是否真"智能"——从"对话结束即遗忘"到"跨会话持续记忆"的进化之路。

---

## 一句话决策
**核心方案**：Claude-mem（自动记忆系统）+ ChatMem（会话记忆管理）+ 本地化 Wechat-Cli（微信记录导入 LLM Wiki）。

---

## 问题定义

面试官经典追问：
> "你的 Agent 说有长期记忆，用户上周问的问题这周还记得，这个记忆存在哪？怎么检索？什么时候更新，什么时候删除？"

这是 2026 年 AI Agent 领域最核心的技术挑战之一。

---

## Claude Code 的记忆系统现状

**真相**：Claude Code 的记忆系统"比想象中初级"：
- ✅ 基础会话内上下文保持
- ❌ 无跨会话持久化
- ❌ 需要手动记录上下文
- ❌ 无法自动更新/删除

**解决方案**：依赖外部记忆插件（如 claude-mem）补充能力。

---

## 记忆系统架构设计

```
┌─────────────────────────────────────────┐
│           记忆管理层                    │
├──────────┬──────────┬──────────────────┤
│ 短期记忆  │ 中期记忆  │ 长期记忆          │
│ (会话级) │ (项目级) │ (Agent 级)       │
├──────────┼──────────┼──────────────────┤
│ 上下文   │ 项目摘要  │ 技能/经验         │
│ 对话历史 │ 决策记录  │ 错误模式          │
│ 临时变量 │ 代码结构  │ 偏好设置          │
└──────────┴──────────┴──────────────────┘

存储方案：
- 短期：内存（会话结束即释放）
- 中期：本地文件（.md / .json）
- 长期：向量数据库（pgvector / Chroma）
```

## 记忆检索与更新策略

| 策略 | 说明 |
|------|------|
| BM25 检索 | 字典 + N-gram 分词，Title×4 超级加权 |
| 向量检索 | Embedding 语义相似度搜索 |
| 时间衰减 | 越久远的记忆权重越低 |
| 自动清理 | 过期标记 + 手动整理 + AI 自动归档 |

---

## 实践建议

1. **不要过度依赖 AI 记忆**：Claude Code 的记忆系统还很初级
2. **手动记录关键上下文**：重要决策和代码变更必须人工标注
3. **使用外部记忆插件**：claude-mem 等工具可显著提升跨会话体验
4. **本地化敏感数据**：微信聊天记录等隐私信息用 Wechat-Cli 本地处理

## 与 [[Agent-as-Teammate]] 的关系

记忆是 Agent 作为"团队成员"的**基础能力**：
- 没有记忆 → Agent 每次都是陌生人
- 有记忆 → Agent 能记住上次协作的上下文，真正成为"同事"
