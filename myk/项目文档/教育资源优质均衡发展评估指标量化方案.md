## 一、背景与目标

## 二、不可量化指标的四类量化方法

### 2.1 制度建设类 → 二元判定 + 完成度百分比

**适用指标**：

|  序号   | 指标名称                       | 教育阶段 |
| :---: | -------------------------- | :--: |
| 学前-05 | 制定幼儿园布局规划                  | 学前教育 |
| 学前-06 | 把普惠性幼儿园建设纳入城乡公共管理设施统一规划    | 学前教育 |
| 学前-21 | 落实幼儿园基本信息备案及公示制度           | 学前教育 |
| 学前-22 | 建立3—5年一轮覆盖所有幼儿园的办园行为督导评估制度 | 学前教育 |
| 义务-08 | 县域内义务教育学校规划布局合理，符合国家规定要求   | 义务教育 |
| 义务-25 | 所有学校制定章程，实现学校管理与教学信息化      | 义务教育 |

**量化方案**：

```
量化公式：完成度 = 已完成项数 / 应完成项数 × 100%

判定标准：
  - 100% = 已制定并正式印发（满分）
  - 1-99% = 草案/征求意见中（按比例得分）
  - 0% = 未启动（0分）
```

**系统实现**：

| 数据字段   | 字段类型 | 说明                    |
| ------ | ---- | --------------------- |
| 规划文件名称 | 文本   | 正式文件全称                |
| 文号     | 文本   | 如"XX教发〔2024〕X号"       |
| 发布日期   | 日期   | 文件正式印发日期              |
| 当前状态   | 枚举   | 已印发 / 草案 / 征求意见 / 未启动 |
| 覆盖范围   | 百分比  | 文件适用的园所/学校覆盖率         |

**计算逻辑**：
```
完成度 = (已完成数 + 部分完成数 × 0.5) / 总数 × 100%
```

**政策依据**：《教育质量监测与评估指标调研报告》中，国家义务教育质量监测采用"校长问卷+督导评估"对制度落实情况进行标准化采集。

### 2.2 政策落实类 → 覆盖率/达标率计算

**适用指标**：

|  序号   | 指标名称                                               | 教育阶段 |
| :---: | -------------------------------------------------- | :--: |
| 学前-09 | 小区配套园办成公办园或普惠性民办园且运转良好                             | 学前教育 |
| 学前-10 | 落实省定公办园生均财政拨款标准或生均公用经费标准                           | 学前教育 |
| 学前-11 | 落实企事业单位、部队、高校、街道、村集体办幼儿园财政补助政策                     | 学前教育 |
| 学前-12 | 落实省定普惠性民办园认定标准、补助标准及扶持政策                           | 学前教育 |
| 学前-13 | 落实公办园收费标准和普惠性民办园收费办法                               | 学前教育 |
| 学前-16 | 落实公办园教师工资待遇保障政策，确保教师工资及时足额发放、同工同酬                  | 学前教育 |
| 学前-17 | 参照公办园教师工资收入水平，合理确定民办园相应教师工资收入水平                    | 学前教育 |
| 义务-09 | 县域内城乡义务教育学校建设标准统一、教师编制标准统一、生均公用经费基准定额统一、基本装备配置标准统一 | 义务教育 |
| 义务-15 | 全县义务教育学校教师平均工资收入水平不低于当地公务员平均工资收入水平                 | 义务教育 |
| 义务-27 | 教师能熟练运用信息化手段组织教学，设施设备利用率达到较高水平                     | 义务教育 |

**量化方案**：

```
量化公式：达标率 = 落实政策的学校(园)数 / 辖区学校(园)总数 × 100%

判定标准：
  - ≥95% = 基本落实（满分）
  - 85-94% = 部分落实（80分）
  - <85% = 落实不到位（按比例得分）
```

**系统实现**：

| 数据表        | 字段设计                        |
| ---------- | --------------------------- |
| 园所/学校基础信息表 | 机构编码、机构名称、机构类型、所属区域         |
| 政策落实对照表    | 机构编码、政策编号、是否落实(是/否)、落实日期、备注 |
| 经费拨付明细表    | 机构编码、拨付年度、拨付金额、拨付标准、拨付状态    |
| 教师工资台账     | 机构编码、教师人数、平均工资、对比基准、差额比例    |

**计算逻辑**：
```
达标率 = 已落实机构数 / 总机构数 × 100%
得分 = if(达标率≥95%, 100, if(达标率≥85%, 80, 达标率/85*80))
```

**政策依据**：[[义务教育优质均衡发展评估指标调研报告]] 明确采用"校校达标"规则，系统可复用此逻辑。

### 2.3 质量水平类 → 评分量表 + 第三方评估

**适用指标**：

|  序号   | 指标名称                                       | 教育阶段 | 量化方法      |
| :---: | ------------------------------------------ | :--: | --------- |
| 学前-26 | 幼儿园园舍条件、玩教具和幼儿图书配备普遍达到规定要求                 | 学前教育 | 达标率       |
| 学前-27 | 2017年后规划设计的幼儿园符合《幼儿园建设标准》                  | 学前教育 | 二元判定      |
| 学前-29 | 按《幼儿园教职工配备标准(暂行)》配足配齐各类幼儿园教职工              | 学前教育 | 达标率       |
| 学前-36 | 以游戏为基本活动，无"小学化"现象                          | 学前教育 | 专家评分      |
| 义务-28 | 所有学校德育工作、校园文化建设水平达到良好以上                    | 义务教育 | 第三方评估     |
| 义务-31 | 在国家义务教育质量监测中，相关科目学生学业水平达到Ⅲ级以上，且校际差异率低于0.15 | 义务教育 | 测试成绩+差异系数 |

**量化方案**：

```
方式A（有量化数据支撑）：
  综合得分 = 达标园数 / 总园数 × 100分

方式B（无量化数据，需评估）：
  综合得分 = 专家评分 × 权重 + 量化数据得分 × 权重
  判定：90-100=优秀，75-89=良好，60-74=合格，<60=不合格

方式C（差异系数法）：
  CV = 标准差 / 均值
  判定：CV ≤ 0.15 为达标
```

**系统实现**：

| 数据模块     | 字段设计                          |
| -------- | ----------------------------- |
| 玩教具配备检查表 | 机构编码、检查年度、园舍达标、玩教具达标、图书达标     |
| 专家评估记录表  | 机构编码、评估年度、评估指标、专家评分、评估机构      |
| 学业监测成绩表  | 学校编码、监测年度、学科、平均分、等级、参与人数      |
| 差异系数计算表  | 指标编号、各学校数值列表、平均值、标准差、CV值、是否达标 |

**差异系数核心算法**：
```
1. 平均值 = Σxi / n
2. 标准差 = sqrt(Σ(xi - 平均值)² / (n-1))
3. 差异系数CV = 标准差 / 平均值
4. 判定：CV ≤  threshold 为达标
```

**政策依据**：[[Education-Resource-Evaluation调研报告]] 明确指出"差异系数是核心计算工具"，[[教育质量监测与评估指标调研报告]] 提供了国家质量监测的标准化测试工具。

### 2.4 安全底线类 → 一票否决 + 事件计数

**适用指标**：

|  序号   | 指标名称                     | 教育阶段 | 否决性质 |
| :---: | ------------------------ | :--: | ---- |
| 学前-18 | 落实安全监管责任                 | 学前教育 | 一票否决 |
| 学前-19 | 督导评估认定前2年内无较大社会影响的安全责任事故 | 学前教育 | 一票否决 |
| 学前-24 | 全面完成无证园治理工作              | 学前教育 | 一票否决 |
| 学前-25 | 民办园没有上市、过度逐利等行为          | 学前教育 | 一票否决 |
| 义务-32 | 社会认可度达到85%以上             | 义务教育 | 底线指标 |

**量化方案**：

```
一票否决逻辑：
  if (触发否决条件) return 0分 (整体不通过)
  else 按常规计算得分

事故计分类：
  得分 = 基础分(100) - 一般事故数 × 扣分权重
  最低分：0分（不低于0）

社会认可度：
  认可度 = 满意人数 / 调查样本数 × 100%
  判定：≥85% 达标
```

**系统实现**：

| 数据模块 | 字段设计 |
|---------|---------|
| 安全事故记录表 | 机构编码、事故日期、事故等级、事故类型、伤亡情况、处理结果、是否影响评估 |
| 无证园治理台账 | 机构名称、发现日期、治理状态、治理完成日期、关停/整合方式 |
| 民办园合规检查表 | 机构编码、是否上市、是否过度逐利、检查结果、检查日期 |
| 社会认可度调查表 | 调查年度、样本总数、满意数、基本满意数、不满意数、认可度 |

**计算逻辑**：
```python
def 一票否决检查(安全记录, 治理台账, 合规检查):
    if 有重大安全事故(安全记录): return {"through": False, "reason": "存在重大安全事故"}
    if 有无证园未治理(治理台账): return {"through": False, "reason": "存在未治理无证园"}
    if 有违规上市逐利(合规检查): return {"through": False, "reason": "民办园违规上市/逐利"}
    return {"through": True}

def 社会认可度计算(调查数据):
    认可度 = (满意数 + 基本满意数 × 0.5) / 样本总数 × 100
    return {
        "认可度": round(认可度, 2),
        "达标": 认可度 >= 85
    }
```

## 三、系统架构设计

### 3.1 指标字典结构

```yaml
指标编码: 学前-05
指标名称: 制定幼儿园布局规划
指标类型: 制度建设类
教育阶段: 学前教育
量化方法: 二元判定
数据来源: 教育局规划科
计算公式: if(存在正式印发文件, 100, 0)
更新周期: 年度
预警阈值: 0分(未制定)
关联政策: 《幼儿园布局专项规划》
```

### 3.2 数据采集策略

| 数据类型 | 采集方式 | 自动化程度 | 负责模块 |
|---------|---------|:---------:|---------|
| 文档类（规划/制度） | 系统上传+人工审核 | 半自动 | 文档管理模块 |
| 数值类（覆盖率/达标率） | 对接业务系统API | 自动 | 数据同步模块 |
| 评估类（质量评分） | 第三方评估导入 | 手动导入 | 评估管理模块 |
| 问卷类（社会认可度） | 在线问卷系统对接 | 自动 | 问卷管理模块 |
| 名单类（骨干教师/持证教师） | Excel导入/对接人事系统 | 半自动 | 师资管理模块 |

### 3.3 核心计算引擎

系统需内置以下计算引擎：

| 引擎名称 | 功能 | 输入 | 输出 |
|---------|------|------|------|
| 差异系数计算器 | 计算校际均衡度 | 各学校某指标数值列表 | CV值、是否达标 |
| 覆盖率计算器 | 计算政策落实率 | 达标数、总数 | 百分比、得分 |
| 一票否决引擎 | 检查底线指标 | 安全/合规记录 | 通过/不通过、原因 |
| 综合评分引擎 | 加权汇总各维度 | 各指标得分 | 综合得分、等级 |
| 完成度计算器 | 计算制度建设进度 | 各制度状态 | 完成百分比 |

### 3.4 数据库表设计（核心表）

```
教育机构表 (edu_institution)
  - id, 机构编码, 机构名称, 机构类型(幼儿园/小学/初中), 所属区域, 状态

指标字典表 (indicator_dict)
  - id, 指标编码, 指标名称, 指标类型, 教育阶段, 量化方法, 计算公式, 权重

政策落实记录表 (policy_implementation)
  - id, 机构id, 指标编码, 是否落实, 落实日期, 证明材料, 审核状态

评估记录表 (evaluation_record)
  - id, 机构id, 指标编码, 评估年度, 评估得分, 评估机构, 专家评分

安全事件表 (safety_incident)
  - id, 机构id, 事件日期, 事件等级, 是否影响评估, 处理结果

问卷调查表 (survey_record)
  - id, 调查年度, 样本总数, 满意数, 基本满意数, 不满意数, 认可度
```

## 四、实施路线图

### 4.1 阶段划分

| 阶段 | 内容 | 交付物 | 周期 |
|------|------|-------|-----|
| **第一期** | 制度建设类二元判定<br>覆盖率/达标率计算<br>一票否决逻辑 | 指标字典模块<br>数据采集表单<br>基础计算引擎 | 4周 |
| **第二期** | 差异系数计算<br>社会认可度调查<br>综合评分引擎 | 差异系数模块<br>问卷系统对接<br>综合评分看板 | 4周 |
| **第三期** | 专家评估数据导入<br>复杂合规性判定<br>历史数据迁移 | 评估管理模块<br>数据迁移工具<br>全量功能上线 | 4周 |

1. **指标字典建立**：将30+13项不可量化指标逐一编码入库
2. **数据采集表单开发**：针对每类指标设计对应的数据录入界面
3. **计算引擎开发**：优先实现覆盖率、二元判定、一票否决三个核心算法
4. **看板原型**：展示各指标量化结果的红绿灯状态

## 五、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 教育局数据未到位 | 无法关联片区和学校信息 | 先用小区名称建立临时映射表，手动标注归属 |
| 骨干教师标识缺失 | 指标2无法量化 | 建立骨干教师名单台账，通过人工导入补充 |
| 专用教室数量无字段 | 指标10无法计算 | 在机构基础信息表中新增"专用教室"字段 |
| 数据丢失待重导 | 指标5/6/7无法计算 | 协调后勤/装备部门重新提供历史数据 |
| 专家评估主观性强 | 质量类指标评分偏差 | 制定统一评分量规，多人评分取平均 |
