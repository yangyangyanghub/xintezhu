## 一、整体架构：底座 → 引擎 → 应用

```
┌─────────────────────────────────────────────────────┐
│              低空监管平台（业务应用层）               │
│  ├─ 无人机空域管理  ├─ 飞行监控  ├─ 低空数据采集     │
│  └─ 巡查任务派发    ├─ 告警处置  └─ 航迹归档        │
└──────────────────────┬──────────┬───────────────────┘
                       │ 调用     │ 推送数据
                       ▼          │
┌─────────────────────────────────────────────────────┐
│           套合分析平台（分析引擎/决策层）             │
│  ├─ 多图层叠加分析  ├─ 合规性审查  ├─ "三线"检测    │
│  ├─ 违法图斑判定    ├─ 耕地保护分析 └─ 督察报告生成  │
└──────────────────────┬──────────────────────────────┘
                       │ 获取管控图层 + 基础数据
                       ▼
┌─────────────────────────────────────────────────────┐
│          时空数据平台（数据底座/基础设施层）           │
│  ├─ 基础地理数据  ├─ 遥感影像  ├─ 倾斜摄影/三维模型 │
│  ├─ BIM/实景三维  ├─ 行政区划  ├─ 地类数据          │
│  ├─ 生态红线/基本农田/城镇开发边界（三线）            │
│  └─ OGC 标准服务发布（WMS/WFS/WMTS）                │
└─────────────────────────────────────────────────────┘
```

## 二、逐一拆解

| 对比维度 | **时空数据平台** | **套合分析平台** | **低空监管平台** |
|---------|:---:|:---:|:---:|
| **定位** | 数据底座 | 分析引擎 | 业务应用 |
| **核心能力** | 数据汇聚、存储、服务发布 | 多图层叠加、合规判定 | 低空空域管控、无人机监管 |
| **服务对象** | 所有上层平台 | 督察/审批/执法部门 | 低空经济管理部门 |
| **数据来源** | 测绘院、遥感、各部门 | 从时空平台读取 | 无人机传感器回传 |
| **输出成果** | 标准化数据服务 | 分析报告、违法图斑 | 飞行监控、轨迹记录 |

## 三、三者联动关系

### 3.1 时空数据平台 → 套合分析平台（数据供给）

- 提供生态红线、永久基本农田、城镇开发边界"三线"图层
- 提供地类变更调查数据、国土空间规划数据
- 提供实景三维、倾斜摄影等可视化底图
- **场景**：磁县天保寨 30.92 亩用地是否压占生态红线，就是从时空平台读红线图层 + 项目范围图层做套合计算

### 3.2 低空监管平台 → 套合分析平台（数据闭环）

- 无人机航拍/巡查发现疑似违法占地
- 采集正射影像或视频回传
- 推送至套合分析平台做合规性判定
- **场景**：广平留女固村 11 层楼违法建设，可通过无人机巡查发现 → 回传影像 → 套合分析判定占基本农田

### 3.3 时空数据平台 → 低空监管平台（底图支撑）

- 提供三维城市模型，用于航线规划和禁飞区设置
- 提供高层建筑、变电站、机场等障碍物数据
- 提供实时地理信息服务

## 四、一句话总结

> **时空数据平台是"地基"**（管数据），**套合分析平台是"大脑"**（管分析），**低空监管平台是"眼睛和手"**（管采集和巡查）。
>
> 三者构成 **"数据采集 → 叠合分析 → 执法决策"** 的全链条闭环。
