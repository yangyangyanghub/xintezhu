# 临漳县自然资源低空智能监管体系汇报思路

## 一、汇报定位

本次给临漳县自然资源局局长汇报，不能只讲“县城违建房监管”，也不宜单独讲“无人机巡查”。建议把违建监管放到自然资源整体监管体系中表达。

推荐定位：

> 以时空数据智能开放平台为底座，以套合监管系统为核心能力，以卫星遥感和低空巡查为感知手段，构建临漳县自然资源空天地一体化智能监管体系。

核心口径：

> 临漳县自然资源低空监管体系不是单一无人机巡查应用，而是依托时空数据智能开放平台汇聚自然资源管理数据，通过套合监管系统自动叠加地类、规划、红线、审批、权属等信息，再结合卫星遥感和低空巡查，实现从“发现变化”到“判断属性”再到“核查处置”的闭环监管。

## 二、推荐汇报主题

推荐标题：**临漳县自然资源空天地一体化智能监管体系建设思路**

副标题：

> 以“时空数据智能开放平台 + 套合监管系统 + 卫星遥感 + 低空巡查”支撑县域自然资源精细化监管

## 三、局长关切判断

面向县自然资源局局长，汇报重点应从“技术能力”转向“业务体系”和“监管成效”。

局长可能关心：

1. **所有管理数据能不能汇聚起来**：地类、规划、审批、红线、权属、执法、影像等数据是否能统一管理。
2. **发现问题后能不能快速判断属性**：占什么地、压什么线、有没有审批、归哪个乡镇或权属主体。
3. **能不能支撑自然资源主责主业**：耕地保护、违法用地、设施农用地、批后监管、违建监管。
4. **能不能形成闭环**：不仅看图和航拍，还要支撑发现、研判、核查、整改、复核、归档。
5. **是否贴合临漳实际**：临漳没有矿山，汇报中不设置矿山监管场景。

## 四、总体架构

建议将体系讲成“四梁”：

```text
一底座：时空数据智能开放平台
一核心：套合监管系统
一抓手：卫星遥感 + 低空智能巡查
一闭环：自然资源业务监管流程
```

### 1. 一底座：时空数据智能开放平台

汇聚自然资源所有管理数据，形成统一、可调用、可分析的县域时空数据底座。

可汇聚数据包括：国土变更调查、三调和年度变更、永久基本农田、耕地保护目标、国土空间规划、城镇开发边界、生态保护红线、建设用地审批、土地供应、规划许可、执法监察图斑、设施农用地备案、临时用地审批、卫星遥感影像、无人机低空影像、倾斜摄影、行政区、村界、宗地、权属等基础数据。

一句话表达：

> 先把自然资源监管需要看的“底账、底图、底线、底数”统一起来。

### 2. 一核心：套合监管系统

对新增变化、疑似问题、低空识别图斑，自动与地类、规划、红线、审批、权属等管理数据进行空间套合，辅助判断问题性质和监管方向。

关键表达：

> 低空影像发现的是“现象”，套合监管系统判断的是“问题属性”。

| 套合对象 | 套合目的 | 输出判断 |
|---|---|---|
| 地类图斑 | 看占用什么地类 | 耕地、建设用地、林地、园地、未利用地等 |
| 永久基本农田 | 看是否触碰底线 | 是否占用永久基本农田 |
| 国土空间规划 | 看是否符合规划用途 | 是否符合规划管控要求 |
| 城镇开发边界 | 看是否在建设管控范围内 | 城镇集中建设区 / 边界外 |
| 生态保护红线 | 看是否涉及生态约束 | 是否压占生态保护红线 |
| 审批红线 | 看是否有合法审批依据 | 未批先建、超范围建设、少批多占 |
| 设施农用地备案范围 | 看是否超备案范围 | 超范围建设、改变用途 |
| 临时用地范围 | 看是否超期、超范围 | 临时用地违规线索 |
| 权属 / 宗地 | 看责任主体 | 关联乡镇、村、单位、权利人 |

### 3. 一抓手：卫星遥感 + 低空智能巡查

卫星遥感负责全域周期性发现变化，低空巡查负责重点区域精细核查、取证和复核。

表达公式：

```text
卫星看全域
低空看细节
套合判属性
平台管闭环
```

### 4. 一闭环：自然资源业务监管流程

```text
影像采集 → 智能识别 → 疑似图斑 → 自动套合 → 属性研判 → 人工核查 → 整改处置 → 低空复核 → 台账归档
```

## 五、临漳县重点应用场景

临漳县没有矿山，汇报中应删除矿山监管场景，避免显得照搬模板。

建议保留 5 类核心场景：

1. 耕地保护监管
2. 违法用地监管
3. 设施农用地监管
4. 建设项目批后监管
5. 县城及村庄违建房监管

可选拓展场景：闲置低效用地监管。

| 场景 | 发现方式 | 套合判断 | 输出成果 |
|---|---|---|---|
| 耕地保护监管 | 卫星变化、低空巡查、田长制巡查线索 | 是否占耕、是否占永久基本农田、是否存在非农化 / 非粮化风险 | 疑似占耕图斑、面积统计、位置清单、影像证据 |
| 违法用地监管 | 新增建设变化图斑、低空巡查、群众举报线索 | 是否有审批、是否超审批范围、是否符合规划、是否少批多占 | 未批先建、少批多占、违法占地线索清单 |
| 设施农用地监管 | 低空影像、备案数据、日常巡查 | 是否超备案范围、是否改变用途、是否硬化、是否存在“大棚房”风险 | 设施农用地问题台账、前后对比图、核查建议 |
| 建设项目批后监管 | 项目红线叠加低空影像、施工进度巡查 | 是否按审批范围建设、是否超红线、是否长期闲置、是否擅自改变用途 | 批后监管清单、项目进度影像、异常问题图斑 |
| 县城及村庄违建房监管 | 低空巡查、变化检测、重点区域航拍 | 套合地类、规划、审批、权属、历史台账 | 疑似违建点位、楼顶 / 院落扩建清单、拆后复核成果 |
| 闲置低效用地监管（可选） | 卫星 / 低空影像、供地数据、项目数据 | 是否已供未建、开工不足、建设停滞、利用效率低 | 闲置低效用地疑似清单、现状影像、项目监管台账 |

## 六、能力特点

### 特点一：不是单一影像平台，而是自然资源全量数据底座

> 依托时空数据智能开放平台，统一汇聚自然资源管理、规划、审批、执法、影像等数据，为监管研判提供完整依据。

### 特点二：不是人工看图，而是自动套合研判

> 对发现的疑似图斑，系统自动进行地类、规划、红线、审批、权属等多维套合，辅助快速判断问题属性。

### 特点三：不是单次航拍，而是常态化监管闭环

> 低空巡查成果进入问题台账，支撑核查、整改、复核和销号，形成持续监管机制。

### 特点四：不是只服务违建，而是服务自然资源全业务

> 体系可扩展到耕地保护、违法用地、设施农用地、批后监管、县城及村庄违建等多个自然资源场景。

## 七、PPT 建议结构

1. 标题页：临漳县自然资源空天地一体化智能监管体系建设思路
2. 建设背景：自然资源监管从人工巡查向空天地一体化转变
3. 总体思路：数据汇聚、智能套合、低空核查、业务闭环
4. 核心能力一：时空数据智能开放平台
5. 核心能力二：套合监管系统
6. 核心能力三：卫星遥感与低空巡查协同
7. 应用场景总览：五类核心场景
8. 重点场景一：耕地保护监管
9. 重点场景二：违法用地监管
10. 重点场景三：设施农用地和批后监管
11. 示范场景：县城及村庄违建房监管
12. 闭环流程：发现、套合、研判、核查、处置、复核、归档
13. 实施路径：建底座、跑场景、成体系
14. 预期成效：提升发现效率、研判效率、核查效率、闭环能力

## 八、汇报话术

### 1 分钟版本

> 局长，我们这次不建议单独讲违建，也不建议单独讲无人机。临漳更适合建设一套自然资源空天地一体化智能监管体系。底层依托时空数据智能开放平台，把自然资源现状地类、规划管控、审批、执法、权属、影像等管理数据统一汇聚起来；中间通过套合监管系统，对卫星遥感、低空巡查和日常巡查发现的问题图斑，自动套合地类、规划、红线、审批和权属信息，快速判断问题属性；上层再服务耕地保护、违法用地、设施农用地、批后监管、县城及村庄违建等业务场景。县城违建可以作为第一个示范应用先跑通闭环，但最终建设的是服务自然资源全业务的监管体系。

## 九、注意事项

1. 不讲矿山监管，临漳没有矿山，讲了会显得套模板。
2. 不要把低空巡查讲成主角，主角应是自然资源监管体系。
3. 不承诺 AI 自动判定违法，只表达为辅助发现线索和辅助研判属性。
4. 违建监管只作为首个示范场景，不作为全部建设内容。
5. 多强调时空数据智能开放平台和套合监管系统，这是方案差异化能力。

## 十、内部记忆点

```text
时空数据智能开放平台解决“数据在哪里”
套合监管系统解决“问题是什么”
卫星遥感解决“全域哪里变了”
低空智能巡查解决“现场怎么看清”
业务闭环机制解决“问题怎么办”
```
