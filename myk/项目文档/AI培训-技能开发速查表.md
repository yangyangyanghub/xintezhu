# OpenCode 技能开发速查表

> 一页纸掌握技能开发核心要点

---

## 一、什么是技能？

**技能 = 提示词 + 工具 + 工作流的封装**

- 通用AI：60分（啥都能干，啥都不精）
- 定制技能：90分（专注一件事，做到极致）

---

## 二、技能文件结构

```
my-skill/
├── SKILL.md          # 技能定义（必需）
├── scripts/          # 辅助脚本（可选）
└── references/       # 参考文档（可选）
```

---

## 三、SKILL.md 核心字段

```markdown
# Skill: 技能名称

简短描述这个技能做什么、什么时候用。

## 核心指令

这是技能的"灵魂"——告诉AI具体怎么做。

### 步骤1：xxx
### 步骤2：xxx
### 步骤3：xxx

## 输出格式

定义输出的结构和格式要求。

## 约束条件

- 必须做：xxx
- 禁止做：xxx

## 示例

输入：xxx
输出：xxx
```

## 四、开发步骤速查

| 步骤 | 动作 | 要点 |
|------|------|------|
| 1. 定需求 | 明确输入输出 | 问自己：用户会怎么触发？期望什么结果？ |
| 2. 写初稿 | 创建SKILL.md | 先写核心指令，再加约束和示例 |
| 3. 测试 | 用真实场景测试 | 至少测3个不同case |
| 4. 优化 | 根据结果调提示词 | 哪里不对改哪里，别大改 |
| 5. 发布 | 分享给同事 | 放到团队共享目录 |

## 五、提示词技巧

### ✅ 好的指令
```
你是一个会议纪要专家。
输入：会议内容（录音转文字）
输出：结构化会议纪要，包含以下部分：
1. 会议主题
2. 讨论议题（按重要性排序）
3. 关键结论
4. 待办事项（责任人+截止日期）
5. 遗留问题
```

### ❌ 差的指令
```
帮我写会议纪要。
```

**核心原则**：
- **角色明确** - 告诉AI它是谁
- **输入清晰** - 说明会收到什么
- **输出具体** - 详细定义输出格式
- **示例必要** - 给1-2个例子

## 六、常见问题

| 问题 | 解决方案 |
|------|---------|
| 输出格式不稳定 | 加示例，用JSON schema约束 |
| AI不按指令做 | 强化约束，用"必须"、"禁止" |
| 效果时好时坏 | 降低温度参数，或加更多示例 |
| 技能触发不了 | 检查description是否包含关键词 |

## 七、技能组合思路

```
用户输入 → [技能A: 分析] → [技能B: 生成] → 最终输出
```

**示例链路**：
```
客户需求文档 → [需求分析技能] → 结构化需求 → [方案生成技能] → 投标方案
```

## 九、建筑五图技能体系（培训演示案例）

### 技能架构

```
building-five-drawings/     # 主控技能（串联流程）
    ├── building-concept/       # 1. 设计概念图
    ├── building-masterplan/    # 2. 总平面图
    ├── building-analysis/      # 3. 分析图
    ├── building-rendering/     # 4. 效果图
    └── building-construction/  # 5. 施工图
```

### 五张图纸

| 序号 | 技能 | 功能 | 工具 |
|------|------|------|------|
| 1 | 设计概念图 | 概念图、意向图 | 千问生图 |
| 2 | 总平面图 | 场地规划、建筑布局 | FreeCAD CLI |
| 3 | 分析图 | 功能分区、流线分析 | 千问+FreeCAD |
| 4 | 效果图 | 透视图、鸟瞰图 | 千问生图 |
| 5 | 施工图 | 节点大样、材料做法 | FreeCAD CLI |

### 技能文件位置

```
.opencode/skill/building-five-drawings/SKILL.md  # 主控
.opencode/skill/building-concept/SKILL.md         # 概念图
.opencode/skill/building-masterplan/SKILL.md      # 总平面
.opencode/skill/building-analysis/SKILL.md        # 分析图
.opencode/skill/building-rendering/SKILL.md       # 效果图
.opencode/skill/building-construction/SKILL.md    # 施工图
```

### 设计理念

**为什么做五个独立技能？**
- 每个技能独立可用，不强制串流程
- 每个技能做精做细，质量更高
- 主控技能串联流程，灵活组合
- 便于维护和迭代
