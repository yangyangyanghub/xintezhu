## 一、战略判断：为什么现在必须行动

三个信号同时亮起，窗口期在6-12个月：

1. **政策信号**：国务院《城市更新"十五五"规划》已发布，住建部2025年城市体检工作手册再次明确"先体检后更新、无体检不更新"，并要求建立"体检发现问题、更新解决问题、评估实施效果、推动巩固提升"的闭环机制；自然资源部2025版体检规程明确体检报告与供地计划挂钩。
2. **市场信号**：2026年1-5月仅河北省城市更新相关采购意向即达23.12亿元，其中城市体检+更新规划打包招标已成标配（约60%的县级公告采用此模式）。
3. **本地信号**：《邯郸市城市更新条例》已施行，邯郸市2026年城市体检和县城体检工作方案已出，各县区规划编制需求即将释放。
4. **产品信号**：2025年手册把住房、小区（社区）、街区、城区（城市）四级单元，以及问题清单、整治建议清单、次年更新计划、整治成效评估串成标准流程，天然适合沉淀为数据标准、采集工具、指标计算引擎和项目库产品。

**核心结论**：城市体检评估是所有城市更新业务的入口，掌握体检能力就掌握业务入口。而**邯郸市规划设计院拥有从体检→规划→设计→监理的全产业链能力**，这是任何单一专业机构无法复制的竞争优势。

## 二、邯郸市规划设计院能力地图

### 2.1 院况概览

- 成立于1987年，隶属邯郸市自然资源和规划局，副县级公益二类事业单位
- 核定编制153名，内设机构15个
- 综合性甲级设计研究单位，2008年通过ISO9001认证
- 服务对象：市委市政府、自然资源和规划局、住建局、城管局及各县委托

### 2.3 院里独特优势（竞争者难以复制）

1. **体制内归属**：隶属于自然资源和规划局，天然承接国土空间规划+规划评估业务，外来机构无法企及
2. **全产业链**：从体检评估到规划设计到建筑设计到市政园林到监理，8大专业一条龙，无需拼凑联合体
3. **本地数据积累**：近40年邯郸规划测绘数据沉淀，外来机构从零开始至少1-2年
4. **政府信任关系**：同时服务市委市政府、自然资源局、住建局、城管局，信息对称优势明显
5. **人才规模**：153人编制，15个内设机构，可同时开展多个项目

## 三、两套体检体系对全院业务的需求映射

### 3.1 需求对比

| 需求维度 | 住建部体检（服务城市更新） | 自然资源部体检（服务规划监督） | 对应院里业务 |
|---------|-------------------------|---------------------------|------------|
| **空间数据** | 小区/街区级微观数据 | 全域宏观+中观数据 | 测绘→地理信息 |
| **遥感影像** | 变化检测辅助问题识别 | 年度国土变更调查核心输入 | 测绘→遥感处理 |
| **GIS分析** | 15分钟生活圈覆盖率 | 空间冲突识别、开发边界监测 | 地理信息→规划 |
| **数据平台** | 城市体检信息平台（CIM底座） | 国土空间规划"一张图"系统 | 信息化→平台建设 |
| **规划编制** | 城市更新专项规划、片区策划 | 国土空间规划动态维护 | **规划设计+空间规划** |
| **建筑设计** | 老旧小区改造方案、危旧房改造 | — | **建筑设计** |
| **市政设计** | 管网更新改造、海绵城市 | — | **基础设施设计** |
| **园林景观** | 口袋公园、绿道、街区品质提升 | 生态修复、绿化覆盖率提升 | **园林+景观设计** |
| **工程监理** | 更新项目施工监理 | — | **监理** |
| **自动化** | 人工采集为主 | 110项指标68项可自动计算 | 信息化→计算工具 |

### 3.2 协同切入点（约1/4指标重叠）

两套体系重叠的指标主要集中在：15分钟社区生活圈覆盖率、公共服务设施步行覆盖、生态环境质量、安全韧性、人均公园绿地。

**策略**：建立统一空间数据库，一次采集、按两套指标体系分别计算输出，实现"一份数据、两份报告"。这同时为后续规划编制和设计工作提供数据底座。

**2025手册带来的标准化机会**：住建部体检已经把工作对象拆成住房、小区（社区）、街区、城区（城市）四级单元，并要求形成问题清单、整治建议清单和下一年度更新整治计划。院里可把这套流程固化为"四级单元编码+指标数据字典+问题整治闭环表"三件标准件，后续无论承接市级、县级还是重点片区体检，都复用同一套数据底板和成果模板。

3. 开发体检指标自动化计算引擎MVP（优先重叠的1/4指标）
4. 先做轻量方案（GIS工具包+数据采集APP+报告生成），CIM作为2.0升级
5. 先沉淀"四级体检单元数据标准包"，把住房、小区、街区、城区四级采集字段、编码规则、质检规则做成可复制交付物
6. 开发"问题清单-整治建议-项目库"闭环模板，支撑从体检结果到次年城市更新计划的自动衔接
7. 针对管网、施工安全、高层建筑消防、CIM三维覆盖率等智慧化指标，建立专项数据接入清单和评估脚本

**核心技术路线**：

| 决策点 | 推荐方案 | 理由 |
|-------|---------|------|
| 前端框架 | Vue3 + Cesium/OpenLayers | 3D场景支持+2D GIS成熟生态 |
| 后端框架 | Python FastAPI | GIS计算生态丰富（geopandas/shapely） |
| 数据库 | PostgreSQL + PostGIS | 空间数据标准选择 |
| 报告生成 | Pandoc + 自定义模板 | 两种体检报告格式一键生成 |
| 标准化配置 | 指标字典+四级单元编码+表单 schema | 把手册要求固化为可复用交付标准 |
| 闭环管理 | 问题库+整治建议库+项目库+年度计划字段 | 支撑"体检发现问题→更新解决问题→评估实施效果" |
| 智慧化指标接入 | API/Excel/空间图层多源适配 | 对接管网、施工安全、消防、CIM等部门数据 |
| 部署方式 | 政务云/私有化部署 | 数据安全合规要求 |

### 6.1 价值链贯通

```
体检评估 ──→ 问题清单 ──→ 整治建议 ──→ 项目库/年度计划 ──→ 规划设计 ──→ 工程实施 ──→ 成效评估
(入口)      (诊断)       (处方)        (转化)           (大宗产值)    (全周期)     (闭环运营)
  │           │            │             │                 │             │             │
  ├─测绘      ├─GIS分析     ├─规划设计    ├─信息化平台       ├─建筑设计     ├─监理        ├─平台运营
  ├─地理信息  ├─问题库      ├─市政设计    ├─年度计划         ├─基础设施     ├─质量管控    ├─数据更新
  └─信息化    └─数据质检    └─优先级排序  └─更新项目储备     ├─园林景观     └─进度反馈    └─年度复检
```

### 6.2 标准投标包设计

| 投标包 | 组合内容 | 目标项目 | 核心优势 |
|-------|---------|---------|---------|
| **A包：体检+规划一体化** | 城市体检评估+城市更新专项规划+数据采集 | 县级"体检+规划"打包招标 | 体检数据直接为规划服务，无信息衰减 |
| **B包：体检+平台一体化** | 城市体检评估+体检信息平台+数据采集APP | 市级城市体检项目 | 数据+平台闭环，持续运营收入 |
| **C包：规划+测绘+设计一体化** | 城市更新专项规划+现状测绘+建筑/市政设计 | 片区策划和项目实施方案 | 一条龙服务，减少协调成本 |

| **E包：全流程一体化** | 体检+规划+测绘+设计+监理+平台 | 年度城市体检更新项目 | 全院8大专业协同，无法被单项竞争者击败 |
| **F包：标准化产品包** | 四级单元数据标准+采集表单+问题整治闭环模板+报告模板 | 首次开展体检的县区/部门试点 | 周期短、价格低、容易先切入，后续可升级平台 |
| **G包：智慧化指标专项包** | 管网智能监测、施工安全监测、消防预警、CIM覆盖率等指标接入评估 | 城市生命线/智慧住建/城管专项 | 对接手册智慧化指标，适合与基础设施、市政业务联动 |

## 七、住建部样本城市经验借鉴（2026年城市体检工作部署视频会）

> 来源：住房城乡建设部2026年城市体检工作部署视频会经验交流材料（2026年5月）
> 参会样本城市：吉林省、江苏省、广州市、呼和浩特市、武夷山市

### 7.1 信息化/数字化实践亮点

| 城市 | 实践 | 对我们的启示 |
|------|------|------------|
| **吉林省** | 建设省级体检信息平台；长春市开发数据采集APP；通化市设置微信小程序；延吉市采用智慧化数据采集+空间可视化分析 | 省级平台+市级APP/小程序是标配，空间可视化是差异化亮点 |
| **江苏省** | "规划评估+综合体检+专项体检+日常巡检"四位一体；利用既有信息化平台完善体检功能模块，支撑闭环管理；15项体检指标纳入省级示范创建 | 体检指标与示范创建挂钩→指标计算工具可复用；"四位一体"模式可产品化 |
| **广州市** | 以CIM基础平台为底座搭建城市体检评估全流程信息系统；开发移动端调查应用（现场调查+定位+拍照举证）；集成百度/高德/大众点评等开放大数据；建立基于深度学习算法+地理加权叠加+多级缓冲区分析的第三方体检评估模型；形成2.3万余栋房屋体检档案 | CIM底座+开放大数据+深度学习模型是最完整的数字化路径；移动端调查APP是刚需 |
| **呼和浩特** | 全面推行体检数据采集程序（手机拍照+数据录入）；技术团队后台实时监测+异常数据即时预警；累计收集4.6万条数据覆盖362个社区；建立城市排水管网地理信息系统 | 手机端数据采集+后台监测预警是低成本方案；管网GIS可直接产品化 |

### 7.3 可复制的业务模式

**模式一：省级体检信息平台+市级数据采集工具（吉林省模式）**
- 省级平台统一标准、统一数据格式
- 市级开发APP/小程序/智慧化采集工具
- 空间可视化分析作为差异化亮点
- **适合场景**：承接河北省体检信息平台建设或市级采集工具开发

**模式二：CIM底座+开放大数据+AI模型（广州市模式）**
- CIM平台作为空间底座
- 整合政府数据+开放大数据（百度/高德/POI）
- 深度学习+地理加权+缓冲区分析构建第三方评估模型
- 移动端调查APP（定位+拍照举证+项目管理）
- **适合场景**：开发"城市体检评估全流程信息系统"，打包数据+平台+模型

**模式三：多部门数据整合+动态更新数据库（武夷山模式）**
- 整合16个部门60余类数据→90余个数据图层
- 建成统一规范、动态更新的体检数据库
- 网格员深度摸排+技术团队后台监测
- **适合场景**：承接市县级"城市体检数据库"建设项目，数据整合是核心交付物

### 7.4 对行动方案的修正建议

基于样本城市经验，对第五章和第六章的行动计划提出3点修正：

1. **P0优先级新增**：开发移动端数据采集APP（长春/通化/呼和浩特均已标配），这是进入体检项目的最低门槛工具
2. **产品化方向新增**："城市体检数据库"建设服务（整合多部门数据+动态更新），参考武夷山90余个数据图层的交付标准
3. **技术路线修正**：CIM底座不是必选项——吉林省用省级信息平台+市级APP也跑通了；建议先做轻量方案（GIS工具包+数据采集APP+报告生成），CIM作为2.0升级
4. **标准化方向新增**：基于2025手册先做"四级单元+指标字典+问题整治闭环"标准包，解决县区首次开展体检时不知道怎么采、怎么填、怎么质检、怎么形成项目库的问题
5. **专项产品方向新增**：围绕管网智能化监测管理率、建筑施工安全监测覆盖率、高层建筑智能化火灾监测预警覆盖率、CIM三维数据覆盖率等智慧化指标，形成可单独投标或作为平台模块销售的专项评估包
