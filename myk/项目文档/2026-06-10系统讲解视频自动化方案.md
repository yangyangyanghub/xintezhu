# 系统讲解视频自动化方案

> 方案日期：2026年6月10日
> 目标：基于已有 Web/App 项目，自动或半自动生成系统 walkthrough / explainer 视频。

---

## 一、核心结论

当前最可行的路线不是单纯使用 HTML 视频生成，也不是单纯使用 OpenScreen 录屏，而是采用分层流水线：

```text
Playwright 自动操作真实系统
→ 录制原始系统操作视频
→ cli-anything-openscreen 自动剪辑、缩放、标注
→ 可选 Hyperframes/html-video 做品牌化包装和 TTS
→ 输出最终 MP4
```

关键判断：

1. Hyperframes/html-video 适合做 HTML composition 渲染，不适合直接录制已有系统。
2. OpenScreen 适合真实屏幕/窗口录制，但本体偏 Electron GUI，不是 headless SDK。
3. cli-anything-openscreen 补齐了 OpenScreen 的自动化后期编辑能力，但不负责录屏。
4. Playwright 最适合作为 Web 系统的自动操作与录制采集层。
5. 推荐工程路线是 Playwright 采集 + OpenScreen CLI 编辑 + Hyperframes/html-video 包装。

---

## 二、工具分工

| 工具 | 最适合角色 | 优势 | 局限 |
|---|---|---|---|
| Playwright | 自动操作真实 Web 系统、录制原始素材 | 可登录、点击、填写、跳转、截图、录制视频 | 主要适合 Web，不适合原生桌面 App |
| OpenScreen | 真实屏幕/窗口录制、演示视频编辑 | 真实录制、支持缩放、字幕、摄像头、系统音频 | 本体偏 GUI，不是命令行录屏 SDK |
| cli-anything-openscreen | 自动化后期编辑器 | 可脚本化添加 zoom、trim、crop、annotation，并导出 MP4 | 不负责录屏，只编辑 `.openscreen` 项目 |
| Hyperframes | HTML composition → MP4 渲染 | 逐帧确定性渲染，适合品牌包装、片头、TTS、字幕条 | 要求自定义 HTML composition，不直接录现有页面 |
| html-video | Agent 驱动的 HTML 视频工作台 | 可把文章、仓库、prompt 生成分镜 HTML 视频 | 当前更像视频生成工作台，不是系统录屏器 |

---

## 三、推荐总体架构

```text
已有 Web/App 系统
  ↓
采集层：Playwright 自动操作、浏览器视频录制、截图、操作日志
  ↓
编辑层：cli-anything-openscreen 进行 zoom、trim、crop、文字标注
  ↓
包装层：Hyperframes/html-video 进行片头、章节、TTS、品牌模板包装，可选
  ↓
最终视频：final.mp4 / gif
```

---

## 四、分层说明

### 4.1 采集层：Playwright

采集层负责对已有系统进行真实操作，并生成原始视频素材。

主要任务：

1. 自动打开系统 URL。
2. 自动登录或注入登录态。
3. 执行业务操作流程，例如点击菜单、填写表单、切换页面、打开弹窗。
4. 记录每个操作的时间点、页面 URL、元素定位信息、截图。
5. 输出原始录屏文件 `raw-recording.mp4` 或 `raw-recording.webm`。
6. 输出操作日志 `actions.json`。

示例操作日志：

```json
{
  "video": "raw-recording.mp4",
  "steps": [
    {
      "timeMs": 2800,
      "action": "click",
      "selector": "text=项目管理",
      "description": "点击左侧项目管理，进入项目列表",
      "focusX": 0.18,
      "focusY": 0.42
    }
  ]
}
```

### 4.2 编辑层：cli-anything-openscreen

编辑层负责把原始录屏加工为适合讲解的视频片段。

可自动完成：

1. 新建 OpenScreen 项目。
2. 绑定原始录屏文件。
3. 添加缩放效果。
4. 添加文字标注。
5. 删除等待片段。
6. 设置裁剪区域。
7. 添加速度变化。
8. 导出 MP4。

示例命令：

```bash
cli-anything-openscreen project new demo.openscreen
cli-anything-openscreen --project demo.openscreen --save project set-video raw-recording.mp4
cli-anything-openscreen --project demo.openscreen --save zoom add --start 2800 --end 6200 --depth 2.2 --focus-x 0.18 --focus-y 0.42
cli-anything-openscreen --project demo.openscreen --save annotation add-text --start 3200 --end 6000 --text "点击左侧项目管理，进入项目列表" --x 0.5 --y 0.86
cli-anything-openscreen --project demo.openscreen export render final.mp4
```

### 4.3 包装层：Hyperframes/html-video

包装层不是必需项，但如果要做更正式、更品牌化的视频，建议加入。

适合处理：

1. 片头、片尾。
2. 章节页。
3. 品牌色背景。
4. 字幕条。
5. TTS 配音。
6. BGM。
7. 功能点卡片。
8. 多段录屏素材之间的转场。

## 五、三种落地路线

### 路线 A：最小可行方案

```text
Playwright 自动操作并录制
→ cli-anything-openscreen 自动剪辑和标注
→ final.mp4
```

适合内部培训、系统功能讲解、低成本客户演示和快速验证。

优点：链路短、实现快、真实系统保真度高。

缺点：视频包装能力一般，品牌化表达不足。

### 路线 B：推荐正式方案

```text
Playwright 自动操作并录制
→ cli-anything-openscreen 自动剪辑和标注
→ Hyperframes/html-video 做品牌化包装和 TTS
→ final.mp4
```

适合客户汇报、产品宣传、项目交付材料、视频号/公众号/小红书传播、系统培训课程。

优点：真实画面 + 自动后期 + 品牌包装，效果最均衡。

缺点：链路较长，需要维护视频模板。

### 路线 C：深度产品化方案

```text
抽离 OpenScreen native helper
→ 封装录屏 SDK
→ Playwright 控制系统
→ SDK 录屏
→ 自动编辑和包装
```

适合长期产品化和一键式系统讲解视频生成平台。

优点：能力最完整。

缺点：工程成本高，涉及 Electron、ScreenCaptureKit、Windows Graphics Capture、权限、音频、跨平台打包等复杂问题。第一阶段不建议直接做。

## 六、最小 MVP 设计

### 6.1 MVP 输入

1. 系统访问地址。
2. 登录信息或登录态文件。
3. Playwright 操作脚本。
4. 讲解步骤说明。
5. 输出视频规格，例如 1080p、30fps、MP4。

### 6.2 MVP 输出

1. `raw-recording.mp4`：原始系统操作录屏。
2. `actions.json`：操作时间线。
3. `edit-plan.json`：自动编辑计划。
4. `demo.openscreen`：OpenScreen 项目文件。
5. `final.mp4`：最终讲解视频。

### 6.3 MVP 流程

```text
Step 1：写 Playwright 脚本
Step 2：自动执行系统操作并录制视频
Step 3：根据操作日志生成 edit-plan.json
Step 4：调用 cli-anything-openscreen 自动加 zoom、annotation、trim
Step 5：导出 final.mp4
Step 6：人工 review 视频效果
```

### 6.4 MVP 验收标准

| 指标 | 标准 |
|---|---|
| 自动化程度 | 一条命令完成录制、编辑、导出 |
| 视频时长 | 30-90 秒 |
| 分辨率 | 1080p |
| 录屏真实性 | 画面来自真实系统操作 |
| 讲解清晰度 | 每个关键操作都有 zoom 或文字提示 |
| 可复现性 | 修改操作脚本后可重新生成视频 |
| 产物完整性 | 输出 raw-recording、edit-plan、final.mp4 |

## 七、关键风险与应对

| 风险 | 说明 | 应对 |
|---|---|---|
| 登录态不稳定 | 自动录制时登录过期或验证码阻断 | 使用预登录 storage state，避开验证码流程 |
| 页面加载等待过长 | 视频中出现大量空白等待 | 操作脚本中等待关键元素，后期自动 trim |
| UI 坐标不稳定 | 不同分辨率下 focusX/focusY 偏移 | 统一录制视口尺寸，并基于元素 bounding box 计算归一化坐标 |
| 录屏文件格式不一致 | Playwright 输出可能是 WebM | 编辑前统一转 MP4 或确认 CLI 支持输入格式 |
| OpenScreen CLI 视觉效果近似 | zoom 是 ffmpeg crop+scale，不完全等同 GUI PixiJS 动效 | MVP 阶段接受近似，正式阶段用 Hyperframes 做高级包装 |
| 桌面 App 自动化困难 | Playwright 只擅长 Web | 桌面 App 先用人工录制或系统级自动化，后期仍用同一编辑链路 |
| 批量视频质量参差 | AI 自动生成标注可能不准 | 首版引入人工 review，后续沉淀模板和规则 |

## 八、技术选型建议

### 第一阶段：验证闭环

使用 Playwright、cli-anything-openscreen、ffmpeg。

暂不引入 OpenScreen 源码改造、Hyperframes 深度集成、TTS、多模板系统。

目标：先证明真实系统操作 → 自动剪辑标注 → MP4 能稳定跑通。

### 第二阶段：提升观感

加入 Hyperframes/html-video、片头/章节页/片尾模板、TTS 配音、BGM、字幕条、品牌视觉规范。

目标：从能看升级为像正式产品演示视频。

### 第三阶段：产品化

加入可视化流程编辑器、脚本模板库、视频模板库、自动识别页面元素、自动生成讲解词、队列化批量生成、失败重试和质量检查。

目标：形成内部系统讲解视频自动生成平台。

---

## 九、当前推荐决策

建议采用路线 B，但实施顺序从路线 A 开始：

```text
先做路线 A 的 MVP，跑通后再增加 Hyperframes/html-video 包装层。
```

具体优先级：

1. P0：Playwright 录制真实系统操作。
2. P0：cli-anything-openscreen 自动加 zoom、annotation、trim 并导出。
3. P1：根据 Playwright 操作日志自动生成 edit-plan.json。
4. P1：统一视频模板和字幕样式。
5. P2：接入 Hyperframes/html-video 做片头、片尾、章节页、TTS。
6. P3：考虑抽离 OpenScreen native helper，做更完整的跨平台录屏能力。

## 十一、后续行动

建议下一步做一个 1 天 MVP：

1. 选择一个已有 Web 系统页面。
2. 写 Playwright 脚本完成 3-5 个关键操作。
3. 录制原始视频并生成操作日志。
4. 根据日志生成 edit-plan.json。
5. 调用 cli-anything-openscreen 自动编辑并导出 final.mp4。
6. 人工 review 视频效果，决定是否进入 Hyperframes/html-video 包装阶段。
