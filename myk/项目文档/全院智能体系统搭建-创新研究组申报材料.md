## 一、研究方案

### 1.1 研究背景

当前，AI 技术正从"辅助工具"向"智能协作体"快速演进，Agent（智能体）技术成为新一代 AI 的核心方向。在规划设计行业，AI 已从单点应用（代码生成、文档撰写）向复杂任务协作（多角色协同、全流程自动化）发展。

**我院信息化建设已具备良好基础**：
- 已建成多个业务系统（耕地保护平台、无人机飞控平台、生产管理系统等）
- 团队已在日常工作中深度使用 AI 工具（Claude Code、OpenCode 等）
- **Multica 多智能体平台已开通官方云服务**，先跑通测试验证，后续按需自行部署
- 积累了丰富的数字化转型经验和数据资产

**但仍存在以下痛点**：

| 痛点 | 说明 |
|------|------|
| **能力分散** | 各部门各玩各的，好的工具无法复用，重复造轮子 |
| **数据安全** | 员工私用外网大模型，存在数据泄露风险 |
| **效果不可控** | 输出质量参差不齐，没有统一标准和审核机制 |
| **审计追溯难** | 谁、什么时候、用 AI 干了什么，无法追溯 |
| **投入产出不清晰** | 买了大模型账号，但不知道产生了多少价值 |
| **知识沉淀难** | 优秀经验和方法难以在全院范围内快速复制推广 |

在此背景下，基于 **Multica + OpenSpace** 多智能体协作平台建设"全院智能体系统"，打造覆盖主要业务场景的 AI 助理矩阵，最终形成一个**通用智能体能力中枢**，对于提升全院生产效率、推动技术创新、培养 AI 应用人才具有重要意义。

> **技术架构说明**：
> - **Multica** 作为统一的智能体调度与管理平台，原生支持对接多种 Agent 运行时
> - **OpenSpace** 提供技能进化和 Token 优化能力
> - **Agent 运行时**可灵活选择：nanobot（轻量化）、Claude Code、OpenCode、自研 Agent 等
> - **最终目标**：通过 Multica 调度层，逐步形成一个能力全面、可处理院内大部分业务的"通用智能体"

### 1.2 目的意义

#### 1.2.1 提升生产效率
通过智能体自动化处理重复性工作，预计可将员工日均重复劳动时间减少 **30%-50%**，让专业人才聚焦于创造性工作。

#### 1.2.2 降低使用成本
通过 OpenSpace 的 Token 智能优化技术，平均减少 **46% Token 用量**，全院年 AI 使用成本预计仅需 **3-5 万元**。

#### 1.2.3 推动技术创新
探索 AI Agent 技术在规划设计行业的应用场景，形成可复制、可推广的技术方案，保持我院在行业内的技术领先地位。

#### 1.2.4 沉淀知识资产
通过技能自我进化机制，将优秀的工作方法、经验自动沉淀为标准化 Skill，实现"一人经验、全院受益"，加速人才培养和知识传承。

#### 1.2.5 保障数据安全
所有组件全部内网部署，数据不出院，全链路可审计，从根本上解决 AI 使用的数据安全问题。

#### 1.2.6 赋能业务创新
通过智能体技术赋能业务创新，探索"AI+规划"、"AI+管理"的新模式，培育新的业务增长点。

#### 1.3.2 核心智能体开发（首批 6 个，含1个GIS专项）

**① 项目管理智能体**
- 自动跟进项目进度，识别风险点
- 自动汇总日报、周报，生成项目进展报告
- 智能提醒关键节点和待办事项
- 项目数据可视化展示

**② 技术文档智能体**
- 自动整理项目文档，构建知识图谱
- 智能回答技术问题（"之前类似项目怎么做的？"）
- 自动生成项目汇报材料
- 规范文档模板和格式

**③ GIS数据处理智能体（专项）**
- ✅ **已验证可用**！自动化 GIS 数据处理流程（格式转换、坐标投影、质量检查）
- 智能识别数据问题，给出处理建议
- 自动生成数据处理报告和统计图表
- 常用数据处理算法封装和复用
- 对接 ArcGIS、QGIS 等专业工具

**④ 代码助手智能体**
- 辅助开发人员编写和审查代码
- 自动生成接口文档和测试用例
- 智能排查常见技术问题
- 推广最佳实践和编码规范

**⑤ 行政助理智能体**
- 会议纪要自动整理和待办提取
- 日程智能安排和冲突检测
- 常用公文模板和格式指导
- 差旅、报销等流程智能化辅助

**⑥ 规划设计辅助智能体**
- 上位规划文件智能解读和要点提取
- 规范条文智能检索和适用性分析
- 规划方案智能比对和优化建议
- 成果报告自动生成和格式规范化

#### 1.3.3 智能体开发平台建设
- 低代码智能体构建工具，让业务人员也能参与智能体开发
- 智能体测试与评估体系
- 智能体版本管理与发布流程
- 智能体效果监控与优化机制

#### 1.3.4 安全与审计体系建设
- 全链路操作留痕，谁、什么时候、干了什么、结果是什么，全可查
- 数据不出院，所有组件全部内网部署，无外网依赖
- 权限最小化设计：每个 Worker 只有自己工作目录的权限
- 人工可干预：任何环节都可以暂停、打回、人工介入审核
- 敏感数据自动检测和脱敏处理

#### 1.3.5 应用场景与最佳实践
- 梳理全院各业务场景的智能体应用机会
- 制定智能体使用规范和最佳实践
- 开展全员培训和推广活动
- 评估智能体应用效果，持续迭代优化

### 1.4 技术路线

```
试用云服务 → 跑通验证 → 需求调研 → 核心能力开发 → OpenSpace集成 → 试点应用 → 自部署评估 → 全面推广 → 通用智能体收敛
```

**技术选型**：
- **多智能体平台**：Multica（✅ 已开通官方云服务，先跑通测试，按需再自部署）
- **Agent 运行时**：灵活选择（nanobot / Claude Code / OpenCode / 自研），由 Multica 统一调度
- **进化与优化层**：OpenSpace（技能自我进化、Token 优化）
- **大模型底座**：支持多模型接入（Claude、GPT、文心一言、通义千问、DeepSeek等）
- **系统集成**：通过 MCP（Model Context Protocol）对接现有业务系统
- **前端交互**：支持钉钉嵌入、Web 页面、IDE 插件等多种接入方式

**演进策略**：
1. **初期（1-3月）**：用 Multica 云服务快速验证，多种运行时并行试用
2. **中期（4-6月）**：根据效果评估，确定最优运行时组合，按需自部署 Multica
3. **后期（6月+）**：基于积累的技能资产，**收敛形成院内通用智能体**，对外提供统一服务
