## 总体技术路线

第三方飞控平台包含设备管理、航线管理、任务管理、飞控操作等能力，接入方式分为两类：

- **MQTT 桥接**：同步机场/飞机 `osd` 信息、成果信息、任务状态信息等，数据内容需与上云 API、物模型设备属性推送保持一致。
- **HTTP 接口**：实现设备管理、直播推流、调度单、航线、飞行任务与成果管理。

## 接口认证说明

采用 **SM2 公私钥对签名认证**。

1. 调用方生成 SM2 公私钥对，密钥格式为 hex。
2. 调用方将公钥注册到被调用方。
3. 被调用方返回注册 ID。
4. 调用接口时在请求头携带 `xfc-token`。

Token 格式：

```text
ID.timestamp.sign
```

| 字段 | 说明 |
|------|------|
| `ID` | 被调用方返回的注册 ID |
| `timestamp` | 毫秒级时间戳 |
| `sign` | 对 `ID + timestamp` 的 UTF-8 字节数组做 SM2 签名，并将签名结果 Base64 编码 |

实现注意：Java 示例要求执行 `sm2.usePlainEncoding();` 禁用默认编码。文档未明确 token 过期时间、刷新机制和具体错误码表。

## 关键接口字段

### 设备基础信息获取

- URL：`GET /openapi/v1/device/device-topos`
- 请求参数：`status`，boolean，非必填，设备在线状态。
- 返回重点：`gateway`、`drone`、`sn`、`callsign`、`device_model`、`device_online_status`、`mode_code`、`camera_list`。

### 设备直播推流

- URL：`POST /openapi/v1/device/live-stream/start`
- Body：`sn`、`camera_index`、`video_expire`、`quality_type`。
- `quality_type`：`adaptive` 自动、`smooth` 流畅、`ultra_high_definition` 超清。
- 返回：`expire_ts`、`url`、`url_type`，其中 `url_type` 为 `webrtc` 或 `flv`。

### 停止直播通知

- URL：`POST /openapi/v1/device/live-stream/stop`
- Body：`sn`、`camera_index`。

### 调度单回调

- 方向：蜂巢系统 → 第三方飞控中心。
- URL：第三方提供回调地址。
- 方法：POST。
- 核心字段：`id`、`name`、`type`、`owner`、`creator`、`condition`、`beginTime`、`endTime`、`status`、`spots`。
- `type`：0-省级直查，2-地市调度。
- `status`：1-已派发，2-已接收，3-执行中，4-已完成，5-已关闭。
- `spots[]`：`bsm`、`tbly`、`objectId`、`xzqdm`、`xzqmc`、`tbmj`、`geom`、`metadata`。

### 调度单详情查询

- URL：`GET /openapi/v1/dispatch/{dispatchId}`。
- 返回重点：`name`、`type`、`owner`、`creator`、`condition`、`beginTime`、`endTime`、`createTime`、`status`、`spots`、`process`、`flights`。
- `flights[]`：`jobId`、`jobName`、`waylineId`、`wayline`、`dockSn`、`device`、`attachs`、`photos`、`progress`、`executeTime`、`completedTime`。

### 调度单飞行轨迹查询

- URL：`GET /openapi/v1/dispatch/{dispatchId}/route`。
- 返回重点：`dispatchId`、`dispatchType`、`dispatchOwner`、`metadata`、`distance`、`spent`、`dockSn`、`dockName`。
- `metadata[]` 点位包含：`x` 经度、`y` 纬度、`z` 高度。

### 航线与任务

- 航线上传：`POST /fczx-server/api/route/v1/upload`，`form-data`，参数 `file`、`groupId`、`name`。
- 创建航线分组：`POST /fczx-server/api/route/v1/group/put`，参数 `parentId`、`name`；根目录 `parentId` 为 `default`。
- 飞行任务推送：`POST /fczx-server/api/waylines/v1/job/record`，重点字段 `jobId`、`name`、`beginTime`、`endTime`、`executeTime`、`completedTime`、`dockSn`、`dockName`、`droneSn`、`mediaCount`、`waylineType`、`status`。
- 飞行任务详情：`GET /openapi/v1/wayline/job/{jobId}`，结构与调度单详情中的 `flights[]` 一致。
- 航线点位：`GET /openapi/v1/wayline/polygon/line`，参数 `jobId` 必填，返回 `x/y/z` 点位数组。
